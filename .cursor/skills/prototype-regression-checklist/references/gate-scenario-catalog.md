# 通用 Gate 场景目录

> 本文件是 `prototype-regression-checklist` skill 的参考资产。
> 列出原型模块回归验收中常见的风险场景类型，帮助快速识别需要创建的 Gate。

---

## 核心场景（几乎每个模块都需要）

### 1. 空状态（Empty State）

**触发条件**：模块主实体不存在或数量为 0。

典型验收点：
- 列表页展示空状态占位区块 + 引导文案
- 摘要卡显示 0（非空白）
- 主操作按钮仍可用（如"新建"）
- 关联模块（如仪表盘）不将空状态实体计入统计

**金样本**：billing Gate 1（无收费计划）、documents Gate 1（无资料项）

### 2. 全部完成（All Completed）

**触发条件**：模块主实体全部达到终态（已结清/已通过/已完成）。

典型验收点：
- 终态实体的状态标识正确（badge 颜色、文案）
- 终态实体的操作按钮正确禁用
- 进度/完成率指标显示 100%
- 终态实体不参与批量操作
- 全选时自动跳过终态行

**金样本**：billing Gate 2（全部结清）、documents Gate 2（全部审核通过）

### 3. 逾期/过期（Overdue / Expired）

**触发条件**：实体的截止日已过或超期未完成。

典型验收点：
- 逾期行视觉强调（红色背景、红色文字、红色 badge）
- 逾期行在默认排序中置顶
- 催款/催办入口对逾期项可用
- 摘要卡"逾期/过期"使用红色强调
- demo 数据中包含逾期样本

**金样本**：billing Gate 3（逾期）、documents Gate 3（过期资料）

### 4. 权限不足（Permission Denied）

**触发条件**：不同角色对模块操作有不同的可见性与可用性。

典型验收点：
- 低权限角色的关键按钮 disabled + tooltip
- 高权限角色全部操作可用
- 批量操作遇到无权限项时跳过并显示原因
- 敏感操作记录操作人到审计日志（`[生产约束]`）

**金样本**：billing Gate 4、documents Gate 5

## 常见补充场景（按模块特性选用）

### 5. 并发修改（Concurrent Modification）

**触发条件**：多用户同时操作同一实体。

> P0 阶段通常为 `[生产约束]`，原型仅在文档中记录。

典型验收点：
- P0-CONTRACT 明确记录并发修改为生产约束
- MIGRATION-MAPPING 中 Repository 接口包含乐观锁参数
- 批量操作中单条失败原因包含"并发冲突"

**金样本**：billing Gate 5、documents Gate 7

### 6. 返回规则（Navigation / Return）

**触发条件**：用户从不同入口进入模块后的返回行为。

典型验收点：
- 主导航入口跳转路径正确
- 侧边栏当前项高亮
- 列表行链接跳转路径正确
- 浏览器后退行为正确
- 弹窗/面板操作不触发页面跳转
- 操作后 toast 反馈在当前页展示
- `[生产约束]` 筛选条件保留、滚动位置恢复

**金样本**：billing Gate 6（4 个子场景）、documents Gate 6（4 个子场景）

### 7. 共享/联动（Cross-entity Linkage）

**触发条件**：模块实体被多个上级实体引用，变更需要联动。

典型验收点：
- 共享实体状态变更时所有引用方联动更新
- 受影响实体在列表中额外标注
- 摘要卡聚合展示受影响计数

**金样本**：documents Gate 4（共享版本过期联动）

### 8. 跨模块动作文案一致性

**触发条件**：同一动作在多个入口（模块主页、案件详情 Tab、仪表盘）出现。

典型验收点：
- 动作文案在所有入口一致
- 状态 badge 色彩在所有入口一致

**金样本**：billing Gate 6E、documents Gate 6D

## Gate 推导流程

```
1. 读 P0-CONTRACT §状态与异常态 → 提取所有状态场景
2. 对照上述目录，确认每个场景对应哪个 Gate 类型
3. 必须包含：空状态 + 全部完成 + 逾期过期 + 权限不足
4. 按需包含：并发修改 + 返回规则 + 联动 + 文案一致性
5. 模块有特殊场景（不在目录中）→ 新增自定义 Gate
```

## 验收项编写要点

| 要点 | 说明 |
|------|------|
| 具体 | "逾期行背景使用红色高亮 `bg-[rgba(220,38,38,0.04)]`"，而非"逾期行有视觉强调" |
| 可判定 | 验收人能在 5 秒内判定通过/不通过 |
| 有引用 | 每项注明 P0-CONTRACT 章节编号 |
| 有标记 | `[原型]` / `[生产约束]` / `[跨模块]` |
| 有样本 | demo 数据中包含触发此场景的至少 1 条记录 |
