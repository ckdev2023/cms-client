# Gate 设计规则与常见风险场景模式

> 本文件辅助 `SKILL.md`，不可独立使用。

---

## 1. 通用 Gate 设计模式

每个原型模块的 REGRESSION-GATE.md 通常包含以下 6 类 Gate。并非每个模块都需要全部 6 类——根据 P0-CONTRACT 中"状态与异常态"章节按需取舍。

| # | Gate 类型 | 适用条件 | 典型 Gate 名 |
|---|----------|---------|-------------|
| 1 | 空状态 | 模块有"无数据"场景 | Gate N — 空状态（无 XX） |
| 2 | 全量完成 | 存在终态（全部结清/全部审核通过） | Gate N — 全部结清/全部通过 |
| 3 | 异常/逾期 | 存在逾期、过期、退回等异常态 | Gate N — 逾期/过期/退回 |
| 4 | 权限不足 | 多角色且操作受限 | Gate N — 权限不足 |
| 5 | 并发修改 | 多用户可同时操作同一数据 | Gate N — 并发修改 |
| 6 | 返回规则 | 多入口（全局导航、仪表盘、案件详情 Tab） | Gate N — 返回规则 |

### 特殊 Gate（按模块需要追加）

| Gate 类型 | 适用条件 | 示例模块 |
|----------|---------|---------|
| 共享版本联动 | 资料被多案件共享且有有效期 | documents |
| 批量操作部分失败 | 批量操作涉及多条记录且可能部分成功 | billing, documents |
| 跨模块数据一致性 | 数据同时展示在多个模块（如仪表盘卡片） | billing, documents |

## 2. Gate 验收项编写规则

### 具体性标准

**不够具体（不合格）：**
- "检查空状态显示正常"
- "逾期样式正确"
- "权限控制生效"

**足够具体（合格）：**
- "空状态下展示引导文案'请先在案件中配置收费计划'+ 主按钮「配置收费计划」"
- "逾期行背景使用红色高亮 `bg-[rgba(220,38,38,0.04)]`"
- "助理角色：审核通过、退回补正、标记 waived 按钮不可见或 disabled + tooltip'权限不足'"

### P0-CONTRACT 引用格式

使用 `§` + 章节编号 + `#` + 条目编号：
- `§16 #1` — 第 16 节第 1 条
- `§7.2` — 第 7.2 小节
- `§3.2, §12.2` — 多章节交叉引用

## 3. 标记分类判断流程

```
验收项 →
  能否在浏览器中打开原型直接看到/操作？
    → 是 → [原型]
    → 否 →
      是否需要后端/运行时才能验证（并发锁、审计日志、API 冲突）？
        → 是 → [生产约束]
        → 否 →
          是否依赖其他模块的实现？
            → 是 → [跨模块]
            → 否 → 重新评估（可能是 [原型] 但需要 demo 数据支持）
```

## 4. 返回规则 Gate 子场景清单

返回规则 Gate 通常包含以下 4 个子场景（参考 billing 和 documents 金样本）：

| 子场景 | 覆盖内容 |
|--------|---------|
| A. 全局入口 → 模块列表页 | 导航跳转、高亮、行内链接、面包屑/后退 |
| B. 仪表盘入口 → 模块页 | 卡片跳转、预筛条件、浏览器后退 |
| C. 页内操作（不跳转） | 弹窗打开/关闭、面板展开/关闭、批量操作结果、toast |
| D. 跨模块动作文案一致性 | 同一操作在不同入口的文案/图标/badge 一致 |

如模块有详情页（案件详情 Tab），追加：

| 子场景 | 覆盖内容 |
|--------|---------|
| E. 详情页内操作 | Tab 切换上下文保持、弹窗关闭后停留、操作后上下文不变 |

## 5. Gate 数量指引

| 模块复杂度 | 建议 Gate 数 | 说明 |
|-----------|-------------|------|
| 简单（单列表页，无异常态） | 3–4 | 空状态 + 权限 + 返回规则 |
| 中等（列表+详情，有异常态） | 5–6 | + 全量完成 + 异常态 + 并发 |
| 复杂（多入口，跨模块联动） | 6–8 | + 特殊场景（共享联动、批量部分失败） |

超过 8 个 Gate 时应考虑合并同类场景。
