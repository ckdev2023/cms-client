# 多 agent 任务编排中的 AI 陷阱

## 1. 假原子任务
表面拆分，实际仍然跨多层、多模块、多目标。

## 2. 隐藏共享文件
多个任务都需要改同一个中心文件，如：
- package.json
- lockfile
- routes.ts
- theme.ts
- barrel exports
- generated types

## 3. 上下文泄漏
任务文档依赖前面聊天历史，而不是显式输入。

## 4. 测试口号化
只写“确保通过测试”，不写命令。

## 5. 并行幻觉
为了并行而并行，忽略真实依赖和竞争写入。

## 6. 结果不可机读
任务只产出一段自然语言总结，后续工作流无法消费。

## 7. 省略合并阶段
认为子任务都成功了就能直接合并。

## 8. 提示词诱发扩 scope
使用“顺便优化”“一起清理”“补齐其他问题”之类措辞。

## 9. 架构盲区
本来需要先调整模块边界，却直接进入并行拆分。

## 10. 把技能当运行时
技能负责定义协议与模板，不负责真实执行 agent 生命周期。
