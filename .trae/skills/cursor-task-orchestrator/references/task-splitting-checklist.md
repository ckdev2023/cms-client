# 任务拆分检查清单

在输出最终多 agent 计划前，逐项检查：

## 原子性
- 每个任务是否只有一个主要目标
- 是否能在全新窗口独立执行
- 是否写清允许修改范围
- 是否写清非目标与禁止改动
- 是否有明确停止条件

## 并行安全性
- 是否先识别共享热点文件
- 多个任务是否会同时修改路由、主题入口、包清单、公共类型、生成文件
- 若有共享热点，是否已拆成串行前置任务
- 是否把不安全的并行改成串行或阶段化执行

## 可执行性
- 是否有具体测试命令
- 是否有明确结果工件
- 是否有回滚说明
- 是否避免“继续前文”“按之前共识”之类隐式上下文

## 常见反模式
- 一个任务同时改前后端、数据库、测试且没有进一步拆分
- 一个任务同时包含设计、实现、重构、修复
- 验收标准写成“基本可用”“符合预期”之类模糊表述
- 用总结代替机器可读结果工件
- 没有合并评审阶段
