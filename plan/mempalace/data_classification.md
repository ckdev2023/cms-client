# MemPalace 数据分类与采集动作

## 1. 目标

给 `MemPalace` 第一期知识采集定义统一分类标签，确保每份输入在进入索引前先被归到“直接允许、范围内允许、待脱敏候选、严格禁止”四类之一。

## 2. 分类表

| 分类 | 定义 | 典型示例 | 一期动作 |
|---|---|---|---|
| `C1-Direct-Allow` | 权威文本资料，且不包含真实客户、正式案件或财务敏感数据 | `P0/P1` 正文 Markdown、`docs/事务所流程/*.md` 场景资料 | 可直接采集，但仍受 `allowed_sources.md` 目录范围限制 |
| `C2-Scoped-Allow` | 与权威文档同层的结构化辅助工件，不单独升格为规则源 | `P0/P1` 需求门禁 `artifacts/*.json`、`artifacts/*.txt`、配置型 `*.config.json` / `*.config.yaml` | 可在白名单目录内采集，且必须依附对应权威正文使用 |
| `C3-Desensitize-Candidate` | 原始内容含业务经验价值，但只有在完全脱敏并经人工复核后才可能作为样本使用 | 改写后的教学案例、规则摘要、聚合统计、脱敏复盘 | 一期不得采集；只可作为后续试点候选 |
| `C4-Hard-Block` | 真实客户、正式业务、证件、财务、日志、备份或无法证明已脱敏的内容 | 客户资料、案件导出、收费流水、护照/在留卡、聊天导出、数据库 dump | 一期严格禁止采集 |

## 3. 判定顺序

1. 先判是否命中 `allowed_sources.md` 的目录和后缀；若不命中，默认不采。
2. 再判是否包含真实客户、正式案件、财务或证件敏感信息；命中则直接归为 `C4-Hard-Block`。
3. 若内容来自真实业务但已被改写，仍先归为 `C3-Desensitize-Candidate`，不得直接降为 `C1/C2`。
4. 只有同时满足“在白名单目录内”且“不含敏感业务数据”时，才能归为 `C1` 或 `C2`。

## 4. `C3` 的最小放行条件

`C3-Desensitize-Candidate` 若要进入后续试点，至少满足以下条件：

1. 内容已改写为不可回推真实客户或真实案件。
2. 直接标识符与可拼接线索均已删除或泛化。
3. 不附带原始截图、扫描件、表格、路径、编号或对照表。
4. 有人工复核记录，且复核人明确确认“不可逆、不可回推”。
5. 已获得后续任务单独放行；本文件本身不提供自动放行资格。

## 5. 默认升级规则

- 不能确定是否含敏感字段时，从宽采集是错误策略，必须升级到 `C3` 或 `C4`
- 不能证明来源合法且经过复核时，必须升级到 `C4`
- 二进制附件、Office 文档和导出文件若未先转换为受审文本，默认升级到 `C4`

## 6. 与来源优先级的关系

- 本分类解决“能不能采”
- `source-priority.md` 解决“采进来后谁更权威”
- 只有先满足本分类的放行条件，才有资格参与后续来源优先级裁决
