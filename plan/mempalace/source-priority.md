# MemPalace 来源优先级规则（冻结）

## 1. 目标

冻结 `MemPalace` 在回答问题时的知识源优先级、冲突裁决顺序和来源展示规则，避免把检索命中或原始材料误当成权威结论。

## 2. 三层来源优先级

| 层级 | 来源目录 | 角色 | 可否直接作为最终结论 |
|---|---|---|---|
| L1 权威定义层 | `docs/gyoseishoshi_saas_md/P0/`、`docs/gyoseishoshi_saas_md/P1/`、`docs/事务所流程/` | 冻结范围、业务规则、流程定义、场景资料说明 | 可以 |
| L2 编译结论层 | `docs/gyoseishoshi_saas_md/_output/` | 已整理的可复用结论、导航和回灌缓冲层 | 仅可在不与 L1 冲突时引用，不可覆盖 L1 |
| L3 原始输入层 | `docs/gyoseishoshi_saas_md/_raw/` | 会议纪要、讨论、碎片信息、待编译输入 | 不可以 |

固定优先级：

`L1 权威定义层 > L2 编译结论层 > L3 原始输入层`

硬规则：

- `_raw` 只能作为“待编译线索”或“冲突线索”，不可直接充当权威结论。
- `_output` 只能复述、汇总或加速定位权威结论，不得反向改写 `P0/P1/事务所流程`。
- 回答中一旦发现 L1 与 L2/L3 冲突，必须以 L1 为准，并显式标出低层来源已失效或待回灌。

## 3. 各来源层的角色差异

### 3.1 L1 权威定义层

- `P0/`：首版范围、共享底座、冻结规则的首要执行依据。
- `P1/`：在不改写 P0 底座的前提下，承接扩展范围、扩展模块和扩展验收。
- `docs/事务所流程/`：场景流程、资料清单、实务输入与业务材料参考。

### 3.2 L2 编译结论层

- 用于沉淀已经整理过的中间结论。
- 若结论已回灌到 L1，回答时应优先引用回灌后的 L1 文档，而不是继续停留在 `_output`。
- 若 `_output` 仍引用 raw 或历史文档，且未被 L1 明确接纳，只能当作辅助说明。

### 3.3 L3 原始输入层

- 只用于收集原始 PRD、会议纪要、IM 讨论和待核实假设。
- 只能触发“需要编译 / 需要确认 / 需要冻结边界”的动作，不能触发“直接实现 / 直接回答为既定规则”。

## 4. L1 内部冲突裁决顺序

先判定问题类型，再套用固定顺序；禁止跨类型混用。

| 问题类型 | 裁决顺序 |
|---|---|
| P0 范围、共享阶段、数据模型、页面职责、通用 Gate / 不变量 | `P0 > P1 > docs/事务所流程 > _output > _raw` |
| P1 扩展能力、经营管理签扩展步骤、P1 专属验收与落地顺序 | `P1 > P0 > docs/事务所流程 > _output > _raw` |
| 场景资料清单、签证类型差异、实务材料准备 | `docs/事务所流程 > P1 > P0 > _output > _raw` |

补充裁决规则：

- 若 `P1` 扩展定义与 `P0` 共享底座冲突，以 `P0` 底座为准，`P1` 只可在扩展层补充，不可重写。
- 若 `docs/事务所流程/` 与 `P0/P1` 的冻结规则冲突，以 `P0/P1` 为准，并把流程文档视为待收敛来源。
- 若同一结论只能由 `_output` 或 `_raw` 支撑，且 L1 没有冻结定义，回答必须降级为“暂存结论/待确认”，不能伪装成既定规则。
- 若冲突无法通过上表收敛，应停止扩写，记录冲突点，不自行脑补统一口径。

## 5. 回答时必须展示的来源层级标记

每个最终结论都必须带来源层级标记，推荐使用以下格式：

- `[L1:P0]`
- `[L1:P1]`
- `[L1:事务所流程]`
- `[L2:_output]`
- `[L3:_raw]`

使用规则：

- 最终结论至少包含一个 `L1` 标记。
- 若答案引用了 `_output` 作为整理桥接，必须同时补上对应的 `L1` 标记。
- 若仅能找到 `_raw`，必须明确写成“待编译 / 待确认”，并只使用 `[L3:_raw]`，不得输出为确定口径。

## 6. 正反例

### 6.1 正例

- “P0 范围内案件主阶段仍以 `Case.stage = S1-S9` 为准，不能被经营管理签扩展步骤改写。” `[L1:P0]`
- “经营管理签 Step 15-20 属于 P1 扩展，下签后收费与续签提醒规则应按 P1 文档承接，但不能改写 P0 的共享阶段模型。” `[L1:P1][L1:P0]`
- “某签证场景所需资料项当前来自场景清单文档，且 P0/P1 尚未冻结同名字段时，可按事务所流程作为场景输入参考。” `[L1:事务所流程]`

### 6.2 反例

- “因为 `_raw` 里有会议纪要，所以把该说法直接当正式规则。” `[错误：L3 不可直接定案]`
- “因为 `_output` 里已经整理过一次，所以可以覆盖 P0 的冻结规则。” `[错误：L2 不可覆盖 L1]`
- “把 P1 的扩展步骤直接当成 P0 的共享主阶段。” `[错误：扩展层不可改写底座层]`

## 7. 最小执行协议

回答前按以下顺序执行：

1. 先判断问题属于 `P0`、`P1` 还是 `事务所流程` 场景输入。
2. 按第 4 节裁决顺序选择首要来源。
3. 只有在 L1 不完整时，才向 L2/L3 下探。
4. 输出时给出来源层级标记。
5. 若最终只拿到 `_raw`，则停止定案，明确“待编译/待确认”。
