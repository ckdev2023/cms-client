# MemPalace 禁止采集与脱敏边界

## 1. 目标

为 `MemPalace` 第一期内部知识记忆层冻结禁止采集范围、可脱敏候选范围和最小脱敏原则，确保一期不接入真实客户正式业务数据。

## 2. 硬规则

- 一期只做内部知识记忆层，不接真实客户、正式案件、正式收费、正式证件数据
- 对边界不确定的内容，默认判为“禁止采集”或“待脱敏”，不得先采后分
- 即使内容位于仓库内、由内部成员生成，若包含真实业务敏感信息，仍按禁止或待脱敏处理

## 3. 严格禁止采集类型

| 类型 | 典型内容 | 禁止原因 | 一期策略 |
|---|---|---|---|
| 真实客户与潜在客户身份信息 | 姓名、公司名、地址、电话、邮箱、生日、国籍、在留卡号、护照号、案件编号 | 可直接识别自然人或法人，且可与业务过程关联 | 严格禁止进入 `MemPalace` |
| 正式业务案件数据 | 正式 `Lead/Customer/Case` 导出、案件状态流转记录、任务执行日志、提醒记录、上传资料目录 | 属于正式运行数据，不是知识层权威定义 | 严格禁止进入一期范围 |
| 证件、申请与签证材料 | 护照、在留卡、申请表、理由书原件、入管回执、签证结果文件、扫描件与截图 | 高敏感个人与案件材料，且常包含多重标识符 | 严格禁止进入一期范围 |
| 收费与支付数据 | 报价单、账单、发票、付款流水、银行信息、税务编号、尾款状态明细 | 包含财务敏感信息与真实交易轨迹 | 严格禁止进入一期范围 |
| 原始聊天与会议导出 | AI 对话导出、IM 聊天记录、邮件导出、会议录音转写，只要带真实客户或案件细节 | 原始语料噪声高且常混入未脱敏身份与决策上下文 | 严格禁止直接采集 |
| 全量导出、备份与日志 | 数据库 dump、CSV/Excel 全量导出、审计日志、对象存储清单、备份压缩包 | 无法保证最小暴露面，且常带批量敏感字段 | 严格禁止直接采集 |
| 未审阅二进制附件 | `.xlsx`、`.xls`、`.docx`、`.pdf`、图片、压缩包、邮件附件 | 难以稳定解析，且容易夹带敏感信息、批注或隐藏元数据 | 一期默认禁止直接采集 |

## 4. 脱敏后可作为后续候选的内容类型

以下内容仅表示“理论上可在后续试点中评估”，不代表已进入一期白名单：

| 候选类型 | 允许前提 | 当前策略 |
|---|---|---|
| 真实案例改写后的教学样本 | 必须重写为不可回推到真实客户的教学案例，删掉全部直接标识符与可拼接线索 | 一期不采集，留待 `W16` 评审 |
| 从正式材料中提炼出的规则摘要 | 只保留规则、例外和流程结论；不得保留原文截图、原表格、原编号或具体申请记录 | 一期不采集，需人工复核后再评估 |
| 聚合统计或模式总结 | 只允许保留区间、比例、趋势，不保留单案粒度、精确时间线或可回推样本量过小的数据 | 一期不采集，后续试点评估 |
| 脱敏后的问答/复盘摘要 | 需改写为“问题-结论-依据”格式，并移除客户身份、案件编号、费用与时点细节 | 一期不采集，后续试点评估 |

## 5. 最小脱敏原则

1. 删除全部直接标识符：姓名、公司名、证件号、联系方式、地址、账号、编号。
2. 删除或泛化可拼接标识符：精确日期、具体国家组合、具体金额、精确时间线、唯一事件描述、文件路径。
3. 不保留可逆映射：禁止保留原始对照表、盐值表、替换词典、可回推占位符。
4. 不保留原始附件：截图、扫描件、原表格、原文档必须重写为文本摘要，不得只做局部打码。
5. 必须人工复核：未经过人工确认的自动脱敏结果，一律视为未脱敏。

## 6. 禁止例外

- 不允许因为“仅内部使用”而放行真实客户或正式案件数据
- 不允许因为“只抽取一部分字段”而放行数据库导出、Excel 导出或日志片段
- 不允许因为“文件在仓库里”而默认视为安全
- 不允许把 `_raw`、聊天导出、邮件摘录直接当作知识源
- 不允许将正式业务数据先写成 Markdown 再绕过本清单导入

## 7. 与白名单的关系

- `allowed_sources.md` 定义“一期允许直接采集”的目录与文件类型
- 本文定义“即使可读取，也不得采集”的红线和“必须先脱敏再另行评审”的灰线
- 若某内容同时命中白名单目录与本红线规则，以本红线规则为准
