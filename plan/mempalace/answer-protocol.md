# MemPalace 回答协议与回链规则

## 1. 目标

定义 `MemPalace` 在回答问题时的最小协议，确保答案始终先判断“能不能定案”，再决定“怎么回答”，并强制展示来源层级与回链位置。

本文只约束回答方式，不定义评估模板、不扩展 MCP 策略，也不修改任何提示词或客户端实现。

## 2. 硬规则

- 固定来源优先级为 `L1(P0/P1/事务所流程) > L2(_output) > L3(_raw)`。
- `_raw` 只能作为补充证据、待编译线索或冲突线索，不能直接充当最终规则。
- `_output` 只能作为桥接层，帮助定位或复述 `L1`，不能覆盖 `L1`。
- 任何最终结论都必须带来源层级标记，并尽量回链到权威文档。
- 若无法回到 `L1`，答案必须降级，不能伪装成“已冻结规则”。

## 3. 先失败、后成功的回答流程

每次回答按以下固定顺序执行：

1. 先判断当前问题能否形成最终结论。
2. 识别问题类型属于 `P0`、`P1` 还是 `事务所流程` 场景资料。
3. 先找对应 `L1`，只有 `L1` 不完整时才允许下探 `L2` 或 `L3`。
4. 先套用失败/降级规则；只有确认可以定案时，才输出成功回答。
5. 输出时强制附来源层级标记与回链位置。

## 4. 失败与降级协议

### 4.1 只命中 `_raw`

若某结论只能由 `_raw` 支撑，而 `L1` 没有冻结定义：

- 不得输出为既定规则。
- 必须明确写成“待编译 / 待确认”。
- 只能使用 `[L3:_raw]` 标记。
- 回答目标从“给结论”降级为“给线索 + 给待确认状态”。

推荐格式：

`当前只在原始材料中发现相关线索，尚无权威冻结结论，因此只能视为待编译 / 待确认。 [L3:_raw]`

### 4.2 只命中 `_output`

若结论只能在 `_output` 找到，而没有对应 `L1` 权威落点：

- 不得把 `_output` 当成最终权威。
- 必须明确写成“暂存结论 / 待回灌确认”。
- 可以引用 `[L2:_output]`，但不得伪装成 `[L1:*]`。

推荐格式：

`当前只找到编译结论层的整理结果，尚未回链到权威定义层，因此该说法只能视为暂存结论，待回灌确认。 [L2:_output]`

### 4.3 `L1` 与低层来源冲突

若 `L1` 与 `_output` 或 `_raw` 冲突：

- 必须直接以 `L1` 为准。
- 必须显式说明低层来源已失效、待回灌或仅作线索。
- 不得试图折中、拼接或脑补统一口径。

推荐格式：

`最终以权威定义层为准；低层来源仅保留为历史整理或待处理线索，不再作为当前规则依据。 [L1:*][L2:_output]`

### 4.4 `L1` 内部无法收敛

若 `L1` 内部存在冲突，且无法按既定裁决顺序收敛：

- 停止扩写。
- 明确指出冲突位置和未决状态。
- 不输出伪确定结论。

推荐格式：

`当前权威定义层内部仍存在冲突，暂不输出确定结论；需先收敛冲突文档后再定案。 [L1:*]`

### 4.5 完全无命中

若 `L1/L2/L3` 都没有可用依据：

- 直接回答“当前无可用依据”。
- 不得根据相邻主题脑补。
- 不得把常识性推测包装成仓库结论。

## 5. 成功回答结构

只有在可以定案时，才使用以下最小结构回答：

1. `结论`：直接回答问题本身，不只列文档名。
2. `依据`：说明为何以该来源为准。
3. `回链`：给出优先来源的文档路径，优先指向 `L1`。
4. `补充说明`：仅在需要解释冲突、边界或低层来源降级时补充。

推荐模板：

```text
结论：{直接结论} [L1:P0/P1/事务所流程]
依据：{为什么以该来源为准；如涉及冲突，说明低层来源为何降级}
回链：{优先填写权威文档路径；若用了 L2，仍需补上对应 L1}
补充说明：{可选；仅用于边界、冲突、待回灌说明}
```

## 6. 来源层级展示规范

### 6.1 标记格式

统一使用以下层级标记：

- `[L1:P0]`
- `[L1:P1]`
- `[L1:事务所流程]`
- `[L2:_output]`
- `[L3:_raw]`

### 6.2 展示规则

- 最终结论至少包含一个 `L1` 标记。
- 若引用 `_output`，必须同时补上对应的 `L1` 回链，不能只停留在 `[L2:_output]`。
- 若只能引用 `_raw`，必须明确降级，不得输出确定口径。
- 同一答案中，`L2` 和 `L3` 只能作为补充，不得放在结论句里单独充当最终依据。
- 回链优先写权威文档路径，其次才写 `_output` 或 `_raw` 的补充位置。

## 7. 冲突、缺失与降级的固定说法

为避免漂移，遇到以下场景时优先使用固定说法：

| 场景 | 固定说法 |
|---|---|
| 只有 `_raw` | `当前仅有原始线索，暂无权威冻结结论，需待编译 / 待确认。` |
| 只有 `_output` | `当前仅有编译结论层结果，尚未回链到权威文档，只能视为暂存结论。` |
| `L1` 与低层冲突 | `最终以权威定义层为准，低层来源仅作历史整理或线索。` |
| `L1` 内部冲突 | `权威定义层内部仍未收敛，暂不输出确定结论。` |
| 无任何依据 | `当前未检索到可支撑该问题的仓库依据。` |

## 8. 正反例

### 8.1 正例

- `P0 案件主阶段仍以 S1-S9 为准，经营管理签扩展步骤不能改写共享主阶段。 [L1:P0][L1:P1]`
- `该资料要求当前应以事务所流程场景清单作为首要参考；若涉及产品内扩展落地，再补充回链 P1。 [L1:事务所流程][L1:P1]`
- `当前只找到 _output 的整理结论，尚未看到对应权威落点，因此只能视为暂存结论。 [L2:_output]`

### 8.2 反例

- `因为 _raw 里提到过，所以这就是正式规则。`
- `因为 _output 整理得更完整，所以覆盖 P0。`
- `P1 经营管理签步骤可以替代 P0 的 S1-S9。`

## 9. 最小执行结论

- 回答前先判定“能否定案”，再决定“如何作答”。
- 回答成功的前提是至少命中一个可回链的 `L1`。
- `_output` 只能桥接，`_raw` 只能补充，二者都不能独立升级为最终规则。
- 一旦发现无法回到 `L1`，就必须显式降级，不再假装确定。
