# MemPalace 评估报告模板

## 1. 目标

定义 `MemPalace` 首轮评估报告的统一模板，用同一套字段记录命中率、误导率、回链质量、失败案例和最终 `Go/No-Go` 建议，避免不同试跑结果不可比。

本文只定义模板，不执行真实评估，不替代运维故障流程，也不直接决定是否进入试点。

## 2. 设计原则

- 字段越少越好，但必须支撑是否扩大范围的决策。
- 明确区分“未命中”和“命中但误导”，不能把二者混成同一类失败。
- 每条问题都要能回链到 `W12` 的题目 ID，并按 `W13` 的协议判断合格与否。
- 最终汇总必须能直接服务 `Go/No-Go` 评审，而不是只堆原始记录。

## 3. 核心评估维度

| 维度 | 记录目标 | 通过关注点 |
|---|---|---|
| 命中率 | 是否答到了问题本身 | 不是只返回关键词片段或文档列表 |
| 误导率 | 是否命中了但给错、越权或混层 | 不把 `_output/_raw` 冒充 `L1` |
| 回链质量 | 是否给出正确来源层级与权威回链 | 结论至少可回到一个 `L1` |
| 协议合规 | 是否遵守 `W13` 的成功/降级协议 | 失败时会降级，不假装确定 |
| 可复盘性 | 是否能复现失败并归因 | 每个失败案例都有问题、答案摘录、原因、改进动作 |

## 4. 指标口径

统一使用以下最小口径：

- `命中`：明确回答了问题本身。
- `未命中`：没有回答问题，只返回片段、噪声或无关内容。
- `命中但误导`：看似回答了问题，但来源层级错误、结论错误、冲突处理错误或把低层来源升格为最终依据。
- `合规回链`：答案至少给出一个正确的 `L1` 标记，并能回到对应权威文档。
- `协议违规`：未按 `W13` 进行降级、未标来源层级、或把 `_raw/_output` 伪装成最终规则。

建议汇总公式：

- `命中率 = 命中题数 / 总题数`
- `误导率 = 命中但误导题数 / 总题数`
- `合规回链率 = 合规回链题数 / 总题数`
- `协议合规率 = 协议合规题数 / 总题数`

## 5. 报告最小结构

每次评估报告固定包含以下五部分：

1. 评估元信息
2. 总体汇总
3. 分组结果
4. 单题记录
5. 失败与误导案例复盘

## 6. 评估元信息模板


## 9. 单题记录模板

每条问题至少记录以下字段：

| 字段 | 说明 |
|---|---|
| `question_id` | 对应 `W12` 的题目 ID，例如 `AQ-S04` |
| `question_group` | `来源追溯 / 规则解释 / 历史决策` |
| `question_text` | 原题文本 |
| `expected_signal` | `W12` 中的成功信号摘要 |
| `answer_excerpt` | 实际答案摘录，保留最少足够上下文 |
| `result` | `hit / miss / misleading` 三选一 |
| `backlink_layer` | `L1 / L2 / L3 / none` |
| `backlink_quality` | `pass / fail` |
| `protocol_compliance` | `pass / fail` |
| `redline_triggered` | `yes / no` |
| `failure_reason` | 失败或误导时必填 |
| `next_action` | 修复建议、补索引建议或提示词/策略复盘方向 |

## 10. 失败与误导案例模板

### 10.1 失败案例

用于记录“未命中”：

```text
案例编号：{F-01}
question_id：{AQ-*}
失败类型：miss
现象：{例如“只返回文档列表，没有回答问题”}
答案摘录：{最少必要摘录}
预期行为：{引用 W12 成功信号}
初步归因：{检索不到 / 命中片段不足 / 主题分类错误 / 其他}
建议动作：{补索引 / 调整权重 / 调整问题路由 / 其他}
```

### 10.2 误导案例

用于记录“命中但误导”：

```text
案例编号：{M-01}
question_id：{AQ-*}
失败类型：misleading
现象：{例如“把 _output 当成最终权威”}
答案摘录：{最少必要摘录}
误导点：{来源混层 / 结论错误 / 冲突处理错误 / 未降级}
是否触发红线：{yes/no}
正确口径：{引用 W13 协议或 W12 成功信号}
建议动作：{降权 / 增加 L1 回链约束 / 暂停放行某层来源}
```

## 11. Go/No-Go 决策模板

最终评审至少回答以下四项：

| 决策项 | 填写要求 |
|---|---|
| 当前建议 | `Go / Conditional Go / No-Go` |
| 主要依据 | 只写 3 条以内，必须包含指标或红线事实 |
| 允许范围 | 例如“仅允许 L1 试跑”“允许 L1+L2，不放行 L3” |
| 前置条件 | 若是 `Conditional Go`，列出放行前必须完成的动作 |

推荐写法：

```text
当前建议：Conditional Go
主要依据：
- 命中率达到 {xx%}，但误导率仍为 {xx%}
- 来源追溯类通过，历史决策类仍有明显缺口
- _raw 相关问题仍存在协议违规，暂不放行 L3

允许范围：{例如“仅允许 L1+L2”}
前置条件：
- {动作 1}
- {动作 2}
```

## 12. 红线判定

出现以下任一情况，整轮默认不得判为 `Go`：

- 把 `_raw` 当成最终权威来源。
- 把 `_output` 独立当成最终定案，且不给 `L1` 回链。
- 把 `P1` 扩展说成可改写 `P0` 底座。
- 对 `W12` 的必过题给出误导性答案。
- 未按 `W13` 做降级，仍输出伪确定结论。

## 13. 随机试填示例

### 示例 1

| question_id | result | backlink_layer | backlink_quality | protocol_compliance | redline_triggered | failure_reason | next_action |
|---|---|---|---|---|---|---|---|
| `AQ-S08` | `hit` | `L3` | `pass` | `pass` | `no` |  | 保持降级回答，不放大为权威规则 |

说明：该题虽然不是 `L1` 回链，但它的正确行为本来就是“只能降级到 `_raw` 线索”，因此只要明确标记 `[L3:_raw]` 且写成“待编译 / 待确认”，仍算协议合规命中。

### 示例 2

| question_id | result | backlink_layer | backlink_quality | protocol_compliance | redline_triggered | failure_reason | next_action |
|---|---|---|---|---|---|---|---|
| `AQ-H04` | `misleading` | `L2` | `fail` | `fail` | `yes` | 把 `_output` 当成最终权威，缺少 `L1` 回链 | 调整回答约束，强制 `_output` 只作桥接层 |

## 14. 最小执行结论

- 这份模板的核心不是“多记字段”，而是稳定区分 `miss` 和 `misleading`。
- 单题记录只保留能支撑复盘和决策的最小字段。
- 最终汇总必须能直接回答“现在能不能放行，最多放行到哪一层”。
