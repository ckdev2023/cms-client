## 结构性发现

### 强项

1. **规则解释类命中率最高**：12 题中 11 题有效命中，且 AQ-R04（score=0.769）为全场最高分，证明业务规则的语义检索链路已经可用。
2. **L1 命中率 100%**：所有 28 题都至少有 1 条 L1 来源命中，没有出现只命中 L2/L3 的情况。
3. **来源层级一致性**：所有结果都正确标记为 L1，无来源层级伪装。
4. **room 覆盖**：6 大高价值 room（state-machine, field-ownership, workflow-gates, biz-mgmt, scenario-materials, submission-audit）全部在至少一题中出现。
5. **红线零违规**：没有把 _raw 当最终权威、没有把 P1 说成可改写 P0 底座。

### 弱项与结构性缺口

1. **元治理问题无法回答**（AQ-S04, AQ-S08, AQ-H04, AQ-H08）：来源优先级、降级规则、索引策略等规则定义在 `plan/mempalace/` 下，不属于业务文档，不在 L1 索引范围内。这 4 题的失败是**结构性的**，不是检索质量问题。
2. **历史追溯能力不足**（AQ-H01, AQ-H02, AQ-H06, AQ-H07）：需要 _output 编译产出和时间线信息才能回答的"回灌链"问题，在 L1-only 索引下无法满足。
3. **低相关度长尾**：AQ-R12 (0.076)、AQ-H08 (0.173)、AQ-S04 (0.192) 的 top hit score 过低，说明嵌入模型对部分中文专业术语组合的语义捕获不够精确。
