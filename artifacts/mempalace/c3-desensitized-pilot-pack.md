# MemPalace C3 脱敏样本试点准备包

## 1. 前提与触发条件

本文件仅在以下条件**全部满足**时激活：

1. `go-no-go-review.md` 当前结论为 `Conditional Go` 或更高
2. 首轮稳定观察期（至少 1 周）已完成，无 R1–R3 级故障
3. L1 索引范围内的业务规则检索增强链路维持可用
4. 元治理文档索引方案已评估（`source-priority.md`、`answer-protocol.md` 是否纳入索引）

当前状态：`Conditional Go` 已通过，观察期尚未开始。本文件为观察期结束后的脱敏样本试点做提前准备。

## 2. C3 分类回顾

依据 `data_classification.md` §2：

> `C3-Desensitize-Candidate`：原始内容含业务经验价值，但只有在完全脱敏并经人工复核后才可能作为样本使用。

C3 的五项最小放行条件（`data_classification.md` §4）：

| # | 条件 | 验证方式 |
|---|------|---------|
| 1 | 内容已改写为不可回推真实客户或真实案件 | 人工逐条审查，确认无法还原原始身份 |
| 2 | 直接标识符与可拼接线索均已删除或泛化 | 自动扫描 + 人工复核 |
| 3 | 不附带原始截图、扫描件、表格、路径、编号或对照表 | 文件类型检查 + 内容审查 |
| 4 | 有人工复核记录，且复核人明确确认"不可逆、不可回推" | 填写本文件 §6 复核记录表 |
| 5 | 已获得后续任务单独放行；`data_classification.md` 本身不提供自动放行资格 | 本文件通过评审后方可激活 |

## 3. 候选样本类型

依据 `blocked_sources.md` §4，以下四类内容在完成脱敏和人工复核后，可作为后续试点候选：

### 3.1 教学案例样本

| 字段 | 说明 |
|------|------|
| 类型 | 真实案例改写后的教学样本 |
| 来源 | 从已结案的经营管理签 / 技人国 / 家族滞在等案件经验中提炼 |
| 脱敏要求 | 改写为虚构身份、虚构公司、泛化时间线；不保留原始案件编号或对照表 |
| 索引用途 | 丰富场景资料检索的教学维度，补充 `docs/事务所流程/*.scenarios/` 中的理论案例 |
| 首批候选数量 | 2–3 条（经营管理签新规 1 条、续签 1 条、技人国 COE 1 条） |

### 3.2 规则摘要样本

| 字段 | 说明 |
|------|------|
| 类型 | 从正式材料中提炼的规则摘要 |
| 来源 | 从入管窗口反馈、实务经验中提炼的"通过/拒否"判定规则 |
| 脱敏要求 | 只保留规则和例外结论；不保留原文截图、原表格、原编号或具体申请记录 |
| 索引用途 | 补充 L1 规则解释类检索的实务验证维度 |
| 首批候选数量 | 2 条（经营管理签事业计划书常见拒否理由摘要、资料补正常见模式摘要） |

### 3.3 聚合统计样本

| 字段 | 说明 |
|------|------|
| 类型 | 聚合统计或模式总结 |
| 来源 | 从多案件中提炼的处理时间区间、通过率趋势等聚合数据 |
| 脱敏要求 | 只保留区间、比例、趋势；不保留单案粒度、精确时间线或样本量过小（<10）的数据 |
| 索引用途 | 为历史决策类问题提供统计性参考背景 |
| 首批候选数量 | 1 条（经营管理签申请处理周期区间摘要） |

### 3.4 问答复盘样本

| 字段 | 说明 |
|------|------|
| 类型 | 脱敏后的问答/复盘摘要 |
| 来源 | 从事务所内部讨论、复盘会议中提炼的"问题-结论-依据"结构 |
| 脱敏要求 | 改写为"问题-结论-依据"格式；移除客户身份、案件编号、费用与时点细节 |
| 索引用途 | 补充历史追溯类检索的经验维度 |
| 首批候选数量 | 1–2 条（补正循环处理经验摘要、COE 海外发送异常处理经验摘要） |

## 5. 脱敏检查清单

每条候选样本在提交复核前，必须逐项通过以下检查：

| # | 检查项 | 判定标准 | 不通过时动作 |
|---|--------|---------|-------------|
| D1 | 直接标识符清除 | 无姓名、公司名、证件号、联系方式、地址、账号、编号 | 退回改写 |
| D2 | 可拼接线索清除 | 无精确日期、具体国家组合、具体金额、精确时间线、唯一事件描述、文件路径 | 退回泛化 |
| D3 | 不可逆映射 | 无原始对照表、盐值表、替换词典、可回推占位符 | 退回删除 |
| D4 | 无原始附件 | 无截图、扫描件、原表格、原文档；仅含改写文本 | 退回重写 |
| D5 | 格式合规 | 纯 Markdown 文本，无二进制嵌入 | 转换为纯文本 |
| D6 | 内容价值 | 脱敏后仍保留足够业务经验价值，对检索增强有实际帮助 | 评估是否值得纳入 |
| D7 | 样本量安全 | 聚合统计来源样本量 ≥10，无法从统计结果反推单案 | 退回泛化或放弃 |

## 8. 试点索引规则

通过复核的 C3 样本进入试点索引时，必须遵守以下规则：

| 规则 | 说明 |
|------|------|
| 独立 source_entry | 在 manifest 中新增 `c3-pilot-samples` 条目，`phase1_mode=enabled`，`authority_layer=L1-pilot`，`classification=C3-Desensitize-Candidate` |
| 独立 wing | 使用 `c3-pilot` wing，与 L1 权威文档的 wing 区分 |
| 检索标记 | 检索结果中 `source_layer` 标记为 `L1-pilot`，明确区分于 `L1` |
| 不覆盖 L1 | C3 样本不得覆盖或改写 L1 权威文档的结论 |
| 可撤回 | 任何 C3 样本在试点期间可随时从索引中撤回，无需走 release 升级流程 |
| 最大数量 | 首批试点最多 5 条样本，超出需重新评审 |

## 9. 从试点到正式的升级路径

| 阶段 | 条件 | 动作 |
|------|------|------|
| 试点准备 | 本文件通过评审 | 开始制作首批 3 条 C3 样本 |
| 脱敏复核 | 3 条样本全部通过 §6 复核 | 允许进入试索引 |
| 试索引 | 索引后观察 1 周，无红线违规 | 收集检索命中率和使用反馈 |
| 扩展评估 | 试索引观察期通过 | 评估是否扩展到 P1 优先级样本 |
| 正式纳入 | 扩展评估通过 + Go/No-Go 复判 | C3 样本正式编入 manifest，wing 从 `c3-pilot` 迁入正式 wing |

## 10. 风险与红线

| 风险 | 应对 |
|------|------|
| 脱敏不彻底，残留可回推线索 | D1–D4 检查 + 人工复核双重把关；任何存疑项默认退回 |
| C3 样本被误当作 L1 权威结论 | 独立 source_layer 标记 + 检索结果明确标注"试点样本" |
| 试点范围失控扩大 | 首批最多 5 条硬限制；超出需重新评审本文件 |
| 脱敏后内容失去参考价值 | D6 检查项确认；无价值样本不纳入 |
