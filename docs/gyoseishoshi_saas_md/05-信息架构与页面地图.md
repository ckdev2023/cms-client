# 05 信息架构与页面地图

> 本文定义系统的导航结构、全局交互原则与分阶段页面收敛策略。
> 各模块的详细页面规格见 [06-页面规格](06-页面规格/)。

---

## 1. 一级导航

1. 仪表盘
2. 咨询线索（可选：如需覆盖销售/前台咨询转化）
3. 客户
4. 案件
5. 资料中心
6. 文书中心
7. 任务与提醒
8. 收费与财务
9. 报表
10. 系统设置
11. 客户门户

## 2. 分阶段导航建议

为避免完整信息架构与 MVP 范围冲突，按版本分层展示导航：

- **P0 主导航**：仪表盘、咨询线索、客户、案件、任务与提醒、收费与财务、系统设置（基础）
- P0 的 Group 管理建议放在"系统设置（基础）"内，而不是单独上主导航，避免首版导航膨胀
- **P0 次级入口**：资料中心、文书中心可先作为案件详情内强入口，减少首版导航负担
- **P1/P2 扩展导航**：报表、客户门户、独立资料中心/文书中心、高级设置

## 3. 全局交互原则

- 全局搜索应常驻顶部，至少支持案件编号、客户名、电话、邮箱、公司名检索
- 列表页统一支持：筛选、排序、保存视图、批量操作、导出权限控制
- 工作台与列表默认按"我的 / 本组 / 全部可见（管理员）"口径切换或过滤，避免把 Group 只做成一个静态展示字段
- 页面中的"内部可见/客户可见"信息必须有明确视觉区分，避免误发与误读
- 可见不等于可导出：导出/下载/分享属于敏感动作，应单独权限控制并强制留痕
- 敏感动作的默认交互（P0 建议冻结）：
  - 导出/下载/分享前要求选择导出范围（本页/筛选结果/选中项），并在敏感范围较大时要求填写原因
  - 导出任务化：大批量导出建议异步生成，结果可追溯（谁、何时、导出范围、是否成功）
  - 分享链接需具备有效期与撤销机制（即使 P0 仅内部导出，也应预留最小模型）
- 空状态页要明确下一步动作，例如"去建案""去催办""去生成模板文书"

## 4. 首版页面收敛原则

- P0 以内部后台工作台为主，不把客户门户作为首版主导航核心模块
- P0 强调"从案件详情完成主要操作"，资料中心、文书中心可以先通过案件页强入口承载
- 复杂报表、独立高级中心、门户高级交互统一后置，避免导航完整但能力不完整
- 首版模板演示与页面字段设计优先围绕"家族滞在""工作签证（技人国）"两类高频案件展开
