# 02 完整用户需求文档

## 1. 用户角色

### 1.1 行政书士 / 主办人
需求：
- 掌握案件全局
- 控制案件质量
- 降低遗漏和退件风险
- 快速审核资料和文书
- 查看全所案件进展和即将到期项目

### 1.2 助理 / 事务员
需求：
- 建档
- 维护资料清单
- 催办缺件
- 更新案件状态
- 记录沟通
- 推动案件流转

### 1.3 咨询顾问 / 销售 / 前台
需求：
- 记录咨询线索
- 识别业务类型
- 跟进签约
- 将咨询转正式案件

### 1.4 财务人员
需求：
- 管理应收与实收
- 查看案件收费节点
- 记录回款和发票信息
- 统计欠款和回款情况

### 1.5 客户本人 / 企业联系人
需求：
- 在线填写资料
- 上传材料
- 查看缺件
- 接收提醒
- 查看案件进度
- 查看费用与付款状态

---

## 2. 核心业务对象

### 2.1 客户
- 个人客户
- 企业客户
- 关联人（配偶、子女、担保人、法定代表人、人事担当）

### 2.2 案件
案件是系统的核心工作单元。一个客户可以对应多个案件，一个案件可以绑定多个关联人和多个文档对象。

### 2.3 资料项
资料项是“案件需要的材料要求”，不是单纯附件。

### 2.4 沟通记录
沟通必须沉淀为正式业务记录，而不是散落在备注里。

### 2.5 文书
文书需支持模板化、变量填充、版本管理和导出。

### 2.6 任务
案件推进依赖任务驱动，而不是仅靠状态。

### 2.7 期限
在留期限、补件期限、付款期限、预约日期等都需要统一管理。

### 2.8 收费记录
收费要和案件阶段联动，并支持分阶段收款。

## 3. 功能需求总览

### 3.1 客户资料采集
目标：标准化采集客户资料，减少重复录入。

需求：
- 支持个人/企业客户创建
- 支持自定义表单
- 支持客户在线填写
- 支持事务所代录
- 支持多语言
- 支持字段校验
- 支持证件上传
- 支持历史变更记录
- 支持客户关系映射

### 3.2 案件建档
目标：把咨询转为标准化案件。

需求：
- 从客户档案一键建案
- 自动生成案件编号
- 绑定主申请人、企业、关联人
- 指派负责人和协作人
- 设置优先级、标签、风险等级
- 上传委托合同、报价单
- 支持案件模板

### 3.3 资料清单管理
目标：清楚知道要什么资料、谁提交、是否通过。

需求：
- 按案件模板自动生成资料清单
- 支持内部资料和客户资料分离
- 支持状态管理
- 支持备注、样例、审核意见
- 支持多版本上传
- 支持到期提醒
- 支持完成率展示

### 3.4 缺件催办
目标：系统化催办，不依赖人工记忆。

需求：
- 自动识别缺件、过期件、不合格件
- 生成待催办列表
- 支持自动提醒与手动提醒
- 支持多语言消息模板
- 支持催办记录留痕
- 支持逾期升级提醒

### 3.5 与客户来回沟通
目标：统一沉淀所有案件沟通。

需求：
- 记录电话、邮件、面谈、微信/LINE 等沟通
- 支持案件时间线
- 支持内部备注与客户可见备注分离
- 支持@同事
- 支持沟通转待办
- 支持消息模板
- 支持附件关联

### 3.6 模板文书输出
目标：提升文书产出效率与一致性。

需求：
- 模板库管理
- 变量自动填充
- 多案件类型模板配置
- 条件段落
- 多语言模板
- 版本管理
- 导出 Word / PDF
- 定稿留痕

### 3.7 提交前校验
目标：降低提交错误和补正率。

需求：
- 校验必填字段
- 校验必需资料是否齐全
- 校验日期逻辑
- 校验证件有效期
- 校验收费节点
- 支持双人复核
- 生成校验报告

### 3.8 期限 / 更新提醒
目标：避免逾期，挖掘续签机会。

需求：
- 管理多种期限
- 自定义提醒规则
- 提醒负责人与客户
- 日历视图
- 90/60/30/7 天多节点提醒
- 自动创建续签待办或新线索

### 3.9 进度可视化
目标：让内部和客户都能看懂案件进度。

需求：
- 看板视图
- 时间线视图
- 阶段进度条
- 当前待办
- 下一步动作
- 风险提示
- 客户侧简化进度页

### 3.10 收费与回款
目标：让财务和案件协同运作。

需求：
- 报价单
- 分阶段收费
- 应收/实收/欠款管理
- 上传支付凭证
- 欠款提醒
- 发票记录
- 财务报表

### 3.11 留痕与审计
目标：满足内控、追责、风控和纠纷防范。

需求：
- 记录关键操作日志
- 记录操作人、时间、对象、变更前后
- 查看案件审计时间线
- 支持敏感操作审批
- 支持日志导出

## 5. 非功能需求

### 5.1 安全性
- 数据加密
- 附件权限控制
- 审计日志不可随意删除
- 二步验证
- 敏感字段脱敏

### 5.2 稳定性
- 大文件上传
- 自动保存
- 误删防护
- 历史版本恢复

### 5.3 易用性
- 步骤化表单
- 批量操作
- 搜索与筛选
- 移动端查看提醒与上传资料

### 5.4 可扩展性
- 自定义字段
- 自定义模板
- 新案件类型扩展
- 开放 API
- 多组织支持
