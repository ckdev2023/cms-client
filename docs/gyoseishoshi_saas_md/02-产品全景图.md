# 02 产品全景图

> 本文用一页串联完整业务流程，帮助读者快速建立端到端认知。
> 每一步标注了应阅读的权威文档，方便按需深入。

---

## 业务主链路

```mermaid
flowchart LR
    A[咨询录入] --> B[客户建档]
    B --> C[案件创建]
    C --> D[资料收集]
    D --> E[资料审核]
    E --> F[文书制作]
    F --> G[提交前校验]
    G --> H[正式提交]
    H --> I[审理跟踪]
    I --> J[出结果]
    J --> K[归档]
    K --> L[回款结算]
    L --> M[续签提醒]
```

## 各环节说明

| # | 环节 | 做什么 | 关键机制 | 权威文档 |
|---|------|--------|---------|---------|
| 1 | 咨询录入 | 接收咨询线索（Lead），记录初始信息与跟进动作 | Lead 生命周期 | [03 §2.1](03-业务规则与不变量.md#21-核心对象语义)、[04 §2](04-核心流程与状态流转.md#2-主要流程) |
| 2 | 客户建档 | Lead 转化为 Customer，完善个人信息与关联关系 | 转化规则、去重 | [03 §2.1](03-业务规则与不变量.md#21-核心对象语义)、[06-客户](06-页面规格/客户.md) |
| 3 | 案件创建 | 创建 Case，选择签证类型与申请类型，模板驱动生成资料清单与默认任务 | CaseTemplate、Stage=S1 | [03 §2.1](03-业务规则与不变量.md#21-核心对象语义)、[06-案件](06-页面规格/案件.md) |
| 4 | 资料收集 | 按资料清单催收客户材料，客户上传附件版本 | DocumentRequirement、Stage=S2 | [03 §7](03-业务规则与不变量.md#7-资料项治理规则)、[06-资料中心](06-页面规格/资料中心.md) |
| 5 | 资料审核 | 内部审核已上传资料，通过或退回补正，缺件催办 | 审核状态、催办循环、Stage=S3 | [04 §3](04-核心流程与状态流转.md)、[03 §7](03-业务规则与不变量.md#7-资料项治理规则) |
| 6 | 文书制作 | 模板生成文书（委任状、理由书等），审核定稿 | Template → GeneratedDocumentVersion、**Gate-A**、Stage=S4 | [03 §8](03-业务规则与不变量.md#8-文书与模板规则)、[06-文书中心](06-页面规格/文书中心.md) |
| 7 | 提交前校验 | 执行 Gate-B/Gate-C，检查硬性阻断项与软性提示 | ValidationRun、Stage=S5 | [03 §4](03-业务规则与不变量.md#4-校验门槛gate规则)、[04 §1.5](04-核心流程与状态流转.md#15-门槛校验gate触发点与流程串联) |
| 8 | 正式提交 | 生成提交包（SubmissionPackage），锁定所有引用版本，记录回执 | 提交包锁定、Stage=S6→S7 | [03 §14](03-业务规则与不变量.md#14-提交动作强规则)、[04 §5](04-核心流程与状态流转.md) |
| 9 | 审理跟踪 | 等待入管审查，记录进度更新与补正通知 | Stage=S7、补正提交包 | [04 §6](04-核心流程与状态流转.md) |
| 10 | 出结果 | 记录审查结果（许可 / 不许可 / 追加资料） | Stage=S8 | [04 §7](04-核心流程与状态流转.md) |
| 11 | 归档 | 案件归档，全量快照不可变 | Stage=S9 | [04 §8](04-核心流程与状态流转.md) |
| 12 | 回款结算 | 确认各收费节点，记录回款，处理欠款风险 | BillingPlan、PaymentRecord | [03 §6](03-业务规则与不变量.md#6-收费与欠款策略)、[06-收费与财务](06-页面规格/收费与财务.md) |
| 13 | 续签提醒 | 在留期限到期前自动提醒，可生成新咨询线索 | CaseDeadline、Reminder | [04 §9](04-核心流程与状态流转.md) |

## 三道门槛（Gate）

流程中设置三道质量关卡，用于阻断关键错误：

| 门槛 | 触发位置 | 作用 | 详细定义 |
|------|---------|------|---------|
| Gate-A | S3→S4（进入文书制作） | 确保关键资料项已通过审核 | [03 §4.3](03-业务规则与不变量.md#43-三道门槛gate规则定义) |
| Gate-B | S4→S5（进入提交前校验） | 确保文书已定稿、CaseParty 齐备 | [03 §4.3](03-业务规则与不变量.md#43-三道门槛gate规则定义) |
| Gate-C | S6（提交 / 生成提交包前） | 最终校验，生成不可变提交快照 | [03 §4.3](03-业务规则与不变量.md#43-三道门槛gate规则定义) |

校验结果分为**硬性阻断**（必须修复）和**软性提示**（风险提示，不阻断），详见 [03 §4.1–§4.2](03-业务规则与不变量.md#41-硬性阻断p0-最小集合)。

## 核心对象关系速览

```
Lead ──转化──→ Customer ──建案──→ Case
                  │                 │
          CustomerRelation    ┌─────┼─────────────────┐
          (客户侧关联关系)     │     │                 │
                        CaseParty  DocumentRequirement  BillingPlan
                        (案件关联人) (资料项)            (收费节点)
                                        │                   │
                                DocumentFileVersion    PaymentRecord
                                (附件版本)              (回款记录)
                                        │
                                SubmissionPackage ←── GeneratedDocumentVersion
                                (提交包)              (文书版本)
                                        │
                                 ValidationRun ←── ReviewRecord
                                 (校验记录)         (复核记录)
```

对象的详细语义见 [08-术语表](08-术语表.md)，字段与关系见 [07-数据模型设计](07-数据模型设计.md)。
