# 06 数据模型设计

## 1. 设计原则

- 以案件为中心
- 客户资料可复用
- 文档与资料项分离
- 沟通、任务、期限、收费独立建模
- 所有关键对象支持审计字段

---

## 2. 核心实体总览

建议至少包含以下实体：

- Customer
- Company
- ContactPerson
- Case
- CaseParty
- DocumentRequirement
- DocumentFile
- CommunicationLog
- Task
- Template
- GeneratedDocument
- Reminder
- BillingRecord
- PaymentRecord
- AuditLog
- User
- Role

## 4. 关键关系说明

- 一个 Customer 可以对应多个 Case
- 一个 Company 可以对应多个 Case
- 一个 Case 可以有多个 CaseParty
- 一个 Case 可以有多个 DocumentRequirement
- 一个 DocumentRequirement 可以有多个 DocumentFile
- 一个 Case 可以有多个 CommunicationLog
- 一个 Case 可以有多个 Task
- 一个 Case 可以有多个 GeneratedDocument
- 一个 Case 可以有多个 Reminder
- 一个 Case 可以有多个 BillingRecord
- 一个 BillingRecord 可以有多个 PaymentRecord
- 所有核心对象都应写入 AuditLog
