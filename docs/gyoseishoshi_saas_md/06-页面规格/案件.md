## 1. 典型任务

- **新建案件**：选择签证类型与申请类型，模板驱动生成资料清单
- **批量建案（家族签）**：一次为多个主申请人建案并绑定扶养者/保证人
- **推进案件阶段**：审核资料 → 生成文书 → 校验 → 提交
- **查看案件进展**：确认当前阶段、资料完成率、阻断项、关键期限
- **提交与归档**：生成提交包、上传回执、归档案件
- **收费回款**：确认收费节点、登记回款

## 4. 关键动作

| 动作 | 触发条件 | 结果 | 权限 |
|------|---------|------|------|
| 新建案件 | 从客户详情或案件列表发起 | 创建 Case，生成资料清单，进入 S1 | 主办人、助理 |
| 批量建案（家族签） | 选择家族类案件进入向导 | 为每个主申请人创建 Case + 绑定 CaseParty | 主办人、助理 |
| 推进阶段 | 满足当前阶段完成条件 | Stage 前进，触发对应 Gate 校验 | 主办人 |
| 执行 Gate 校验 | 阶段推进触发 Gate-A/B/C | 生成 ValidationRun，通过或阻断 | 系统自动 |
| 生成提交包 | Gate-C 通过 + 复核通过（如启用） | 锁定资料/文书版本、生成不可变快照 | 主办人 |
| 上传回执 | 提交包已生成 | 关联回执，Stage → S7 | 主办人、助理 |
| 归档 | 出结果且确认完毕 | Stage → S9，全量快照不可变 | 主办人 |
| 标记资料项 waived | 资料项因条件变化无需提供 | 标记 waived 并记录原因，从完成率分母剔除 | 主办人 |
| 登记回款 | 收费节点已到 | 记录 PaymentRecord | 财务、主办人 |
| 跨组建案 | 选择非默认 Group | 须填写原因并审计留痕 | 主办人（需权限） |

## 5. 状态与异常

- **空状态（无案件）**：提示"去建案"并引导进入新建流程
- **Gate 校验失败**：高亮阻断项，提供修复入口与责任人指派
- **资料逾期未提交**：资料清单 Tab 标红逾期项，触发催办任务
- **补正通知**：自动生成补正任务，关联补正提交包
- **欠款状态下提交**：弹出风险确认弹窗，须完成留痕后方可继续
- **归档后只读**：S9 后除日志外全部字段不可编辑

## 6. 权限与可见性

| 角色 | 可见范围 | 可编辑范围 | 可导出 |
|------|---------|-----------|--------|
| 管理员 | 全所案件 | 全部字段 | 需留痕 |
| 主办人 | 本组 + 负责/协作的案件 | 负责案件全部字段 | 需留痕 |
| 助理 | 本组 + 协作案件 | 资料、沟通、任务等操作字段 | 受限，需留痕 |
| 财务 | 全所案件（仅收费相关字段） | 收费 Tab | 财务数据，需留痕 |

可见性口径受三维访问控制约束（角色 × Group × 负责人/协作者），详见 [03 §10.3](../03-业务规则与不变量.md#103-三维访问控制)。

## 附：家族签（多对象）建档与批量建案向导

> 覆盖"扶养者已有在留资格，配偶/子女分别办理家族滞在/配偶等"的高频场景。
> 口径：Customer 始终为"个人"，Case 始终为"针对该个人的一案"，家族关系通过两层表达——客户侧"关联人关系"（检索/跳转）+ 案件侧"CaseParty"（门槛校验/文书变量/审计）。

### 向导入口

- 咨询线索：已签约 → 转案件（选择"家族类/扶养类"意向后进入批量建案）
- 新建案件：选择家族类案件 → 进入"多对象批量建案"
- 客户详情：从客户发起"为关联人批量建案"

### 向导步骤（每人一案）

**Step 1：选择/新建关键关系人（扶养者/保证人）**

- 推荐按客户创建（Customer）：便于后续续签提醒与信息复用；若仅作为临时联系人，可用最小联系人信息承接，但不建议作为长期口径
- 必填：识别名/姓名、电话或邮箱、来源/介绍人、归属团队/项目组（group）
- 可选：在留期限到期日、备注

**Step 2：添加办理对象（主申请人列表）**

- 支持一次添加多个：配偶、子女
- 每个主申请人创建为独立客户（Customer）
- 建议必填：识别名/姓名、フリガナ、性别、生日（或证件号二选一）、联系方式（未成年人可选）

**Step 3：批量创建案件（Case）**

- 必填：案件类型（家族滞在/配偶/扶养类）、申请类型（认定/变更/更新）
- 规则：对每个主申请人生成 1 条案件（支持批量创建）
- 案件自动继承：group、来源、默认负责人（可调整并留痕）

**Step 4：为每个案件自动绑定案件关联人（CaseParty）**

- 必绑：主申请人 = 本案客户
- 必绑：扶养者/保证人（角色枚举明确）
- 可选：配偶/子女/同住等（用于文书变量与材料提供方拆分）

**Step 5：生成资料清单与模板变量**

- 资料清单按提供方分组：主申请人提供、扶养者/保证人提供、事务所内部产出
- 支持同一扶养者材料在多案间复用：多个案件引用同一"附件版本"，提交包锁定引用版本

### 门槛与补齐策略

向导允许先完成"建档 + 建案"，不在新增客户阶段阻塞：

- 系统自动生成"关键关系人信息补齐"任务
- 进入"生成文书/提交前校验"硬性阻断项：扶养者在留资格、在留期限到期日、在留卡影像（按模板要求）
- 对已带入的历史信息应触发更新确认：在留期限到期日、住址、雇主/收入类字段（如启用）

### 既有客户复用

> 目标：降低重复录入并避免带入过期信息。

入口：

- 新建案件：先选择扶养者客户 → 添加多个主申请人 → 批量建案
- 客户详情：从扶养者客户一键发起 → 新增配偶/子女并批量建案

复用与确认规则：

- 自动带入：扶养者在留资格、在留期限、联系方式、历史证件附件引用（在留卡/护照等）
- 附件时效提示：若证件附件距今超过配置阈值，提示重传并在资料清单中标记为待更新

### 结案后续：扶养者续签/变更跟进

触发：家族类案件结案/归档时（可按模板开关）

交互与动作：

- 默认生成：扶养者在留期限的 90/60/30/7 天提醒 + 跟进任务（联系确认续签意向）
- 可选动作：一键创建扶养者续签/变更案件（带入已沉淀的在留字段与证件材料引用）
