# 客户门户（P1）

> 客户门户作为 P1 优先增强模块上线，不纳入 P0 主导航与主流程。

## 2. 客户体验原则

- 少专业术语
- 明确下一步动作
- 明确资料示例
- 明确谁在负责
- 明确是否已收到和审核通过
- 默认只展示客户需要采取的动作与结果确认（资料项状态、退回原因、截止日、付款状态）
- 内部风险/审计明细不对客户展示，避免误解与敏感信息泄露

## 3. 客户侧信息范围（默认原则）

**展示**：

- 当前阶段、下一步动作
- 资料项清单（含样例/截止日/退回原因）
- 消息记录（客户可见部分）
- 付款状态
- 可下载的最终文书

**不展示**：

- 内部备注
- 风险等级
- 审计日志明细
- 内部协作信息（@同事等）
