## 1. 页面概述

- 页面：`customers/detail.html`
- 目标：在 `customer` 详情页承接经营管理签的签约前流程，完成 `咨询/问卷/报价/签约/转案件` 的高仿真演示。
- 入口：客户列表中的经营管理签客户行、后续从 `leads-message` 转入的客户详情。
- 边界：仍以 P0 客户页为主体，仅增加 P1 承接卡片与建案门禁，不扩展真实后端、真实问卷引擎、真实合同流。

## 2. 列表/主视图字段

| # | 字段/列头 | 规格字段 | 控件类型 | 必填 | 可见性 | 备注 |
|---|---|---|---|---|---|---|
| 1 | 客户名称 + 身份标签 | `displayName` / `bmvProfile.intakeStatus` | 文本 + Chip | 否 | 始终可见 | 经营管理签客户显示 `经营管理签问卷`、`报价待确认` 等标签 |
| 2 | 客户编号 / 国籍 | `id` / `nationality` | 文本 | 否 | 始终可见 | 保持 P0 客户表结构 |
| 3 | 联系方式 | `phone` / `email` | 文本 | 条件必填 | 始终可见 | 电话/邮箱至少一项 |
| 4 | 案件概览 | `cases.total` / `cases.active` | 链接/文本 | 否 | md+ | 未签约前允许为 `累计 0 · 活跃 0` |
| 5 | 最近联系 | `lastContact.date` / `lastContact.channel` | 文本 | 否 | md+ | 经营管理签前置动作会回写此字段 |
| 6 | 负责人 / 分组 | `owner` / `group` | 文本/Chip | 是 | md+/lg+ | 沿用 P0 |

## 3. 搜索与筛选

| 维度 | 控件 | 默认值 | 说明 |
|---|---|---|---|
| 搜索 | 搜索框 | 空 | 继续搜索客户名 / 假名 / 电话 / 邮箱 |
| 所属分组 | Select | 全部 | 沿用 P0 |
| 负责人 | Select | 全部 | 沿用 P0 |
| 活跃案件 | Select | 全部 | 对经营管理签前置客户允许落在 `无活跃案件` |

## 4. 操作列表

| # | 操作名 | 控件 | 前置条件 | 副作用 | 约束 |
|---|---|---|---|---|---|
| 1 | 发送问卷 | 按钮 | 当前客户存在 `bmvProfile` | 更新问卷状态、写沟通记录、更新最近联系 | demo-only，本地持久化 |
| 2 | 生成报价 | 按钮 | 已发送问卷 | 更新报价状态、写操作日志与沟通记录 | 不生成真实报价单 |
| 3 | 确认签约 | 按钮 | 报价已生成 | `signStatus -> signed`，开放建案 | 原型以按钮模拟签约完成 |
| 4 | 转正式案件 | 主按钮 | `signStatus = signed` | 跳转 `case/create.html` 并带上经营管理签模板参数 | 签约前不可建案 |
| 5 | 一键建案 / 批量建案 | 顶部按钮 | 无 P1 时按 P0；有 `bmvProfile` 时同样受签约门禁约束 | 跳转建案页 | 不允许绕过 P1 门禁 |

## 5. 表单/弹窗字段

| # | 字段/列头 | 规格字段 | 控件类型 | 必填 | 可见性 | 备注 |
|---|---|---|---|---|---|---|
| 1 | 前置阶段 | `bmvProfile.intakeStatus` | 状态文本 | 是 | 详情页 | 取值：`问卷准备中 / 报价待确认 / 待签约 / 可建案` |
| 2 | 问卷状态 | `bmvProfile.questionnaireStatus` | 状态文本 | 是 | 详情页 | 取值：`未发送 / 已发送 / 已回收` |
| 3 | 报价状态 | `bmvProfile.quoteStatus` | 状态文本 | 是 | 详情页 | 取值：`待生成 / 已生成 / 已确认` |
| 4 | 签约状态 | `bmvProfile.signStatus` | 状态文本 | 是 | 详情页 | 取值：`待签约 / 已签约` |
| 5 | 来源线索 | `bmvProfile.sourceLeadId` | 文本 | 否 | 详情页 | 仅展示，用于说明承接来源 |
| 6 | 签证计划 | `bmvProfile.visaPlan` | 文本 | 否 | 详情页 | 例如 `4年 + 1年经营计划` |
| 7 | 报价金额 | `bmvProfile.quoteAmount` | 文本 | 否 | 详情页 | 原型仅演示文案，不做真实财务计算 |
| 8 | 下一步 | `bmvProfile.nextStep` | 文本 | 否 | 详情页 | 作为演示脚本引导 |

## 6. 状态与异常态

| 状态名 | 规格要求 | 当前原型状态 |
|---|---|---|
| 无经营管理签承接 | 不显示 P1 承接卡片 | 已支持 |
| 问卷未发送 | 报价按钮禁用 | 已支持 |
| 报价未生成 | 签约按钮禁用 | 已支持 |
| 未签约 | 建案按钮禁用，并明确提示 `签约前不可建案` | 已支持 |
| 已签约待建案 | 建案按钮启用 | 已支持 |
| 暂无关联案件 | 案件概览保留 `0` 状态，不视为异常 | 已支持 |

|---|---|---|

## 8. 权限与可见性

| 角色 | 可见范围 | 可编辑范围 | 可建案 |
|---|---|---|---|
| Admin | 全部 P0 + P1 信息 | 全部 | 是（受签约门禁） |
| Owner/主办 | 本人客户 + P1 承接卡 | 问卷/报价/签约推进 | 是（受签约门禁） |
| 助理 | 仅查看与记录沟通 | 不可绕过签约门禁 | 否 |

## 9. P0 明确不做

| # | 不做项 | 后置到 | 原因 |
|---|---|---|---|
| 1 | 真实问卷发送与回收 | P1 生产实现 | 原型仅展示承接链路 |
| 2 | 真实报价审批与签章 | P1 生产实现 | 先冻结页面与门禁 |
| 3 | 合同附件上传与归档 | P1 生产实现 | 不在本次高仿真最小范围 |
| 4 | 企业客户主数据扩展 | 后续版本 | 当前仍以个人客户详情承接 |

## 10. 原型演示专属能力

| 能力 | 说明 |
|---|---|
| 本地持久化前置阶段 | 使用 localStorage 保存问卷/报价/签约推进 |
| 一键推进状态 | 用按钮直接模拟问卷、报价、签约完成 |
| 静态报价金额 | 仅用于高仿真展示，不作为生产财务规格 |
