## 1. P1 目标与边界

### 1.1 P1 目标

P1 的目标不是改写 P0，而是以 P0 为底座，为经营管理签新增模板专属扩展，使系统能够完整跑通：

`咨询 → 问卷 → 报价 → 签约 → 建案 → 资料 → 文书 → 提交 → 补正 → 下签后收费 → COE → 海外返签 → 在留期间 → 续签提醒 → 结案`

### 1.2 P0 / P1 分层原则

- **P0 保持管理层阶段**：`Case.stage = S1–S9`
- **P1 承接业务细节**：通过 `CaseWorkflowStep` 表达经营管理签专属步骤
- **P0 保持通用收费策略**：默认 `warn`
- **P1 承接专属硬门禁**：如 `COE_SENT` 前尾款必须结清
- **P0 保持手动提醒兜底**
- **P1 承接 ResidencePeriod + 自动提醒链路**

## 3. P1 模块清单

### M1 经营管理签模板

- 新增经营管理签 `CaseTemplate`
- 填充 `workflow_steps_blueprint`
- 填充 `extra_fields_schema`
- 填充 `requirement_blueprint`
- 绑定默认任务与校验规则

### M2 问卷与报价

- 启用 `DocumentRequirement.category=questionnaire`
- 启用 `survey_data`
- 增加 `visa_plan`、`quote_price` 等模板专属字段
- 支持从问卷结果进入签约/建案

### M3 资料清单与资料审核

- 经管签资料项蓝图
- 问卷类资料项与普通资料项统一参与完成率、审核、催办
- 继续沿用本地归档路径登记与版本追加

### M4 文书、校验与提交包

- 经管签专属文书模板
- 继续复用 Gate-A / Gate-B / Gate-C
- 继续复用 `ValidationRun` 与 `SubmissionPackage`
- 补正循环继续通过新提交包承接

### M5 经营管理签业务子步骤

至少启用以下步骤：

- `WAITING_MATERIAL`
- `MATERIAL_PREPARING`
- `REVIEWING`
- `APPLYING`
- `UNDER_REVIEW`
- `NEED_SUPPLEMENT`
- `SUPPLEMENT_PROCESSING`
- `APPROVED`
- `WAITING_PAYMENT`
- `COE_SENT`
- `VISA_APPLYING`
- `ENTRY_SUCCESS`
- `VISA_REJECTED`
- `RESIDENCE_PERIOD_RECORDED`
- `RENEWAL_REMINDER_SCHEDULED`

### M6 收费节点与 COE 守卫

- 明确定义 `final_payment` 节点
- 设置 `BillingPlan.gate_trigger_step=COE_SENT`
- 设置 `gate_effect_mode=block`
- 未结清尾款不得推进到 `COE_SENT`

### M7 海外返签与结果

- `COE_SENT → VISA_APPLYING`
- 登记返签结果：`ENTRY_SUCCESS / VISA_REJECTED`
- 海外拒签直接进入失败结案分支

### M8 在留期间与续签提醒

- 启用 `ResidencePeriod`
- 写入 `period_start / period_end / residence_years / card_number / entry_date`
- 基于 `reminder_schedule_blueprint` 自动生成 180 / 90 / 30 天提醒

### M9 结案与异常兜底

- `S9` 仍作为统一归档阶段
- 通过 `result_outcome / close_reason` 区分成功与失败
- 成功结案前置条件：入境成功、已录入在留期间、已生成续签提醒
- 提醒创建失败时不得自动进入成功结案

## 4. 建议实施顺序

### 4.1 P1-A：先跑通 Step 1–14

优先完成：`M1 + M2 + M3 + M4 + M5(基础版)`

目标：跑通经营管理签从咨询、签约、建案到提交、补正、结果登记的主链路。

### 4.2 P1-B：再补齐 Step 15–20

优先完成：`M6 + M7 + M8 + M9`

目标：跑通下签后收费、COE、海外返签、在留期间、续签提醒与结案收敛。

---

## 5. 最小验收收敛口径

### 5.1 P1-A 完成标准

- 经营管理签案件可创建并生成专属资料清单
- 问卷与报价数据可结构化保存
- 文书、校验、提交包、补正循环可跑通
- 业务子步骤与 `S1–S9` 并行显示且不冲突

### 5.2 P1-B 完成标准

- 发 COE 前尾款未结清时被硬阻断
- 海外返签成功 / 失败两条路径都能收敛
- 入境成功后可录入 `ResidencePeriod`
- 自动生成 180 / 90 / 30 天提醒
- 提醒创建失败时不得进入成功结案

---

## 6. 收敛结论

经营管理签不是对 P0 的替代，而是建立在 P0 通用底座之上的**首个 P1 专属模板扩展**。

执行上应保持以下口径：

- **P0 负责主链路底座与通用对象**
- **P1 负责经营管理签专属步骤、专属字段、专属收费门禁与在留期间链路**
- **完整满足原流程文档，以本文作为 P1 规划入口，以 `P0/` 作为底座权威来源**
