## 1. 走查总览与建议顺序

### 走查路线图

```
第一轮（主成功路径 · 约 45 分钟）
  场景 A → B → C → D → E → F（含 G）

第二轮（异常分支 · 约 30 分钟）
  场景 H → I → J → K → L → M → N

第三轮（边界态验证 · 约 15 分钟）
  场景 O → P → Q
```

### 浏览器准备

- 在本地启动原型 HTTP 服务（例如 `npx serve packages/prototype` 或直接用文件协议）
- 推荐 Chrome DevTools 打开 Console，观察 `console.log` 输出的状态变更日志
- 每个场景独立可走，不依赖上一场景的持久化状态（原型为 demo 模式，刷新即重置）

## 附录 B：跨模块断点走法

原型各模块之间的跳转为 demo 级别（toast 模拟），需手动衔接：

| 断点 | 走法 |
|------|------|
| leads「转为案件」→ case 建案 | leads 上完成转化 toast 后，**手动打开** `case/create.html` 继续建案向导 |
| leads「转为客户」→ customers | leads 上完成转化 toast 后，**手动打开** `customers/index.html` 验证 |
| case 详情 → 客户详情 | case 关联人无直接跳转，**手动打开** `customers/detail.html` |
| case 列表 → 建案 | 点击「新建案件」按钮跳转 `create.html` |
| 建案成功 → 案件详情 | 成功 banner 的「查看案件详情」链接，或手动打开 `detail.html?sample=family` |
