### 2.2 详情页 `detail.html`

#### Tab 结构

| Tab | 内容 |
|-----|------|
| 基础信息 | 线索编号、姓名、电话、邮箱、来源、介绍人、业务类型、Group、负责人、备注 |
| 跟进记录 | 跟进时间线（电话/邮件/面谈/IM）；每条含摘要、结论、下一步、下次跟进 |
| 转化信息 | 去重结果面板 + 已转客户/已转案件卡片 + 转化历史 |
| 日志 | 状态/人员/Group 变更记录 |

#### 可点击动作

| # | 元素/按钮文案 | 操作说明 | 业务 Step | 预期反馈 | 可用条件 |
|---|-------------|---------|----------|---------|---------|
| LD1 | 「转为客户」按钮 | 打开转化弹窗；预填姓名/电话/邮箱/Group | Step 5（签约→转客户） | toast「客户已创建（示例）」 | status ≠ lost；未已转 |
| LD2 | 「转为案件」按钮 | 打开转化弹窗；预填主申请人/业务类型/负责人/Group | Step 5→6（签约→建案） | toast「案件已创建（示例）」 | status ≠ lost；需先转客户 |
| LD3 | 「标记流失」按钮 | 标记线索为 lost | — | toast「已标记流失（示例）」→ readonly banner 出现 | status ≠ lost |
| LD4 | 状态切换按钮 | 变更线索状态 | Step 1–5 | toast「状态已更新（示例）」 | status ≠ lost |
| LD5 | 编辑信息按钮 | 编辑基础信息字段 | Step 2 | toast「信息已更新（示例）」 | status ≠ lost |
| LD6 | 「查看客户」链接 | 跳转至 customers/index.html | — | 页面跳转 | 已转客户时显示 |
| LD7 | 「查看案件」链接 | 跳转至 case/detail.html | — | 页面跳转 | 已转案件时显示 |

#### 详情页样本场景（8 个）

| 样本 key | 状态 | Banner | 按钮状态 | 特殊验证点 |
|---------|------|--------|---------|-----------|
| `following` | 跟进中 | 无 | 全部 enabled | 3 条跟进记录（电话/邮件/面谈） |
| `lost` | 已流失 | readonly 灰色 | 全部 disabled | 只读态验证 |
| `signed` | 已签约(未转化) | warning 黄色 | 转化按钮高亮 | 签约未转化警告 |
| `converted-customer` | 已签约(已转客户) | 无 | 转客户→查看客户 | 转化卡片显示 CUS-2026-0195 |
| `converted-case` | 已签约(已转案件) | 无 | 查看客户+查看案件 | 转化卡片显示 CAS-2026-0210 |
| `dedup-lead` | 新咨询 | 无 | 全部 enabled | 去重面板：电话匹配 Lead |
| `dedup-customer` | 新咨询 | 无 | 全部 enabled | 去重面板：邮箱匹配 Customer |
| `empty-followups` | 新咨询 | 无 | 全部 enabled | 跟进记录为空态 |

| Tab | 内容 |
|-----|------|
| 基础信息 | 法定姓名、假名、生年月日、国籍、性别、联系方式、地址、Group |

### 4.2 建案向导 `create.html`

#### 向导步骤

| Step | 内容 | 可点击/输入 |
|------|------|-----------|
| 1. 模板选择 | 选择签证类型模板卡片 | 「家族滞在」「技人国」卡片（P0 高频） |
| 2. 申请类型 | 选择认定/变更/更新 | 单选按钮 |
| 3. 基础信息 | Group、负责人、关联客户、案件到期日、金额 | 下拉/输入/搜索客户 |
| 4. 关联人确认 | 显示预填的关联人列表 | 可编辑/新增/删除 |
| 5. 资料清单确认 | 按模板展开的资料清单预览（分 provider 分组） | 勾选/取消勾选 |
| 6. 确认建案 | 汇总预览 + 确认按钮 | 「确认创建」按钮 |

#### 特殊模式

| 模式 | 入口 | 说明 |
|------|------|------|
| 家族签批量建案 | `create.html#family-bulk` | 锁定扶养者/保证人 → 为多个家庭成员批量建案 |
| 跨组建案 | 选择非当前 Group | 需填写跨组原因 + 审计留痕 |

#### 预期反馈

| 动作 | 反馈 |
|------|------|
| 确认建案 | toast「案件已创建（示例）」→ 跳转到案件列表 |
| 选择客户 | toast「客户已加入（示例）」→ 回填关联人 |

#### 模板与资料清单

| 模板 | 资料分组 |
|------|---------|
| 家族滞在 | 主申请人(3项) + 扶养者/保证人(3项) + 事务所内部(2项) |
| 技人国 | 主申请人(3项) + 雇主(3项) + 事务所内部(2项) |

| D5 | 「标记无需提供」按钮 | Waive 该资料项 | — | 状态变为 `waived` + 从完成率分母剔除；toast「已标记无需提供」 |

| 状态 | 标签 | badge | 出现场景 |
|------|------|-------|---------|

| `revision_required` | 退回补正 | red | 审核退回 |
