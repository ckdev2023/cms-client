# 客户模块（admin）— 浏览器自动化走查脚本

> 生成日期：2026-04-27
> 用途：为 `chrome-devtool-mcp` 与 `playwright mcp` 提供可直接消费的浏览器走查步骤、断言与取证要求
> 目标模块：`packages/admin/src/views/customers/`、`packages/admin/src/views/cases/`
> 关键代码锚点：`CustomerDetailView.vue`、`CustomerBasicInfoTab.vue`、`CustomerBmvIntakeCard.vue`、`query-create.ts`、`useCreateCaseModel.ts`
> 默认基准地址：`http://127.0.0.1:4173`

---

## 0. 执行约定（给 AI Agent）

### 0.1 统一环境

1. 打开 `http://127.0.0.1:4173/`
2. 执行：`localStorage.setItem("cms-admin-locale", "en-US")`
3. 刷新页面，确保界面文案使用英文；后续优先按英文文案定位按钮与 Tab
4. 若可控制浏览器网络面板，开启 `Preserve log` 与 `Disable cache`

### 0.2 结果记录规范

- 每个场景输出：`PASS` / `FAIL` / `BLOCKED`
- `FAIL` 时必须附带：
  - 当前 URL
  - 1 张截图
  - Console error 摘要
  - 失败请求摘要（status / url / method）
- `BLOCKED` 仅用于环境前提不满足，例如：
  - 本地服务未启动
  - `/api/feature-flags/resolve?key=bmv` 返回 `enabled=false`
  - 当前样本数据不具备“已签约可转正式案件”状态

### 0.3 定位优先级

1. **路由**：优先直接访问 `#/customers/...`、`#/cases/create`
2. **可见文案**：优先找英文文案，例如 `Basic info`、`Contacts`、`Record signing`
3. **URL query / hash**：用来验证跨模块透传
4. 非必要不要依赖 CSS class；仅在调试时使用

## 1. 样本数据与关键预期

| 样本 | 路由 | 角色 | 关键预期 |
|---|---|---|---|
| `cust-001` | `#/customers/cust-001` | 普通客户 | 可正常查看详情、可单建案、可家族批量建案 |
| `cust-004` | `#/customers/cust-004` | BMV 客户 | 默认存在 BMV 承接卡片；未签约时建案应被门禁；满足条件后可“转正式案件” |

### 1.1 BMV 场景前提

- BMV 卡片依赖 feature flag：`GET /api/feature-flags/resolve?key=bmv`
- 若该接口返回 `enabled=false`、403、网络错误，前端会将 BMV 视为关闭，相关场景应标记为 `BLOCKED`

### 1.2 关键跨模块契约

从客户详情进入建案页时，自动化至少要检查以下字段：

- 普通建案：`customerId`、`customerName`、`customerKana`、`customerGroup`、`customerGroupLabel`、`customerContact`
- 家族批量：在上述基础上，URL hash = `#family-bulk`
- BMV 转正式案件：
  - 必须：`templateCode=bmv`、`customerId`
  - 推荐：`sourceLeadId`、`ownerUserId`、`customerGroup`、`customerGroupLabel`
  - 可选：`templateId=bmv`（当签约已完成时）

## 3. 可直接执行的步骤脚本

### CUS-AUTO-001：客户列表 smoke

- 起始 URL：`#/customers`
- 步骤：
  1. 打开客户列表
  2. 确认能看到 `cust-001` 与 `cust-004` 对应的客户行
  3. 点击 `cust-001` 所在行进入详情
- 断言：
  - 当前 URL 变为 `#/customers/cust-001`
  - 页面存在客户姓名 `田中太郎`
  - 页面不存在白屏、整页报错、明显未捕获异常

### CUS-AUTO-002：普通客户详情 Tab 走查

- 起始 URL：`#/customers/cust-001`
- 步骤：
  1. 等待详情页稳定
  2. 验证默认可见 `Basic info`
  3. 依次点击 `Cases`、`Contacts`、`Communications`、`Activity log`
- 断言：
  - `Basic info` 页签下能看到基础字段区域
  - `Communications` Tab 中可见示例记录 `确认补件时间表`
  - `Activity log` Tab 中可见日志内容，如 `更新客户信息` 或同类操作日志
  - Tab 切换过程中无空白面板、无错误 toast

### CUS-AUTO-003：普通客户 → 单建案

- 起始 URL：`#/customers/cust-001`
- 步骤：
  1. 点击 header 主按钮 `Start case`
  2. 等待路由完成
- 断言：
  - 当前路由名对应 `#/cases/create`
  - URL query 包含：
    - `customerId=cust-001`
    - `customerName`
    - `customerKana`
    - `customerGroup`
    - `customerGroupLabel`
    - `customerContact`
  - URL query **不包含** `templateCode=bmv`
  - URL query **不包含** `templateId=bmv`

### CUS-AUTO-004：普通客户 → 家族批量建案

- 起始 URL：`#/customers/cust-001`
- 步骤：
  1. 点击 header 次按钮 `Batch create cases`
  2. 等待路由完成
- 断言：
  - URL 路径为 `#/cases/create`
  - URL hash 为 `#family-bulk`
  - query 仍包含 `customerId`、`customerGroup`、`customerGroupLabel`
  - query **不包含** `templateCode=bmv`
  - 建案页来源区可见 `Family visa bulk creation`

### CUS-AUTO-005：BMV 未签约门禁

- 前置条件：`/api/feature-flags/resolve?key=bmv` 返回 `enabled=true`
- 起始 URL：`#/customers/cust-004`
- 步骤：
  1. 等待详情页稳定
  2. 确认页面存在 BMV 承接卡片
  3. 记录页面中签约状态文案（例如 `Sign pending`）
  4. 尝试点击 header 中的 `Start case`
  5. 尝试点击 header 中的 `Batch create cases`
  6. 切到 `Contacts` Tab，尝试点击 `Batch create cases for contacts`
- 断言：
  - 三个入口都应被禁用或点击后不发生跳转
  - 页面仍停留在 `#/customers/cust-004`
  - 页面能看到门禁提示：`BMV customers must complete signing before a case can be created.`

### CUS-AUTO-006：BMV 转正式案件

- 前置条件：
  - BMV feature flag 已开启
  - `cust-004` 页面存在 BMV 卡片
- 执行策略：
  1. 进入 `#/customers/cust-004`
  2. 若页面已出现 `Create formal case`，直接点击
  3. 若未出现 `Create formal case` 但出现 `Record signing`，先点击 `Record signing`，等待详情刷新后再次查找 `Create formal case`
  4. 若执行完第 3 步仍未出现 `Create formal case`，记为 `BLOCKED`，并截屏保留当前 BMV 状态
- 成功路径断言：
  - 跳转到 `#/cases/create`
  - URL query 至少包含：
    - `templateCode=bmv`
    - `customerId=cust-004`
  - 若来源数据存在，还应检查：
    - `sourceLeadId`
    - `ownerUserId`
    - `customerGroup`
    - `customerGroupLabel`
  - 若当前状态已签约，允许并优先检查：`templateId=bmv`
  - 建案页来源区能看到 `Template locked`
  - 步骤一模板卡片中，当前模板保持选中，其他模板不可切换（disabled）
  - Step 3 / Summary 区负责人已优先继承 `ownerUserId`；若页面无明确前显值，则至少记录 Summary 中 `Owner` 显示值

### CUS-AUTO-007：请求失败兜底（可选）

- 建议做法：拦截客户详情请求并返回 500
- 拦截目标：匹配客户详情 GET 请求（路径通常为 `/api/customers/:id`）
- 起始 URL：`#/customers/cust-001`
- 断言：
  - 页面显示 `Couldn't load this customer right now.`
  - 页面显示 `Retry`
  - 页面显示 `Back to customers`
  - 恢复接口正常后点击 `Retry`，详情页可恢复正常

### CUS-AUTO-008：无权限兜底（可选）

- 建议做法：拦截客户详情请求并返回 403
- 起始 URL：`#/customers/cust-001`
- 断言：
  - 页面显示 `You do not have access to this customer.`
  - 页面显示 `Back to customers`
  - 页面不应呈现空白主内容区

---

## 4. 失败分级建议

| 类型 | 判定 |
|---|---|
| P0 阻断 | 无法进入客户详情；普通客户无法建案；BMV 门禁失效；BMV 转正式案件未带 `templateCode=bmv` |
| P1 重要 | 负责人/Group 继承错误；模板未锁定；异常态无 retry/back |
| 可记录观察项 | 文案细微偏差、非阻断样式错位、chunk size warning 等非功能问题 |

---

## 5. 建议输出格式（给执行 Agent）

每个场景建议按下面结构输出，便于后续汇总：

- `scenario_id`：如 `CUS-AUTO-006`
- `status`：`PASS | FAIL | BLOCKED`
- `current_url`：最终 URL
- `key_assertions`：通过/失败的断言列表
- `evidence`：截图路径、console 摘要、network 摘要
- `notes`：是否受 feature flag / fixture 状态影响
