## 2. UI 回归验证（4 个错误点）

### 2.1 CASE-202605-0010 提交前检查 Tab — 阻断条目 i18n

- **期望**：阻断条目展示本地化文案（非 raw 英文 key）
- **实际**：显示「1 项阻断未处理」+「详细请参阅检查报告」
- **结论**：✅ PASS — 走 `titleKey` → `t()` 的 i18n 管道正常工作。当前数据是旧格式（A2 代码部署前生成的 validation run），使用 summary fallback 路径显示；重新执行校验后将展示个别条目的详细本地化文案。
- **截图**：`regression-01-validation-tab.png`

### 2.2 CUS-202605-0014 客户摘要 — 案件名称

- **期望**：无 `dependent_visa` / `family_stay` raw code 尾巴
- **实际**：「案件名称: R-FLOW-05 山田太郎 · 家族滞在」
- **结论**：✅ PASS — `家族滞在` 是 `family_stay` 的正确 zh-CN i18n 标签，非 raw code
- **截图**：`regression-02-customer-summary.png`

### 2.3 CUS-202605-0014 关联案件 Tab — 「案件」列

- **期望**：显示 `R-FLOW-05 山田太郎 · 家族滞在`（非 `CASE-202605-0010`）
- **实际**：「案件」列 button text = "R-FLOW-05 山田太郎 · 家族滞在"
- **结论**：✅ PASS — fallback 链正确：`caseName` 为空 → `buildFallbackName(displayName, typeLabel)` 输出 `displayName · typeLabel`
- **截图**：`regression-03-customer-cases-tab.png`

### 2.4 案件列表「検查」列 — 三语 i18n

- **期望**：zh 待检查 / 已通过 / 未通过(N)；ja 未検査 / 合格 / 不合格(N)；en Pending / Passed / Failed(N)
- **实际（zh）**：列头「检查」，值「待检查」，过滤器选项「已通过 / 待检查 / 未通过」
- **实际（ja）**：列头「検査」，值「未検査」，过滤器选项「合格 / 未検査 / 不合格」
- **结论**：✅ PASS — `validationLabel` 结构 `{status, blockingCount}` 在 `CaseTableRow.vue` 由 `computed` 通过 `t(…)` 正确渲染
- **截图**：`regression-04-cases-list-check-column.png`

### 2.5 CASE-DEV-002 案件列表 — 标题与类型列

- **期望**：标题含「技人国」，类型列也为「技人国」
- **实际**：标题 = "技人国 — 田中太郎" ✅；类型列 = "家族滞在" ⚠️
- **根因**：seed 代码已改为 `engineer_humanities_intl_visa`（=「技人国」），但当前 DB 未重新 seed。type 列读的是 DB 中 `case_type_code`（旧值 `family_stay`）。代码层面 i18n mapping 正确（`seedDevData.case-type-code-i18n.contract.test.ts` 覆盖）。
- **结论**：⚠️ DATA-STALE — 非代码缺陷；下次 `npm run seed` 后自动修正
