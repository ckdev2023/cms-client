## P0 一句话

**为 1-5 人的小型行政书士事务所（以签证/入管业务为主），提供"从建档到提交"的案件作业系统，让事务所在 2-4 周内摆脱 Excel 主表管理案件。**

---

## P0 三个核心问题

| 问题 | 答案 |
|------|------|
| **谁先买？** | 1-5 人的小型事务所，以在留资格（签证/入管）为主营业务，目前用 Excel + 聊天工具管理案件 |
| **先跑什么？** | 一条案件主链路：咨询→建档→收资料→催资料→做文书→校验→提交→跟进→回款→留痕；2 类高频模板（家族滞在、工作签证/技人国） |
| **什么算成功？** | 1 家事务所在 2-4 周内用系统跑通至少 1 类高频案件的完整流程，产生留存或付费意愿 |

## P0 关键概念速览

首版只需理解以下 5 个核心对象（详细定义见 [08-术语表](08-术语表.md)）：

| 概念 | 一句话定义 |
|------|-----------|
| **Customer** | 个人客户主数据，一个客户可关联多个案件 |
| **Case** | 核心工作单元，一案一单，P0 只覆盖签证/入管类 |
| **DocumentRequirement** | "需要什么材料"的要求（≠ 文件本身） |
| **SubmissionPackage** | 一次对外提交的完整快照，生成后锁定 |
| **Stage** | 案件在主流程中的阶段（S1–S9） |

```
Customer ──建案──→ Case ──收资料──→ DocumentRequirement
                     │                      │
                     │              DocumentFileVersion
                     │              (附件版本)
                     │                      │
                     └──── SubmissionPackage ←── GeneratedDocumentVersion
                           (提交包)              (文书版本)
```

## P0 明确不做

以下内容不属于首版范围，在 P0 文档中不会出现：

- 客户门户（客户侧上传/查看）
- 企业客户主数据与批量案件
- 高级报表（补正率、成功率统计）
- 自动化流程引擎与条件触发器
- 字段级权限、审批流、模板审批
- 外部消息渠道深度集成（LINE / WhatsApp / 会计系统）
- Group 层级树与复杂组间审批
- 多语言支持

如需了解这些能力的长期规划，请查阅主目录 [02-版本范围与优先级](../02-版本范围与优先级.md) 的 P1/P2 章节。
