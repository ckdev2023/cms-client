## 1. 典型任务

- **查看我的待办**：确认今日和本周需要处理的任务
- **处理逾期任务**：快速定位已逾期未完成的任务
- **创建跟进任务**：手动为案件创建后续跟进任务
- **完成任务**：标记任务已完成

## 2. 列表页

### 2.1 字段

- 任务名称
- 所属案件
- 所属 Group
- 责任人
- 优先级
- 截止时间
- 完成状态（todo / doing / done / canceled）
- 来源（模板/催办/校验失败/补正/续签提醒/手动）

### 2.2 默认筛选

- 完成状态
- 截止时间范围（今日/本周/逾期）
- 责任人
- 所属 Group

### 2.3 批量动作

- 批量指派责任人
- 批量调整截止日
- 批量完成
- 批量取消（需原因）

## 3. 核心区块

### 工作台视图

- **我的待办**：按截止日排序的任务列表
- **今日到期**：截止日为今天的任务
- **已逾期**：截止日已过但未完成的任务

### 任务详情

- 任务描述
- 关联案件（可跳转）
- 责任人
- 截止日
- 操作记录

### 提醒日志

- 系统发出的站内提醒记录（发送时间、接收人、状态）

## 4. 关键动作

| 动作 | 触发条件 | 结果 | 权限 |
|------|---------|------|------|
| 创建任务 | 手动或系统自动 | 新建任务关联案件 | 主办人、助理 |
| 完成任务 | 任务处理完毕 | 状态→done，记录完成时间 | 任务责任人 |
| 指派责任人 | 新建或重新指派 | 更新责任人，站内通知新负责人 | 主办人 |
| 调整截止日 | 需要延期或提前 | 更新截止日并留痕 | 主办人 |
| 取消任务 | 任务不再需要 | 状态→canceled，须填原因 | 主办人 |

## 5. 状态与异常

- **空状态（无任务）**：显示"暂无待办"引导
- **逾期未完成**：列表标红逾期任务，仪表盘计入风险
- **批量自动生成**：模板建案或补正通知后系统批量生成任务，需确保不重复
- **提醒发送失败**：提醒日志中标记失败原因

## 6. 权限与可见性

| 角色 | 可见范围 | 可编辑范围 | 可导出 |
|------|---------|-----------|--------|
| 管理员 | 全所任务 | 全部操作 | 需留痕 |
| 主办人 | 本组 + 负责/协作案件的任务 | 创建、指派、完成、取消 | 需留痕 |
| 助理 | 分配给自己的任务 + 本组可见 | 完成自己的任务 | 不可 |

## P0 明确不做

| 不做项 | 后置到 | 原因 |
|--------|-------|------|
| 自定义提醒规则 | P1 | P0 用预置规则 |
| 外部触达（邮件/IM/短信） | P1 | P0 以站内提醒为主 |
| 高级催办升级链（多级主管） | P1 | P0 仅支持基础升级 |
| 本周到期视图 | P1 | P0 仅做今日到期和已逾期 |
