## 1. 典型任务

- **管理 Group**：新建、停用、重命名归属团队
- **配置可见性策略**：设置跨组建案/查看等基础权限开关
- **查看 Group 成员与关联统计**：确认各 Group 的活跃案件数与成员数

## 2. 列表页

### Group 管理列表

#### 2.1 字段

- Group 名称
- 状态（启用/停用）
- 创建时间
- 活跃案件数
- 成员数

#### 2.2 默认筛选

- 状态（启用/停用）

## 3. 核心区块

### Group 详情

- Group 名称、编号
- 状态（启用/停用）
- 成员列表
- 关联的客户与案件统计

### 可见性基础配置

- 是否允许跨组建案
- 是否允许负责人查看非本组协作案件

## 4. 关键动作

| 动作 | 触发条件 | 结果 | 权限 |
|------|---------|------|------|
| 新建 Group | 管理员发起 | 创建新 Group | 管理员 |
| 停用 Group | Group 不再使用 | 停用标记，禁止新对象选择该 Group | 管理员 |
| 重命名 Group | 管理员发起 | 更新名称并留痕 | 管理员 |
| 修改可见性配置 | 管理员发起 | 更新全局配置 | 管理员 |

## 5. 状态与异常

- **空状态（无 Group）**：提示"创建第一个团队"引导
- **停用有引用的 Group**：弹出确认提示，说明已被引用的客户/案件不受影响但不可再选择
- **不可物理删除**：已被客户/案件引用的 Group 只能停用，不能删除

## 6. 权限与可见性

| 角色 | 可见范围 | 可编辑范围 | 可导出 |
|------|---------|-----------|--------|
| 管理员 | 全部设置 | 全部配置 | 配置日志，需留痕 |
| 主办人 | 不可见 | 不可 | 不可 |
| 助理 | 不可见 | 不可 | 不可 |

## P0 明确不做

| 不做项 | 后置到 | 原因 |
|--------|-------|------|
| 高级权限管理（细粒度角色配置） | P1 | P0 用预置角色 |
| 审批流配置 | P2 | P0 不做审批流 |
| 企业客户开关 | P1 | P0 不引入企业客户实体 |
| 字段级权限 | P2 | 复杂度过高 |
| Group 层级树 | P2 | P0 只做单层治理 |
| 模板自定义管理 | P1 | P0 用预置模板 |
| 敏感导出权限独立配置 | P1 | P0 默认仅管理员可导出敏感数据 |
