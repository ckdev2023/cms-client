## 1. 典型任务

- **查看待审核资料**：快速找到客户已上传但尚未审核的附件
- **识别缺件**：按案件或全局查看缺失/退回的资料项
- **处理过期资料**：定位证件或材料已过有效期的资料项
- **发送催办**：对缺件/退回资料发起催办并留痕

## 2. 列表页（次级入口：跨案件资料视图）

### 2.1 字段

- 资料项名称
- 所属案件
- 提供方（主申请人/扶养者或保证人/事务所内部）
- 状态（待提交/已提交待审核/通过/退回补正/过期/waived）
- 截止日
- 最近催办时间
- 本地归档相对路径（可复制）

### 2.2 默认筛选

- 状态（待审核 / 缺件 / 过期）
- 所属案件
- 提供方

### 2.3 批量动作

- 批量发送催办
- 批量标记审核通过
- 批量标记 waived（需原因）

## 3. 核心区块（案件内资料清单 Tab）

案件详情 Tab 3 是 P0 资料管理的主操作界面：

- 资料项列表（按提供方分组）
- 每项：名称、状态、说明与样例、附件版本列表
- 附件版本列表（每次上传生成新版本，历史保留）
- 审核记录（通过/退回，含原因与操作人）
- 催办记录时间线

## 4. 关键动作

| 动作 | 触发条件 | 结果 | 权限 |
|------|---------|------|------|
| 审核通过 | 资料项已有附件待审核 | 状态→通过，计入完成率 | 主办人 |
| 退回补正 | 附件不符合要求 | 状态→退回补正，须填写原因 | 主办人 |
| 发送催办 | 资料项缺件或退回 | 生成催办记录并留痕 | 主办人、助理 |
| 标记 waived | 资料项无需提供 | 须记录原因码+文本，从完成率分母剔除 | 主办人 |
| 新增资料项 | 案件需要额外材料 | 在案件资料清单新增一项 | 主办人、助理 |

### 4.1 登记资料（本地归档）（P0 必做）

> **定义**：P0 默认不把文件上传到 SaaS 服务器；“登记资料”用于记录本地资料服务器的存放位置与版本元数据，并驱动资料项状态流转。

- 触发入口：案件详情「资料清单」Tab 的“登记资料（本地归档）”；资料中心可作为次级入口定位到对应资料项后执行同一动作
- 最小录入字段：关联案件、关联资料项、`relative_path`（相对路径，可复制）、文件名/描述、版本号（系统自动递增）
- 结果：生成新的 DocumentFileVersion（`storage=local_server`），资料项进入 `uploaded_reviewing`，等待审核

## 5. 状态与异常

- **空状态（无资料项）**：提示"该案件暂无资料清单，请先确认案件模板"
- **全部通过**：资料完成率显示 100%，允许推进阶段
- **存在逾期资料**：列表标红逾期项，触发催办升级提醒
- **waived 项追溯**：被标记 waived 的项仍可查看原因

## 6. 权限与可见性

| 角色 | 可见范围 | 可编辑范围 | 可导出 |
|------|---------|-----------|--------|
| 管理员 | 全所资料 | 全部操作 | 需留痕 |
| 主办人 | 本组 + 负责案件的资料 | 审核、催办、标记 waived | 需留痕 |
| 助理 | 本组 + 协作案件的资料 | 催办、上传 | 受限 |

## P0 明确不做

| 不做项 | 后置到 | 原因 |
|--------|-------|------|
| 独立跨案件资料中心（全功能） | P1 | P0 以案件内资料清单为主，次级入口仅提供快速定位 |
| 资料复用管理（跨案件附件复用台账） | P1 | P0 允许引用同一附件版本但不做复用管理 UI |
| 高级筛选（案件类型、负责人交叉筛选） | P1 | P0 用基础筛选 |
