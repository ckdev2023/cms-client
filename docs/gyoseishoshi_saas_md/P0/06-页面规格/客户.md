## 1. 典型任务

- **创建个人客户**：从线索转化或手动新建，录入基本信息与归属 Group
- **查找客户**：按姓名、フリガナ、电话、邮箱检索
- **维护客户信息**：更新联系方式、关联人关系
- **从客户发起建案**：一键建案或批量建案（家族签场景）
- **查看客户关联案件**：确认该客户下所有案件的阶段与负责人

## 2. 列表页

### 2.1 字段

- 客户名称（识别名/法定名）
- フリガナ
- 当前活跃案件数
- 最近联系时间
- 当前负责人
- 介绍人/来源
- 所属分组

### 2.2 默认筛选

- 负责人
- 所属分组
- 是否有活跃案件

### 2.3 批量动作

- 批量指派负责人
- 批量调整所属分组（需留痕）

## 3. 详情页

### Tab 1：基础信息

- 客户编号
- 识别名/姓名
- フリガナ
- 性别
- 生日
- 联系方式（电话、邮箱）
- 写真/头像
- 介绍人/来源
- 归属团队/项目组（Group）
- 备注

### Tab 2：关联案件

- 案件列表（编号、类型、阶段、负责人、更新时间）
- 入口：一键建案、批量建案（家族签向导）

### Tab 3：关联人（客户关系）

- CustomerRelation 列表（关系类型、关联客户、备注）
- 入口：新增关联人、为关联人批量建案

### Tab 4：沟通记录

- 时间线展示（渠道、摘要、下一步动作）
- 内部备注与客户可见备注分离（visible_scope 标记）

### Tab 5：操作日志

- 客户信息变更日志
- 关系变更日志

> P0 不设独立的"证件信息"Tab——证件附件通过案件资料清单管理，客户详情不重复维护证件台账。
> P0 不设"收费记录"Tab——收费数据通过案件详情收费 Tab 和收费与财务模块查看。
> P0 不设"资料附件"Tab（跨案件客户级附件汇总）——后置到 P1。

## 4. 关键动作

| 动作 | 触发条件 | 结果 | 权限 |
|------|---------|------|------|
| 新建个人客户 | 手动新建或线索转化 | 创建 Customer，继承 Lead.group | 主办人、助理、销售 |
| 编辑客户信息 | 客户已存在 | 更新主数据，变更留痕 | 主办人、助理 |
| 一键建案 | 客户详情入口 | 跳转建案表单，自动绑定客户 | 主办人、助理 |
| 批量建案 | 家族签场景，客户详情入口 | 进入家族签批量建案向导 | 主办人、助理 |
| 新增关联人 | 客户详情 Tab 3 | 创建 CustomerRelation | 主办人、助理 |
| 调整 Group | 客户详情或列表批量 | 变更归属，须留痕（不回写历史案件） | 主办人（需权限） |

## 5. 状态与异常

- **空状态（无客户）**：提示"录入第一个客户"或"从线索转化"
- **重复客户提示**：根据电话/邮箱匹配，新建时弹出去重提示
- **关联案件全部归档**：客户状态显示为"无活跃案件"，列表可筛选

## 6. 权限与可见性

| 角色 | 可见范围 | 可编辑范围 | 可导出 |
|------|---------|-----------|--------|
| 管理员 | 全所客户 | 全部字段 | 需留痕 |
| 主办人 | 本组 + 负责的客户 | 负责客户全部字段 | 需留痕 |
| 助理 | 本组客户 | 基础信息、沟通记录 | 受限，需留痕 |
| 销售/前台 | 本组客户（基础信息） | 基础信息 | 不可 |

可见性口径受三维访问控制约束（角色 × Group × 负责人/协作者），详见 [03 §10.3](../03-业务规则与不变量.md#103-访问控制维度)。

## P0 明确不做

| 不做项 | 后置到 | 原因 |
|--------|-------|------|
| 企业客户主数据 | P1 | P0 以个人客户为主；雇主信息沉淀在案件字段 |
| 客户门户关联 | P1 | P0 无客户门户 |
| 客户合并 | P1 | P0 仅做去重提示，不做物理合并 |
| 客户级跨案件附件汇总 | P1 | P0 附件通过案件资料清单管理 |
| 证件台账（独立 Tab） | P1 | P0 证件通过案件资料清单管理 |
| 批量导出 | P1 | P0 不做批量客户数据导出 |
