## 1. P0 产品定位

一套面向**小型行政书士事务所**的案件作业系统，首版聚焦将事务所分散在 Excel、聊天工具、Word、共享网盘中的**签证/入管案件作业流程**，统一沉淀到一个标准化系统中。

P0 不是通用 CRM，不是通用 OA，也不是完整的事务所管理平台。它是围绕**一条案件主链路**构建的最小作业闭环。

---

## 2. P0 目标客群

| 维度 | P0 聚焦 |
|------|---------|
| **事务所规模** | 1-5 人的个人事务所或小型法人 |
| **主营业务** | 在留资格（签证/入管）为主——家族滞在、工作签证（技人国）为首版模板覆盖 |
| **当前工具** | Excel 主表 + LINE/微信 + Word/共享网盘，无专业案件管理系统 |
| **角色构成** | 行政书士（主办人）、事务员/助理，可能有兼职财务 |

### P0 不优先服务

- 大型法人事务所（10 人以上、多部门、复杂审批流）
- 以公司设立、许可申报为主营的事务所（模板不在首版覆盖）
- 需要企业客户批量案件管理的场景
- 已有专业系统、需要迁移集成的事务所

---

## 3. P0 要解决的痛点

P0 首版聚焦 3 个最痛的问题，这 3 个问题是小型签证事务所日常中损失最大、频次最高的：

### 3.1 Excel 主表不可持续

案件一多，Excel 主表的行列式管理就崩溃：找不到最新版本、多人编辑冲突、无法追溯变更历史、案件状态靠人工维护。

**P0 解法**：用结构化的案件系统替代 Excel 主表，案件有阶段、有状态机、有变更留痕。

### 3.2 缺件催办混乱

不知道哪个客户还缺什么材料、催了几次、谁催的、客户回了没有。催办靠记忆和聊天记录翻找。

**P0 解法**：每个案件有结构化的资料清单，催办留痕，缺件可追踪。

### 3.3 提交版本不可追溯

提交给入管局的材料包，事后无法确认当时提交的是哪个版本的文件、哪个版本的文书。补正时不知道上次提交了什么。

**P0 解法**：提交包锁定引用版本，生成不可变快照，历史提交可复盘。

---

## 4. P0 核心价值

| 价值 | P0 提供什么 |
|------|-----------|
| **结构化** | 案件从 Excel 行列变为有阶段、有状态、有资料清单、有期限的结构化数据 |
| **可追溯** | 资料版本不覆盖、提交包锁定、操作留痕，所有变更有迹可循 |
| **防遗漏** | 缺件催办留痕、到期提醒、提交前校验，关键节点不再依赖人工记忆 |
| **可复用** | 案件模板驱动资料清单和文书生成，减少重复作业 |

### P0 不承诺的价值

- **自动化**：P0 不做自动化流程引擎、条件触发器、智能分派
- **门户化**：P0 不做客户门户，客户侧交互仍在线下完成
- **高度配置化**：P0 模板和规则为预置版，不要求事务所自定义

---

## 5. P0 业务范围

### 首版覆盖的案件类型

| 类型 | 预置模板 | 备注 |
|------|---------|------|
| 家族滞在（配偶/子女签证） | 有 | P0 首选高频案件 |
| 工作签证——技术·人文知识·国际业务 | 有 | P0 首选高频案件 |

### 首版不覆盖的案件类型

永住、归化、特定技能、经营管理、公司设立、许可申报等——这些案件的模板、资料清单和文书在 P1/P2 逐步扩展。

> 系统架构不阻止这些案件类型的手动创建（不选模板即可建案），但 P0 不预置模板、不保证流程完整。

---

## 6. P0 一句话产品定义

**为 1-5 人的小型签证事务所提供案件主链路作业系统，用结构化案件管理替代 Excel 主表，解决缺件催办混乱与提交版本不可追溯问题。**
