## 1. P0 导航目标

P0 的信息架构只服务一个目标：让首个试点事务所能在 **2-4 周内**用系统跑通从咨询到提交包的主链路，并把“今天该做什么”找得到、点得进、做得完。

设计原则：

- **主链路优先于模块完整**：先保证咨询、客户、案件、任务、收费这些主流程入口清晰。
- **重操作集中在案件详情**：资料、文书、校验、提交包优先收敛到案件详情，避免首版导航膨胀。
- **可见不等于可导出**：全局搜索、列表、详情、导出遵循同一权限边界。
- **页面结构服务培训**：一线用户需要按角色知道“从哪里开始、下一步去哪里”，而不是记住所有模块名。

## 2. P0 主导航

### 2.1 一级导航

| 导航项 | P0 定位 | 主要用户 | 说明 |
|------|---------|---------|------|
| 仪表盘 | 主导航 | 全角色 | 今日待办、逾期、待提交、风险案件总览 |
| 咨询线索 | 主导航 | 销售、前台、主办人、助理 | 承接签约前信息，转化为 Customer / Case |
| 客户 | 主导航 | 主办人、助理、销售 | 个人客户主数据、关系人、发起建案 |
| 案件 | 主导航 | 主办人、助理、财务 | 单案全流程总入口 |
| 任务与提醒 | 主导航 | 主办人、助理 | 处理催办、补正、校验修复、跟进任务 |
| 收费与财务 | 主导航 | 财务、主办人 | 回款、欠款风险、收费节点 |
| 系统设置 | 主导航（基础） | 管理员 | Group 管理、基础权限配置 |

### 2.2 不上主导航、但属于 P0 强入口的模块

| 模块 | 入口 | P0 口径 |
|------|------|---------|
| 资料中心 | 案件详情内强入口；案件列表快捷跳转 | 首版不独立上主导航，避免“主导航完整但使用分散” |
| 文书中心 | 案件详情内强入口；案件列表快捷跳转 | 与资料、校验、提交包同属单案操作链 |
| 校验与提交包 | 案件详情 Tab | 不做独立中心，避免把“提交”从案件上下文剥离 |
| 沟通记录 | 客户详情 / 案件详情 Tab | 首版不单独拆“沟通中心” |

### 2.3 P0 明确排除的导航模块

| 模块 | 后置到 | 原因 |
|------|-------|------|
| 客户门户 | P1 | P0 聚焦内部作业闭环 |
| 独立报表中心 | P2 | 首版不做复杂分析 |
| 独立资料中心 / 文书中心 | P1 | P0 先收敛在案件详情 |
| 高级设置（字段级权限、自动化规则、审批流） | P2 | 首版不做复杂治理能力 |

## 3. 页面地图

### 3.1 主链路页面地图

```text
仪表盘
  ├─ 我的待办 / 今日到期 / 已逾期 / 风险案件
  ├─ 待提交案件
  └─ 快速入口：新建线索 / 新建客户 / 新建案件

咨询线索
  ├─ 线索列表
  ├─ 线索详情
  └─ 转客户 / 转案件

客户
  ├─ 客户列表
  ├─ 客户详情
  │   ├─ 基础信息
  │   ├─ 关联案件
  │   ├─ 关联人
  │   ├─ 沟通记录
  │   └─ 操作日志
  └─ 从客户发起建案 / 批量建案

案件
  ├─ 案件列表
  └─ 案件详情
      ├─ 概览
      ├─ 基础信息
      ├─ 资料清单
      ├─ 文书
      ├─ 任务
      ├─ 期限
      ├─ 校验与提交
      ├─ 收费
      ├─ 沟通记录
      └─ 日志

任务与提醒
  ├─ 我的待办
  ├─ 今日到期
  └─ 已逾期

收费与财务
  ├─ 应收 / 实收 / 欠款
  └─ 风险确认留痕

系统设置
  └─ Group 管理
```

### 3.2 角色视角的推荐入口

| 角色 | 登录后优先进入 | 次级入口 |
|------|--------------|---------|
| 主办人 | 仪表盘 / 案件 | 校验与提交、收费、沟通记录 |
| 助理 | 任务与提醒 / 案件 | 资料清单、文书、沟通记录 |
| 销售/前台 | 咨询线索 / 客户 | 客户详情、转化入口 |
| 财务 | 收费与财务 | 案件收费 Tab |
| 管理员 | 仪表盘 / 系统设置 | Group 管理、风险聚合 |

## 4. 全局交互原则

### 4.1 全局搜索

- 固定放在顶栏。
- P0 最少支持：案件编号、客户名、电话、邮箱。
- 搜索结果必须与详情页使用**同一套服务端权限判断**，不能因为“搜得到”就突破对象可见边界。
- P0 不要求跨全库模糊搜索企业主数据；工作类雇主信息以案件字段辅助检索。

### 4.2 列表页统一原则

- 统一支持基础筛选、排序、分页。
- 默认视图按角色落在“我的 / 本组 / 全部可见（管理员）”。
- 批量动作只保留高频且低风险项；敏感批量动作后置到 P1。
- 空状态必须给出明确下一步动作，如“去建案”“去转化线索”“去补录回款”。

### 4.3 敏感动作统一原则

- 导出、下载、分享、跨组变更、负责人交接均属于敏感动作。
- 敏感动作默认要求二次确认，并写入审计日志。
- P0 不要求复杂审批流，但必须明确“谁可以做、做了能追溯”。

### 4.4 沟通与资料的现实补位原则

- P0 无客户门户，因此外部渠道收到的资料和沟通，允许由助理/主办人在系统内补录。
- 补录不等于无规则：至少要记录来源渠道、时间、责任人、摘要或对应资料项。
- 页面文案必须承认这一现实，避免把真实工作方式假装成“全部在线上完成”。

## 5. P0 导航冻结口径

以下口径在 P0 冻结，不因实现阶段讨论而随意变动：

1. **咨询线索属于 P0 主导航**，不能只在文档中存在、页面规格中缺席。
2. **资料中心 / 文书中心在 P0 作为案件详情内强入口**，不独立膨胀为一级导航。
3. **提交、补正、提交包查看必须在案件上下文内完成**，不做“独立提交中心”。
4. **客户门户、独立报表、高级设置不进入 P0 导航**。
5. **任何新增导航项都必须先回答：若删掉它，试点是否仍能跑通？**

---

## 6. 与页面规格的对齐要求

`06-页面规格/` 中的每一份 P0 页面文档都必须能映射到本文的一个导航入口或案件详情强入口。若范围文档存在某能力，但页面规格没有落点，则视为 **P0 尚未闭环**。

当前 P0 页面规格应至少覆盖：

- 仪表盘
- 咨询线索
- 客户
- 案件
- 任务与提醒
- 收费与财务
- 系统设置
- 资料中心（案件详情强入口）
- 文书中心（案件详情强入口）
