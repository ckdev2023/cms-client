# 03 MVP 功能优先级

## 1. MVP 目标

MVP 的目标不是一次性覆盖全部事务所场景，而是优先打通案件主链路：

**咨询转化 → 建档 → 收资料 → 催资料 → 做文书 → 校验/复核 → 提交快照 → 跟进 → 回款 → 留痕**

### 1.1 范围原则
- 首版优先保障**内部作业闭环**，先让事务所内部用户能够不依赖 Excel 主表完成主要工作。
- 首版优先实现"能跑通"的能力，而不是"全部高级配置化"；模板、规则、门户、自定义能力先做基础版。
- 与安全、审计、可见性直接相关的基础能力不应被视为"以后再做"，至少要有最小可用版本。

### 1.2 冻结口径

> 版本策略、对象边界、最小状态机、校验门槛分层、审计事件集合等冻结规则的完整定义见 [03-业务规则与不变量](03-业务规则与不变量.md)。以下仅列出与版本范围直接相关的边界摘要：

- P0 保留主链路广度，但控制单模块复杂度，不追求门户化、流程引擎化、强配置化一步到位。
- 客户门户放入 P1，不纳入 P0 范围。
- 双人复核能力在 P0 数据与流程层预留，但默认不做全局强制，由事务所或模板配置决定是否开启。
- 收费节点默认做提醒与风险提示，不作为统一提交阻断项。
- 首版仅预置 2 个高频案件模板：家族滞在、工作签证（技人国）。

## 2. P0（首版必须上线）

### 2.0 P0 验收口径

> 完整的冻结口径与不可变规则见 [03-业务规则与不变量](03-业务规则与不变量.md)。以下为验收时必须可被验证的关键点：

- "不覆盖历史"可被验证：提交包生成后，其引用的资料/文书版本与关键字段快照不可被覆盖式替换
- "可见≠可导出"可被验证：导出/下载/分享等敏感动作受权限控制且必留痕
- "下一步动作"可被验证：案件阶段展示与工作台聚合必须绑定下一步动作，避免只有状态没有行动

### 2.1 咨询线索（咨询转案件）
- 新建咨询线索（联系方式、来源/介绍人、意向业务类型、group、备注）
- 线索状态（新咨询/跟进中/待签约/已签约/已转案件/已流失）
- 线索转客户/转案件（含基础去重提示：电话/邮箱优先，默认继承 Lead.group）

### 2.2 客户管理
- 个人客户创建（P0 主体）；工作类案件的雇主/受入机构信息先沉淀在案件字段，企业主数据后置到 P1/P2
- 客户基础字段（识别名、フリガナ、写真、介绍人/来源、所属分组、备注、基础联系方式）
- 关联案件
- 关联人关系（客户-客户）：配偶/子女/父母/代理人等的轻量维护与快速跳转
- 基础搜索与筛选

### 2.3 案件管理
- 创建案件
- 绑定客户 / 关联人 / 雇主信息（工作类案件）
- 家族签/扶养类案件支持批量建案：一次选择多个主申请人生成多案，并为每案强制绑定关键关系人（扶养者/保证人）
- 案件编号
- 案件阶段（最小阶段状态机 → [03 §3.1](03-业务规则与不变量.md#31-案件阶段)）
- 负责人/协作者
- 默认继承客户 group，支持跨组建案/转组留痕
- 截止日期
- 下一步动作（用于工作台聚合）

### 2.4 资料清单管理
- 案件模板生成资料清单
- 内置通用材料字典与签证类型模板（基础版：预置家族滞在、工作签证（技人国）2 类高频模板，不要求事务所自定义）
- 资料项必须标记提供方（主申请人 / 扶养者或保证人 / 事务所内部），支持按提供方展示与催办留痕
- 资料状态切换（状态枚举 → [03 §3.2](03-业务规则与不变量.md#32-资料项状态)）
- 附件上传
- 审核意见
- 完成率展示（口径 → [03 §7.1](03-业务规则与不变量.md#71-完成率口径)）

### 2.5 缺件催办
- 待催办列表
- 手动发送催办
- 自动提醒规则基础版
- 催办记录

### 2.6 沟通记录
- 手工记录电话/消息/面谈/邮件摘要
- 案件时间线
- 内部备注

### 2.7 模板文书生成
- 模板库基础版
- 变量填充
- 导出 Word / PDF

### 2.8 提交前校验与提交包（基础版）
- 提交前校验（硬性阻断项最小集合 → [03 §4.1](03-业务规则与不变量.md#41-硬性阻断p0-最小集合)）
- 校验报告（基础展示）
- 提交包（提交日期、回执/凭证、锁定的资料版本/文书版本与校验结果）
- 复核能力预留（如启用则作为待提交前附加门槛，不默认对全部案件强制）
- 欠款策略（→ [03 §6](03-业务规则与不变量.md#6-收费与欠款策略)）

### 2.9 期限提醒
- 在留到期提醒
- 补件截止提醒
- 提交截止提醒
- 负责人提醒
- P0 默认以内部联系人与站内提醒为主，外部客户触达仅保留最小扩展空间

### 2.10 收费与回款
- 费用总额
- 收费节点
- 回款记录
- 欠款标记
- 默认提供提醒与风险标识，不将欠款统一作为提交硬阻断

### 2.11 审计日志
- 关键操作记录（事件类型 → [03 §5.1](03-业务规则与不变量.md#51-事件类型p0-必须覆盖)）
- 状态变更记录
- 上传与审核记录
- 提交包生成与导出记录
- 跨组建案/转组、负责人交接、导出/下载/分享、欠款风险确认等敏感动作记录

### 2.12 Group 与协作归属（基础版）
- Group 基础管理（创建/停用/重命名，入口放在系统设置）
- Lead → Customer → Case 的默认归属继承链（→ [03 §13.3](03-业务规则与不变量.md#133-归属与继承)）
- 跨组建案/转组留痕（操作人、时间、原因）

### 2.13 最小权限与可见性（基础版）
- 基础角色权限（管理员/主办人/助理/销售/财务）
- 客户可见 / 内部可见的数据分域
- 用户与 group 的最小可见范围（→ [03 §10.2](03-业务规则与不变量.md#102-可见性口径)）
- 敏感导出、下载、批量发送的留痕

### 2.14 工作台与全局检索（基础版）
- 我的待办 / 今日到期 / 已逾期 / 待我处理
- 按案件编号、客户名、电话、邮箱的全局检索
- 按 group 的过滤与基础聚合视图
- 风险案件与待提交案件的基础聚合视图

## 3. P1（首版后优先增强）

### 3.1 客户门户
- 客户在线上传资料
- 查看缺件
- 查看进度
- 查看付款状态
- 后续再扩展复杂沟通中心与多文书下载能力

### 3.2 高级校验与复核
- 有效期/日期逻辑/收费节点等扩展校验项
- 双人复核（通过/驳回原因）与关卡阻断
- 校验规则配置化（按案件类型/模板）

### 3.3 多语言支持
- 中/日/英表单与提醒模板

### 3.4 高级工作台
- 看板视图
- 即将到期聚合
- 风险案件视图

### 3.5 文书版本管理
- 初稿 / 审核版 / 定稿
- 模板版本历史

### 3.6 模板与材料库配置化
- 事务所级案件模板覆盖（增删改资料项、文案/样例、截止日规则）
- 模板变量字典与缺失策略治理（与文书模板协同）

### 3.7 续签商机自动创建
- 90/60/30 天提醒
- 自动创建续签跟进任务

## 5. MVP 明确暂不包含

- 外部消息渠道深度集成（LINE / WhatsApp / 会计系统 / 电子签约）
- 字段级权限、审批流、模板审批等高级权限体系
- Group 层级树、复杂组间审批、模板按 Group 的高级治理
- 通用自动化流程引擎与复杂条件触发器
- 面向大型企业客户的批量案件协作工作台
- 高度自定义的多组织后台管理能力

## 6. MVP 成功标准

如果首版上线后能做到以下几点，就说明 MVP 是成功的：

- 事务所可以不用 Excel 主表管理案件
- 案件资料清单可在线追踪
- 缺件可被系统提醒和留痕
- 模板文书能自动生成基础文档
- 到期案件不再依赖人工记忆
- 收费和案件进度能对应起来
- 至少一个事务所能完整跑通从建档到归档的流程
- 首版范围在有限模板与有限角色下可稳定交付，而不是因为过度扩张导致主链路失真

### 6.1 建议验收口径
- 从"已签约线索"到"生成首个提交包"可在系统内完成，不依赖线下主表补记关键节点
- 任一案件的资料、文书、校验、提交、回款均能追溯到责任人、时间和最新状态
- 不同角色登录后能看到与自身职责匹配的待办与风险聚合视图
- 至少能覆盖一类高频案件模板（如家族滞在/工作签证）的完整闭环
- 任一已生成的提交包，其引用的资料/文书版本在后续新增版本时不被覆盖；历史提交可复盘
- 任一导出/下载/分享与跨组变更动作均可在审计日志追溯到操作人、时间、对象与原因（如适用）
