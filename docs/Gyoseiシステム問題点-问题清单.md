## 咨询与会话

在 Word 原件中，本段标题通常显示为 **「一、咨询与会话」**（见上文 `numbering.xml` 说明）；下列 **2.** 起为作者**手打编号**。

### 2. 跟进记录：「录入跟进」按钮不好用

**正文**：跟进记录的【录入跟进】按键不好用。

**截图补充**（图 1：咨询线索详情 / 跟进记录 Tab）：

- 页面上下文：`咨询线索 / …`、`跟进记录` 与 `新增跟进记录` 表单（渠道、下次跟进时间、跟进摘要必填等）。
- 表单项观察：**渠道 / 跟进摘要** 带必填星号；**下次跟进时间** 无星号（若业务强制「下次跟进」则属**规则与 UI 标识不一致**的排查点）。
- 「录入跟进」位于表单右下，用户用红框标出；可与「点击无反应 / 难发现 / 与校验状态不一致」等具体现象对照复现。

**期望**：按钮可稳定触发提交，或与表单校验、禁用态提示一致。

## 客户与建档

### 4. 资料不全时补资料的入口

**正文**：当客户资料不**完整**（原文「不晚上」，疑笔误）无法建档时，能否设置链接直接点进去补资料。

**截图补充**（图 2：`签约并开始建档` 弹窗）：

- 背景为某条**咨询线索详情**（示例中出现线索名 **「liuliu」**、意向 **经营管理** 等，用于复现环境对齐）。
- 黄底提示示例文案：**无法创建经营管理签约案件**；要求先回客户档案完成后再返回签约建案。
- **门禁与主按钮一致性**：黄条已声明无法建案时，底部 **「确认创建案件」仍呈主按钮（蓝色）样式** 的截图状态，需与真实禁用/拦截逻辑对照，避免「看得到点得到却失败」或无明显反馈。
- 文中列出的阻塞点与后文第 8 条呼应，更清晰的可操作列表为：
  1. 客户问卷尚未回收；
  2. 报价尚未确认；
  3. 客户尚未完成签约；
  4. 经营管理签承接流程尚未就绪（提示刷新客户档案后重试）。

**期望**：从该弹窗到「补问卷 / 确认报价 / 登记签约 / 刷新就绪状态」有可点链或稳定路径，且完成后可回到当前签约建案上下文。

## 案件（分派与期限）

### 5. 「乱码」、布局错位与「跨组原因」含义不清

**正文**：案件第 3 步「分派与期限」处有乱码感；跨组原因意义不明，不跨组时如何填写不清楚。

> **术语**：正文「**案件-3**」与截图顶部 **「步骤 3/4：分派与期限」** 指同一步骤，下文化一便于与我们产品文案对照。

**截图补充**（图 3、图 5：`分派与期限`）：

- 页眉提示示例：**「分组继承自主申请人：…」**（分组来源规则可在规格里核对是否需在窄屏下换行展示）。
- **更可能是布局/渲染问题而非纯字符编码**：左侧标签区被压窄，出现**单字竖排**、**「收费金额 (税込)」「模板自动创建契约」等与复选框区域重叠**，可读性极差（用户描述的「乱码」可能指此现象）。
- **跨组原因**：界面上带红色 `*`（必填），但与「不跨组是否仍必填」的业务规则未在 UI 说明；图 5 中该区域仍与竖排重叠文案混在一起。
- **截止日期**：右侧「截止日期」与左侧「分组 / 负责人」**纵向不对齐**，上方大块留白，疑似栅格或表单项顺序问题。
- **页脚**：出现「步骤 3/4 — 尚有前置条件未满足」，但「下一步」类主按钮在图上仍可呈现为可点击样式时，需核对是否与真实禁用态一致（避免误导）。

**期望**：修复分派步骤表单的布局与标签可读性；明确跨组原因在「同组 / 跨组」下的填写规则；对齐截止日期与前置校验提示。

## 签约前门禁与页面导航

### 8. 从「前往客户详情」返回与关键子流程难找

**正文**：不满足签约前门禁时，点【前往客户详情】补全后无法回到本建档页，只能重填案件；找不到问卷回收、确认报价方案、签约手续等页面（原文「签约手续**也**页面」疑为「**页**面」笔误）。

**截图补充**（图 6：步骤 4/4 `签约前门禁`）：

- 门禁条目中明确写出：**问卷尚未回收**、**报价尚未确认**、**客户尚未签约**、**承接流程未就绪**，并引导至客户详情完成。
- **开始办案** 在条件未满足时为禁用态属合理；可走查是否在 **过晚步骤** 才集中暴露门禁（前置若可在更早步骤拦截，可减少填完再返回的成本）。

**期望**：客户详情补完后 **深链或草稿恢复** 回到原新建案件向导；各子流程入口在信息架构上可发现；承接流程「可建案」状态与客户详情一致。

## 仅见于截图、正文未逐条写明的补充点（便于测试用例）

下列内容 Word **正文未写死字句**，但从截图可归纳，**二次分析前易遗漏**：

1. **日期控件**：可输入明显非法生日（如极长年份），需长度与范围校验。
2. **分派与期限**：左侧竖排文字与金额、复选框、「模板自动创建契约」等 **严重重叠**，应作为 **P0 布局缺陷** 单列回归。
3. **签约弹窗 / 门禁**：经营管理建案的 **四条前置**（问卷、报价、签约、承接就绪）与正文第 4、8 条同源，测试时应逐条勾销验证返回路径。
4. **向导步骤与门禁**：第 3 步已提示前置未满足时，按钮态与文案是否一致；第 4 步门禁与禁用「开始办案」的联动。
5. **阻塞状态下的主按钮**：图 2「确认创建案件」与图 6「开始办案」等 **应在「不可建案」时统一为禁用或与黄条提示一致**，避免视觉可点与后端/门禁不一致。
6. **跟进表单字段形态**：结论 / 下一步为**单行输入**，若实务需长文本可考虑与「跟进摘要」区隔或改为多行（轻量 UX，优先级由产品定）。
