# 门禁系统说明（Vue 3版）

## 核心理念

**规范要落地，必须由脚本强制执行。**

---

## 推荐门禁

### 强制项

1. `pnpm typecheck`
2. `pnpm lint`
3. `pnpm test`
4. `pnpm build`

### 建议项

5. 检查大组件文件
6. 检查硬编码文案 / 魔法数字
7. 检查未使用导出和死代码

---

## package.json 建议脚本

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vue-tsc -b && vite build",
    "typecheck": "vue-tsc --noEmit",
    "lint": "eslint .",
    "format": "prettier --write .",
    "test": "vitest run"
  }
}
```

---

## pre-commit / CI 推荐顺序

```bash
pnpm typecheck
pnpm lint
pnpm test
pnpm build
```

---

## 建议拦截的问题

- TypeScript 类型错误
- ESLint 错误
- 测试失败
- 构建失败

建议告警但不强拦：

- 单组件超过 300 行
- 新增硬编码颜色
- 大量重复样式

---

## Git Hook 示例

```bash
#!/usr/bin/env sh

pnpm typecheck || exit 1
pnpm lint || exit 1
pnpm test || exit 1
pnpm build || exit 1
```

---

## 常见问题

### 门禁太慢

解决方式：

- 本地 pre-commit 先跑核心检查
- 详细检查放 CI
- 测试按影响范围拆分

### 门禁误报

不要直接关闭规则，先确认是否规则过严或代码结构需要调整。

---

## 原则

1. 能自动检查的，不靠口头约定。
2. 能在提交前发现的，不留到联调后。
3. 能在 CI 拦截的，不留到线上。

---

**记住：门禁不是增加负担，而是提前止损。**
