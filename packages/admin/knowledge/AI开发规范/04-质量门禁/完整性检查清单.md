# 完整性检查清单（Vue 3版）

## 交付总目标

```text
需求明确 + 测试可执行 + 类型安全 + 自动化门禁 + 可回滚
= 可稳定交付
```

---

## 开发前

- [ ] 需求已拆成验收标准
- [ ] 设计稿与 Token 已确认
- [ ] API 契约或 Mock 数据已准备
- [ ] 原子任务已拆分

---

## 开发中

- [ ] 核心逻辑先有测试
- [ ] 组件覆盖 loading / empty / error / success
- [ ] Props / Emits / 返回类型明确
- [ ] service / store / view 分层清晰
- [ ] 无明显魔法数字与硬编码文案

---

## 提交前

- [ ] `pnpm typecheck` 通过
- [ ] `pnpm lint` 通过
- [ ] `pnpm test` 通过
- [ ] `pnpm build` 通过
- [ ] 手动验证核心流程
- [ ] 关键截图或录屏已准备

---

## 合并前

- [ ] Self Review 完成
- [ ] Code Review 通过
- [ ] CI 通过
- [ ] 至少 1 个 Reviewer 认可

---

## 发布后

- [ ] 版本号或 Tag 已记录
- [ ] 测试环境已验证
- [ ] 监控无异常升高

---

## 质量红线

1. 类型检查失败，不允许合并。
2. 核心测试失败，不允许合并。
3. 关键页面状态缺失，不允许合并。
4. 无法回滚的发布，不允许上线。

---

**记住：完整性交付不是“功能写完”，而是“风险闭环”。**
