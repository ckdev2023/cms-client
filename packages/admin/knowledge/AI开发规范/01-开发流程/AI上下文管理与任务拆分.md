# AI上下文管理与任务拆分（Vue 3版）

## 核心问题

AI 质量下降，通常不是模型变差，而是任务过大、上下文过长、验证点过少。

---

## 四层任务拆分

### Layer 1：原子任务

特点：

- 30 分钟内完成
- 单一目标
- 只改 1-3 个相关文件

示例：

- 实现 `PostListEmpty.vue`
- 为 `usePostList` 增加重试逻辑
- 给 `postStore` 补一组失败测试

### Layer 2：Session 任务

由 3-5 个原子任务组成。

示例：

```text
Session：完成帖子列表空态与错误态
1. 补测试
2. 实现空态
3. 实现错误态
4. 接入重试按钮
5. 跑测试与修正
```

### Layer 3：模块任务

示例：

- 帖子列表模块
- 帖子详情模块
- 发布帖子模块

### Layer 4：项目任务

多个模块串成一个迭代目标。

---

## 窗口级工作规则

### 好的任务描述

- 只实现 `PostFilterPanel` 的重置与确认行为
- 只补 `usePostEditor` 的保存失败测试

### 不好的任务描述

- 把帖子模块都做完
- 把这个页面整体重构一下

---

## Vue 3 项目推荐拆法

### 组件任务

- 只做模板结构
- 只做交互行为
- 只做样式还原

### 状态任务

- 只做一个 composable
- 只做一个 Pinia store action
- 只做一个接口映射函数

### 测试任务

- 只补某个组件的 4 个核心状态
- 只补某个 store 的成功/失败分支

---

## 检查点

每完成一个原子任务，至少做一项验证：

- `pnpm vitest run <file>`
- `pnpm lint`
- `pnpm typecheck`
- 手动点一遍关键交互

---

## 提示词模板

```text
任务：只实现 PostList.vue 的 empty / error 两个状态

边界：
1. 不修改 API 层
2. 不新增全局 store
3. 不处理 success 列表项样式

输出：
- 更新 PostList.vue
- 更新对应组件测试

验证：
- pnpm vitest run src/modules/post/__tests__/PostList.spec.ts
```

---

## 实用原则

1. 一次只解决一个用户可感知问题。
2. 一次只让 AI 持有一小段上下文。
3. 每个小任务结束必须有验证结果。
4. 验证通过再进入下一个任务。

---

**记住：小任务、短上下文、强验证，AI 才能稳定输出。**
