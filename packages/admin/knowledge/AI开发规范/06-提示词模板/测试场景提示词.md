# 测试场景提示词（Vue 3版）

## 模板1：补组件测试

```text
任务：为【PostList.vue】补组件测试

要求：
1. 使用 Vitest + Vue Test Utils
2. 覆盖 loading / empty / error / success 四种状态
3. 测试描述使用用户行为语言
4. 不测试内部私有实现细节
5. 输出可直接运行的 spec 文件
```

---

## 模板2：补 store / composable 测试

```text
任务：为【usePostList】补单元测试

要求：
1. 覆盖加载成功、加载失败、重试三条路径
2. 使用 mock service，不依赖真实接口
3. 断言状态变化，而不是只断言函数被调用
4. 不修改生产逻辑以迎合测试
```

---

## 模板3：补回归测试

```text
问题：修复了一个【重复提交】bug

任务：
1. 先写一条失败的回归测试复现 bug
2. 再修复实现
3. 最后确认测试变绿

要求：
- 保留原有行为
- 只修复当前 bug
- 不顺手重构无关代码
```

---

**使用方法：先给组件或 composable 路径，再给验收标准或 bug 描述。**
