# 开发场景提示词（Vue 3版）

## 模板1：实现新页面

```text
任务：实现【帖子列表页】

技术栈：
- Vue 3
- TypeScript
- Pinia
- Vue Router
- Vitest
- Vue Test Utils

要求：
1. 先补测试，覆盖 loading / empty / error / success
2. 页面只负责展示与交互
3. 业务逻辑提取到 composable 或 store
4. 接口请求放 service 层
5. 不要过度设计

验证：
- pnpm typecheck
- pnpm lint
- pnpm test
- pnpm build
```

---

## 模板2：实现表单

```text
任务：实现【用户资料编辑表单】

要求：
1. 使用 Vue 3 script setup + TypeScript
2. 有校验、提交中、失败提示、成功提示
3. 禁止重复提交
4. 为提交成功和失败补测试
5. 不直接在组件里写裸请求
```

---

## 模板3：实现状态管理

```text
任务：实现【筛选面板】状态管理

要求：
1. 先判断应该使用 composable 还是 Pinia
2. 只实现当前模块所需状态
3. 提供 reset / apply 等明确动作
4. 补单元测试
5. 避免把纯 UI 临时状态放入全局 store
```

---

**使用方法：把模块名、文件路径、验收标准补进去即可直接发给 AI。**
