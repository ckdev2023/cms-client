# 重构场景提示词（Vue 3版）

## 模板1：组件拆分

```text
任务：重构【PostListPage.vue】

目标：
1. 拆分过大的页面组件
2. 保持功能不变
3. 优先提取纯展示组件和 composable
4. 不引入新的业务能力

约束：
- 保留现有 props / emits 行为
- 保持测试通过
- 不修改接口契约

验证：
- pnpm test
- pnpm typecheck
- pnpm lint
```

---

## 模板2：状态下沉/上提

```text
任务：整理【筛选面板】状态

目标：
1. 区分局部状态与全局状态
2. 只把跨页面共享状态保留在 Pinia
3. 把页面私有逻辑下沉到 composable
4. 删除无用状态和重复派生状态
```

---

## 模板3：Service 层整理

```text
任务：重构【postService.ts】

目标：
1. 统一请求方法和错误处理
2. 把接口字段映射集中到 service 层
3. 删除组件中的重复请求逻辑
4. 保持对外返回类型稳定

要求：
- 不改接口语义
- 为关键分支补测试
```

---

**使用方法：重构任务一定要写清楚“不改变什么”，否则 AI 很容易越界。**
