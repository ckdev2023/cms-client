# 测试分层策略（Vue 3版）

## 目标

用分层测试覆盖关键风险，而不是盲目堆测试数量。

---

## 推荐比例

```text
单元测试 / store / composable：50%
组件测试：30%
集成测试：10%
E2E 测试：10%
```

---

## 分层说明

### 1. 单元测试

测试对象：

- 工具函数
- 数据转换函数
- composable
- Pinia store action

特点：

- 快
- 稳定
- 定位问题清晰

### 2. 组件测试

测试对象：

- 页面状态渲染
- 交互按钮
- 表单输入
- 事件触发

特点：

- 最贴近前端日常风险
- 性价比最高

### 3. 集成测试

测试对象：

- 路由 + store + service 的协同
- 模块级流程

### 4. E2E 测试

测试对象：

- 登录
- 下单
- 提交表单
- 核心审批流

---

## Vue 3 推荐工具

- `Vitest`
- `@vue/test-utils`
- `msw`
- `Playwright`

---

## 必测状态

每个核心页面至少覆盖：

- [ ] loading
- [ ] empty
- [ ] error
- [ ] success

每个核心表单至少覆盖：

- [ ] 提交成功
- [ ] 提交失败
- [ ] 前端校验失败
- [ ] 禁止重复提交

---

## 目录建议

```text
src/modules/post/
  __tests__/
    PostList.spec.ts
    usePostList.spec.ts

tests/
  e2e/
    post-list.spec.ts
```

---

## 原则

1. 单元测试验证逻辑。
2. 组件测试验证行为与呈现。
3. E2E 只覆盖关键主路径。
4. 一个 bug 至少补一条回归测试。

---

**记住：测试金字塔的重点不是“多”，而是“稳”和“准”。**
