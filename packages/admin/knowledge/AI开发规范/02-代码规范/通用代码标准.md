# 通用代码标准（Vue 3 / TypeScript 侧重）

## 命名规范

### 组件名

使用 `PascalCase`：

```text
PostList.vue
UserProfileCard.vue
```

### 变量 / 方法名

使用 `camelCase`：

```ts
const isLoading = ref(false);
function fetchPostList() {}
```

### 常量名

使用 `UPPER_SNAKE_CASE`：

```ts
const REQUEST_TIMEOUT = 10000;
```

### Composable

必须以 `use` 开头：

```ts
usePostList;
useAuthGuard;
```

---

## 代码组织

1. 组件负责展示与交互绑定。
2. 组合逻辑放 `composables`。
3. 跨页面状态放 `Pinia`。
4. 数据请求放 `services`。
5. 类型定义放 `types`。

---

## TypeScript 要求

- 公共函数必须有明确返回类型
- 不滥用 `any`
- 优先 `type` / `interface` 描述接口结构
- 对外暴露的数据结构必须稳定

示例：

```ts
export interface PostItem {
  id: string;
  title: string;
}

export async function fetchPosts(): Promise<PostItem[]> {
  return request.get("/posts");
}
```

---

## 错误处理

不要吞异常。

```ts
export async function savePost(payload: SavePostPayload): Promise<void> {
  try {
    await request.post("/posts", payload);
  } catch (error) {
    console.error("savePost failed", error);
    throw error;
  }
}
```

要求：

- 用户可见错误要有可读文案
- 日志和用户提示分离
- service 层负责归一化错误

---

## Vue 组件规范

- 使用 `script setup`
- `defineProps` / `defineEmits` 必须显式声明
- 避免在模板里写复杂表达式
- 超过 300 行的组件优先拆分

---

## 响应式规范

- 原始值用 `ref`
- 结构对象用 `reactive` 时注意解构丢失响应性
- 导出的派生状态优先 `computed`
- 副作用逻辑集中处理，不散落在模板事件里

---

## 样式规范

- 优先复用设计 Token
- 禁止随手写魔法数字
- 类名或样式语义化
- 样式方案保持项目统一，不混用多套规则

---

## 测试规范

- 核心业务必须可测
- bug 修复优先补回归测试
- 测试描述写行为，不写实现
