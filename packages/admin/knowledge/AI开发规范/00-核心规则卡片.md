# 核心规则卡片（Vue 3 / AI必读）

**精简版，每次开发前先看这一张。**

---

## 开发流程

```text
1. 先看标准：Vue / Pinia / Vue Router / Vitest 官方文档
2. 再看项目：已验证的组件、composable、service
3. 复制改：基于标准实现，不盲抄历史代码
```

优先级：

```text
官方标准 > 项目内已验证模式 > 现有代码
```

---

## 代码标准

### 命名

```ts
const userName = "Tom"; // camelCase
const REQUEST_TIMEOUT = 5000; // 常量
function fetchPostList() {} // 动词开头
function usePostList() {} // composable 必须 use 开头
```

```text
PostList.vue                 // 组件 PascalCase
postService.ts               // 模块文件 camelCase
```

### 必需要素

```vue
<script setup lang="ts">
interface Props {
  title: string;
  loading?: boolean;
}

const props = withDefaults(defineProps<Props>(), {
  loading: false,
});

const emit = defineEmits<{
  submit: [];
}>();
</script>
```

要求：

- `script setup + TypeScript`
- `Props / Emits` 显式声明
- 公共函数返回类型明确
- 不滥用 `any`

---

## 分层规则

```text
组件：展示 + 交互绑定
composable：页面组合逻辑
Pinia：跨页面共享状态
service：请求与数据映射
```

禁止：

- 组件里直接写复杂请求逻辑
- 把局部状态滥放到全局 store
- 在模板里堆复杂业务判断

---

## 错误处理

```ts
const loading = ref(false);

async function submit() {
  if (loading.value) return;

  loading.value = true;
  try {
    await savePost();
  } catch (error) {
    console.error("savePost failed", error);
    throw error;
  } finally {
    loading.value = false;
  }
}
```

必须做到：

- 请求失败有兜底
- 提交操作防重复
- loading / empty / error / success 四态完整

---

## 样式规则

优先：

- 复用设计 Token
- 复用现有基础组件
- 统一间距、颜色、字号、圆角

禁止：

- 随手写魔法数字
- 直接硬编码一堆颜色
- 为一点差异复制整份组件

---

## 禁止事项

```text
❌ any 到处传
❌ 组件 300+ 行还不拆
❌ 只做 happy path
❌ 直接改测试迎合实现
❌ 页面、状态、请求三层混在一起
```
