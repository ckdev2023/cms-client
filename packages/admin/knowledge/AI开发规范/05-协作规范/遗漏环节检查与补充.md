# 遗漏环节检查与补充（Vue 3版）

## 已覆盖环节

- 需求拆分
- TDD 与测试分层
- 代码规范
- 类型检查
- Lint / Build 门禁
- Code Review

---

## 常见遗漏项

### 1. UI 还原

问题：

- 功能对了，但视觉不对

补充：

- 设计 Token
- 截图对比
- 设计稿走查

### 2. 错误监控

问题：

- 线上报错无法追踪

补充：

- Sentry
- 请求错误日志
- 关键操作埋点

### 3. 性能监控

问题：

- 页面能用，但首屏慢、切换卡

补充：

- 路由级懒加载
- 大列表虚拟化
- 首屏资源分析

### 4. 依赖管理

问题：

- 升级依赖后静默破坏

补充：

- 锁定版本
- 升级前跑完整测试
- 每月安全扫描

### 5. 文档维护

问题：

- 代码变了，文档没变

补充：

- PR 检查文档是否需要更新
- 关键模块保留变更记录

---

## 优先级

### P0

- UI 还原
- 错误监控
- 回滚方案

### P1

- 性能监控
- 依赖治理
- 文档维护

### P2

- 无障碍
- 国际化
- 埋点体系

---

**记住：真正拖慢团队的，通常不是开发本身，而是这些被漏掉的收尾环节。**
