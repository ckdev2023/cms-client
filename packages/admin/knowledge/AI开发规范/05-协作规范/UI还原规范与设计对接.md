# UI还原规范与设计对接（Vue 3版）

## 核心目标

**功能正确 + 视觉一致，才算真正完成。**

---

## 开发前准备

需要拿到：

1. Figma 链接
2. 设计稿版本
3. 设计 Token
4. 响应式断点规则
5. 交互说明和动效说明

---

## 代码侧要求

### 1. 统一 Token

至少统一以下内容：

- 颜色
- 间距
- 圆角
- 阴影
- 字体

### 2. 不直接写魔法数字

优先：

```ts
const spacingMd = "16px";
```

不要：

```ts
padding: 13px 17px;
```

### 3. 状态齐全

UI 必须确认：

- 默认态
- hover / active / disabled
- loading / empty / error
- 暗色模式（如果项目支持）

---

## 对接流程

### Step 1：读设计稿

- 看结构层级
- 看栅格与间距
- 看状态变体

### Step 2：映射为组件

- 哪些是已有组件
- 哪些需要新建组件
- 哪些是变体而不是新组件

### Step 3：落地样式

- 优先用已有设计系统
- 没有再补 Token 或组件能力

### Step 4：核对截图

- 自测截图
- 和设计稿并排比对
- 记录有意偏差

---

## 检查清单

- [ ] 间距一致
- [ ] 字号和字重一致
- [ ] 圆角一致
- [ ] 颜色一致
- [ ] 响应式布局合理
- [ ] 状态完整

---

## AI 提示词增强

```text
任务：实现用户卡片组件

要求：
1. 严格参考 Figma
2. 优先复用现有 Button / Tag / Avatar 组件
3. 不要硬编码颜色和间距
4. 输出前自查 hover / disabled / loading 三种状态
5. 如与设计稿有偏差，明确写出原因
```
