# 版本管理与快速回滚（Vue 3版）

## 提交信息规范

推荐格式：

```text
<type>(<scope>): <subject>
```

示例：

```text
feat(post): add post list page
fix(auth): handle token refresh failure
test(user): cover profile form submit flow
refactor(filter): extract useFilterPanel
docs(vue3): add testing guideline
```

---

## 分支建议

- `main`：稳定分支
- `develop`：集成分支
- `feature/*`：功能分支
- `hotfix/*`：紧急修复

---

## Tag 建议

```text
v0.1.0
v0.2.0
v1.0.0
```

适用场景：

- 模块完成
- 测试环境验收通过
- 正式发布

---

## 回滚方案

### 方案 1：`git revert`

适合：

- 已推远程
- 需要保留历史

### 方案 2：回滚到已知稳定 Tag

适合：

- 紧急线上问题
- 需要快速恢复稳定版本

### 方案 3：临时热修分支

适合：

- 生产问题需要边回滚边修复

---

## 回滚前检查

- [ ] 确认问题提交范围
- [ ] 确认目标稳定版本
- [ ] 通知相关成员
- [ ] 保留复现场景

---

## 回滚后检查

- [ ] 核心路径恢复正常
- [ ] 监控恢复稳定
- [ ] 问题 issue 已登记
- [ ] 后续修复方案已明确

---

## 原则

1. 小步提交，方便定位问题。
2. 关键版本打 Tag。
3. 主分支只接收可回滚变更。

---

**记住：能快速回滚的系统，才是真正可发布的系统。**
