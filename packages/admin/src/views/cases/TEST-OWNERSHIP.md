# Cases Module — Test Ownership Matrix

> Frozen by `p0-fe-002a-05-freeze-test-ownership`.
> Every test file has exactly one owner layer.
> Adding tests that cross boundaries requires updating this matrix first.

## Ownership Rules

1. **Each layer tests only its own public API** — never reach into another layer's internals.
2. **No duplicate coverage** — if an adapter function is tested in `CaseAdapterMappers.test.ts`, the repository test must NOT re-test the mapping logic; it only verifies the wiring (correct adapter is called with the response body).
3. **Composable tests inject stubs/mocks** — they do not exercise real HTTP or adapter logic.
4. **Fixture tests verify data shape** — they do not exercise repository behaviour.

## Known dual-function note

`matchesCaseFilters` exists in two modules with different signatures:

- `repository.ts` → `(item, params: CaseListQueryParams)` — used by mock repository, tested in `repository.test.ts`
- `model/useCaseListModel.ts` → `(item, filters: CaseListFiltersState, customerId?)` — used by list composable, tested in `useCaseListModel.test.ts`

Both are legitimate test owners of their respective function.
When the mock repository is retired (after real API integration), the `repository.ts` version and its tests will be removed; the composable version survives.
