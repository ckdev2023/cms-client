# Cases Module — Test Ownership Matrix

> Frozen by `p0-fe-002a-05-freeze-test-ownership`.
> Every test file has exactly one owner layer.
> Adding tests that cross boundaries requires updating this matrix first.

## Ownership Rules

1. **Each layer tests only its own public API** — never reach into another layer's internals.
2. **No duplicate coverage** — if an adapter function is tested in `CaseAdapterMappers.test.ts`, the repository test must NOT re-test the mapping logic; it only verifies the wiring (correct adapter is called with the response body).
3. **Composable tests inject stubs/mocks** — they do not exercise real HTTP or adapter logic.
4. **Fixture tests verify data shape** — they do not exercise repository behaviour.

## Support Module Seam Boundary (frozen by p0-fe-002e-03)

The `CaseAdapterSupportSeams` module defines adapter stub functions for detail tab support modules.
These seams are **intentionally unimplemented** (return `null`) at this stage; real adapters will be
filled by their respective tasks:

| Seam function                  | Module         | Planned task                 |
| ------------------------------ | -------------- | ---------------------------- |
| `adaptCaseDocumentGroups`      | documents      | `p0-fe-006b`                 |
| `adaptCaseFormsData`           | forms          | `p0-fe-006b`                 |
| `adaptCaseValidationData`      | validation     | `p0-fe-006b`                 |
| `adaptCaseBillingData`         | billing        | `p0-fe-006b`                 |
| `adaptCaseSubmissionPackages`  | submissionPkgs | `p0-fe-006b`                 |
| `adaptCaseDoubleReviewEntries` | doubleReview   | `p0-fe-006b`                 |
| `adaptCaseTaskList`            | tasks          | ~~`p0-fe-006d`~~ implemented |
| `adaptCaseDeadlineList`        | deadlines      | `p0-fe-006d`                 |

Boundary rules:

1. **Do not implement** any seam adapter before its planned task — the contract test (`boundary contract (p0-fe-002e-03)`) will fail if a seam returns non-null.
2. **Adding a new seam** requires updating `SUPPORT_SEAM_REGISTRY` in `CaseAdapterSupportSeams.ts` and the `SEAM_FUNCTIONS` lookup in `CaseAdapterSupportSeams.test.ts`.
3. **Filling an existing seam** requires removing or updating the boundary test assertion for that function.
4. The `messages` / `log` adapter is **not** a seam — it lives in `CaseCommsLogsAdapter.ts` with a real implementation (p0-fe-002e-01).
5. `adaptCaseTaskList` was implemented in `p0-fe-006d-01` and removed from the seam registry.

## Known dual-function note

`matchesCaseFilters` exists in two modules with different signatures:

- `repository.ts` → `(item, params: CaseListQueryParams)` — used by mock repository, tested in `repository.test.ts`
- `model/useCaseListModel.ts` → `(item, filters: CaseListFiltersState, customerId?)` — used by list composable, tested in `useCaseListModel.test.ts`

When the mock repository is retired (after real API integration), the `repository.ts` version and its tests will be removed; the composable version survives.
