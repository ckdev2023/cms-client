# 客户页 P0 拆分约束清单

> 权威来源：[docs/gyoseishoshi_saas_md/P0/06-页面规格/客户.md](../../../../docs/gyoseishoshi_saas_md/P0/06-页面规格/客户.md)
>
> 用途：原型拆分时的唯一验收基线。任何改动后必须覆盖以下清单全量回归。

---

## 1 列表页必保留字段

| # | 列头 | 规格字段 | 原型中的呈现 |
|---|------|---------|-------------|
| 1 | 客户 | 客户名称（识别名/法定名） | 识别名 + 客户编号/电话/邮箱辅助行 |
| 2 | フリガナ | フリガナ | 文本列（md+ 可见） |
| 3 | 累计案件 | 累计案件数 | 数字链接 / 0 灰色 |
| 4 | 活跃案件 | 当前活跃案件数 | 数字链接 / 0 灰色 |
| 5 | 最近联系 | 最近联系时间 | 日期 + 渠道辅助行 |
| 6 | 负责人 | 当前负责人 | 头像缩写 + 姓名 |
| 7 | 介绍人/来源 | 介绍人/来源 | 文本（lg+ 可见） |
| 8 | 所属分组 | 所属分组 | chip（lg+ 可见） |

拆分时 **不可删减上述 8 列**。

## 2 列表页搜索与筛选

### 2.1 搜索

- 范围：客户名 / フリガナ / 电话 / 邮箱
- 控件：`search-input`，位于筛选栏右侧

### 2.2 筛选器

| # | 筛选维度 | 控件类型 | 默认值 |
|---|---------|---------|-------|
| 1 | 所属分组 | select | 全部 |
| 2 | 负责人 | select | 全部 |
| 3 | 活跃案件 | select（有/无/全部） | 全部 |

### 2.3 数据范围切换

- 三段式 segmented control：`我的` / `本组` / `全所（管理员）`
- 映射权限可见性（§6 角色 × Group × 负责人/协作者）

### 2.4 重置

- 筛选栏末尾「重置」按钮，清空所有筛选条件

## 3 列表页批量动作

| # | 动作 | 控件 | 约束 |
|---|------|------|------|
| 1 | 批量指派负责人 | select + 应用按钮 | 需至少选中 1 条客户，select 选定后才可应用 |
| 2 | 批量调整所属分组 | select + 应用按钮 | 需至少选中 1 条客户，变更 **须留痕**（不回写历史案件） |

### 3.1 批量选择机制

- 表头全选 checkbox（支持 indeterminate 态）
- 每行独立 checkbox（草稿行禁用）
- 选中计数显示 + 清除按钮
- 批量操作栏仅在 selectedCount > 0 时可见

## 4 新建客户弹窗（列表页入口）

### 4.1 表单字段

| # | 字段 | 控件 | 必填 | 备注 |
|---|------|------|------|------|
| 1 | 识别名（对内显示） | text input | 否 | |
| 2 | 所属 Group | select | **是** | 选项来源于组织 Group |
| 3 | 姓名（法定） | text input | **是** | |
| 4 | 假名（片假名） | text input | 否 | |
| 5 | 性別 | select | 否 | 不限 / 男 / 女 |
| 6 | 生年月日 | date input | 否 | |
| 7 | 国籍 | text input | 否 | 作为个人客户基础主数据保留 |
| 8 | 电话 | tel input | **条件必填** | 电话/邮箱至少填一项 |
| 9 | 邮箱 | email input | **条件必填** | 电话/邮箱至少填一项 |
| 10 | 来源/介绍人 | text input | 否 | |
| 11 | 头像 | file input | 否 | |
| 12 | 备注 | text input | 否 | |

### 4.2 去重提示

- 触发条件：电话或邮箱任一有值时显示去重提示区域
- 展示内容：可能重复客户卡片（姓名 + 编号 + 联系方式 + 分组）
- 规格依据：[03 §2.6 去重规则](../../../../docs/gyoseishoshi_saas_md/P0/03-业务规则与不变量.md#26-去重规则)

### 4.3 按钮可用态

- 「创建客户」启用条件：姓名（法定） + 所属 Group + (电话 ∨ 邮箱)
- 「保存草稿」始终可用
- 「取消」关闭弹窗

### 4.4 `#new` 入口

- URL hash 为 `#new` 时自动打开新建弹窗

## 5 草稿能力

| 行为 | 描述 |
|------|------|
| 保存 | 序列化表单字段至 `localStorage`（key: `gyosei_os_customer_drafts_v1`） |
| 恢复 | 草稿行显示在列表顶部，点击「继续」载入弹窗 |
| 删除 | 成功创建客户时自动移除对应草稿 |
| 列表呈现 | 草稿行 checkbox 禁用，不参与批量操作 |

> **原型演示说明**：草稿 localStorage 仅为原型交互演示，不代表生产存储方案。

## 6 状态与异常态

| # | 状态 | 规格要求 | 原型现状 |
|---|------|---------|---------|
| 1 | 空状态（无客户） | 提示"录入第一个客户"或"从线索转化" | 未实现（现有 3 条示例数据） |
| 2 | 重复客户提示 | 新建时按电话/邮箱弹出去重提示 | 已实现（`#dedupeHint`） |
| 3 | 无活跃案件 | 列表可筛选，案件数显示 0 | 已实现（`Li Wei` 行展示 0） |

拆分后必须保留 **状态 2** 和 **状态 3** 的交互，并为 **状态 1** 预留空状态占位。

## 7 Toast 通知

| 触发场景 | title | desc |
|----------|-------|------|
| 创建客户 | 客户已创建（示例） | 已生成客户档案，可继续建案 |
| 保存草稿 | 草稿已保存 | 可在客户列表中点击"继续"完成建档 |
| 载入草稿 | 已载入草稿 | 继续完善后即可完成建档 |
| 批量指派 | 批量指派（示例） | 已选择 N 条，负责人：xxx |
| 批量调分组 | 批量调整分组（示例） | 已选择 N 条，分组：xxx（需留痕） |

> **原型演示说明**：toast 中标注「示例」的为原型静态反馈，不代表真实后端操作。

## 8 权限与可见性（规格约束，原型仅做 segmented control 切换）

| 角色 | 可见范围 | 可编辑范围 | 可导出 |
|------|---------|-----------|--------|
| 管理员 | 全所客户 | 全部字段 | 需留痕 |
| 主办人 | 本组 + 负责的客户 | 负责客户全部字段 | 需留痕 |
| 助理 | 本组客户 | 基础信息、沟通记录 | 受限 |
| 销售/前台 | 本组客户（基础信息） | 基础信息 | 不可 |

## 9 详情页（规格定义，原型尚未实现）

以下为规格要求的详情页 Tab 结构，原型拆分阶段暂不涉及，但结构设计需为其预留入口。

| Tab | 内容 | 关键入口 |
|-----|------|---------|
| 基础信息 | 客户编号、识别名/姓名、フリガナ、性别、生日、联系方式、写真/头像、介绍人/来源、归属 Group、备注 | 编辑客户 |
| 关联案件 | 案件列表（编号、类型、阶段、负责人、更新时间） | 一键建案、批量建案（家族签） |
| 关联人 | CustomerRelation 列表（关系类型、关联客户、备注） | 新增关联人、为关联人批量建案 |
| 沟通记录 | 时间线（渠道、摘要、下一步动作）；内部 / 客户可见备注分离 | — |
| 操作日志 | 信息变更日志、关系变更日志 | — |

## 10 P0 明确不做

| # | 不做项 | 后置到 | 原因 |
|---|--------|-------|------|
| 1 | 企业客户主数据 | P1 | P0 以个人客户为主；雇主信息沉淀在案件字段 |
| 2 | 客户门户关联 | P1 | P0 无客户门户 |
| 3 | 客户合并 | P1 | P0 仅做去重提示，不做物理合并 |
| 4 | 客户级跨案件附件汇总 | P1 | P0 附件通过案件资料清单管理 |
| 5 | 证件台账（独立 Tab） | P1 | P0 证件通过案件资料清单管理 |
| 6 | 批量导出 | P1 | P0 不做批量客户数据导出 |
| 7 | 高级搜索与标签筛选 | P1 | P0 仅基础搜索 |
| 8 | 收费记录 Tab | — | 通过案件收费 Tab 和财务模块查看 |
| 9 | 资料附件 Tab | P1 | 跨案件客户级附件汇总后置 |

拆分过程中 **禁止** 引入上述能力的 UI 或逻辑占位。

|------|------|
