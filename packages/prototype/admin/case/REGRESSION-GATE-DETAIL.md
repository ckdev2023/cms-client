## 验收流程

### 每轮实现提交前

1. **自检**：开发者按 Gate 1–8 逐项确认自己的实现范围
2. **回归**：至少覆盖当前实现范围涉及的所有 Gate 项
3. **标记**：在本文档中标注已通过的项（`[x]`），未通过的项必须修复后再提交

### 最终验收（全部实现完成后）

1. 本文档全部 `[原型]` 项通过
2. `P0-CONTRACT-DETAIL.md` 拆分回归清单全部通过
3. `split-manifest-detail.json` 的 `regressionChecklist` 全部通过
4. `npm run fix` + `npm run guard` 通过

### Gate 失败处理

- 任何 Gate 中有 `[原型]` 项未通过：该轮实现 **不可合入**，修复后重新验收
- `[生产约束]` 项在原型阶段可标记为 `[deferred]`，但必须在 `MIGRATION-MAPPING-DETAIL.md` 中有对应的生产实现说明
- `[跨模块]` 项若受限于其他模块未就绪，标记为 `[blocked: 模块名]`，待对方就绪后补验
