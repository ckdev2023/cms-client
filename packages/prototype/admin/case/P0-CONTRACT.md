# 案件新建页 P0 拆分约束清单

> 权威来源：[docs/gyoseishoshi_saas_md/P0/06-页面规格/案件.md](../../../../docs/gyoseishoshi_saas_md/P0/06-页面规格/案件.md)
>
> 用途：作为 `packages/prototype/admin/case/create.html` 的唯一验收基线。

---

## 1 必保留区块

新增案件原型必须保留以下 7 个区块：

| # | 区块 | 说明 |
|---|------|------|
| 1 | 页面头部 | 面包屑、标题、P0 范围 chips |
| 2 | Stepper | 业务信息 / 关联人与资料 / 分派与期限 / 完成创建 |
| 3 | 模板选择 | 至少支持 `家族滞在`、`技人国` 两种模板 |
| 4 | 主申请人与关联人 | 主申请人选择、快速新建、追加关联人 |
| 5 | 资料清单模板 | 按模板展示资料项分组与必交口径 |
| 6 | 分派与初始设置 | 负责人、Group、预计提交期限、合同金额、自动动作 |
| 7 | 创建反馈 | Step 4 汇总、成功 banner、toast |

## 2 建案基础字段

### 2.1 Step 1 必保留字段

| 字段 | 控件 | 必填 | 说明 |
|------|------|------|------|
| 模板 | 卡片选择 | 是 | `家族滞在` / `技人国` |
| 申请类型 | select | 是 | 认定 / 变更 / 更新 |
| 归属 Group | select | 是 | 东京一组 / 东京二组 / 大阪组 |
| 案件标题 | text input | 是 | 默认由主申请人 + 模板 + 申请类型生成，可手改 |

### 2.2 Step 2 必保留字段

| 字段 | 控件 | 必填 | 说明 |
|------|------|------|------|
| 主申请人 | select / modal | 是 | 已有客户选择或快速新建 |
| 关联人 | 列表 + modal | 否 | 家族签用于配偶/子女；工作类可补充雇主联系人 |
| 资料模板 | 只读列表 + checkbox | 是 | 模板驱动生成，需区分必交/可选 |

### 2.3 Step 3 必保留字段

| 字段 | 控件 | 必填 | 说明 |
|------|------|------|------|
| 负责人 | select | 是 | 创建后案件归属主办人 |
| 预计提交期限 | date input | 是 | 用于列表风险/到期展示 |
| 合同金额 | number input | 是 | P0 仅记录总额，不展开收费节点 |
| 自动生成资料清单 | checkbox | 否 | 默认勾选 |
| 同步生成初始任务 | checkbox | 否 | 默认勾选 |
| 跨组建案原因 | textarea | 条件必填 | Group 非默认时必须填写 |

## 3 模板与批量模式

### 3.1 家族滞在模板

- 显示家族签批量建案说明卡
- 显示角色 chips：配偶 / 子女 / 扶养者 / 保证人
- 显示至少 2 条默认办理对象预览
- URL hash 为 `#family-bulk` 时默认进入家族模板，并保留批量模式提示

### 3.2 技人国模板

- 隐藏家族签批量说明卡
- 显示雇主信息沉淀说明
- 资料清单切换为主申请人 / 雇主 / 事务所内部三段

## 4 Modal 快速新建客户

### 4.1 字段

| 字段 | 必填 |
|------|------|
| 姓名 | 是 |
| 角色 | 是 |
| 所属 Group | 是 |
| 电话 | 电话/邮箱二选一 |
| 邮箱 | 电话/邮箱二选一 |
| 备注 | 否 |

### 4.2 规则

- 姓名为空时不可保存
- Group 为空时不可保存
- 电话与邮箱都为空时不可保存
- `mode=primary` 时回填为主申请人
- `mode=related` 时追加到关联人列表

## 5 关键可用态

| 步骤 | 启用条件 |
|------|---------|
| 下一步（Step 1 → Step 2） | 模板 + 申请类型 + 案件标题完整 |
| 下一步（Step 2 → Step 3） | 已选择主申请人 |
| 下一步（Step 3 → Step 4） | 负责人 + 预计提交期限 + 合同金额完整；跨组时原因必填 |
| 创建案件 | Step 4 展示，沿用 Step 3 校验结果 |

## 6 状态与异常态

必须保留以下状态：

- 家族签批量模式提示
- 技人国雇主信息说明态
- 无关联人空状态
- 成功创建 banner
- modal 校验错误提示

## 7 Toast / 反馈

| 场景 | title | desc |
|------|-------|------|
| 快速新建客户成功 | 客户已加入（示例） | 已回填到建案流程，可继续完善关联人与资料模板 |
| 创建案件 | 案件已创建（示例） | 已生成 Case 与资料清单草稿，可继续分派任务与补录资料 |

## 8 Demo-only 行为

以下行为必须保留，但需明确为原型演示：

- 模板、客户、负责人均为硬编码示例数据
- 创建案件仅显示 banner / toast，不真实落库
- 资料清单 checkbox 仅演示模板结构，不回写进度
- `#family-bulk` 仅切换默认模式，不真实批量创建多个 Case

## 9 P0 明确不做

- 不实现案件详情页 10 个 Tab 的真实创建后跳转
- 不执行 Gate-A/B/C 校验
- 不生成真实提交包、补正包、收费节点
- 不引入企业客户实体
- 不做案件草稿持久化

## 1 列表页必保留字段与区块

### 1.1 顶部主区块

- 页面标题：`案件`
- 副标题：覆盖建档、资料收集、Gate 校验、提交包与归档前的统一列表视图
- 顶部入口：
  - `家族签批量建案`
  - `新建案件`

### 1.2 摘要卡片

拆分时必须保留以下 4 张 summary card：

| # | 卡片 | 原型文案 |
|---|------|---------|
| 1 | 进行中案件 | 当前过滤结果中的活跃案件 |
| 2 | 失败校验 | 存在硬性阻断或待修复案件 |
| 3 | 近 7 日截止 | 优先跟进补件 / 提交 / 到期事项 |
| 4 | 未收金额 | 过滤结果中的未收合计 |

### 1.3 列表列

| # | 列头 | 规格字段 | 原型中的呈现 |
|---|------|---------|-------------|
| 1 | 案件编号 / 名称 | 案件编号、案件名称 | 主文案 + 辅助编号 |
| 2 | 案件类型 | 案件类型 | 文本 |
| 3 | 主申请人 | 主申请人 | 文本 |
| 4 | Group | Group | chip |
| 5 | 当前阶段 | 案件阶段 | `status-pill` |
| 6 | 负责人 | 当前负责人 | 头像缩写 + 姓名 |
| 7 | 资料完成率 | 完成率 | 文案 + progress bar |
| 8 | 最新校验 | 最新校验状态与阻断数 | `passed/failed/pending` + 阻断数 |
| 9 | 未收金额 | 未收金额 | 右对齐金额 |
| 10 | 最近更新 | 最近更新时间 | 文本 |
| 11 | 截止日 | 截止日 | 日期 + 辅助行 |
| 12 | 风险状态 | 风险状态 | `status-pill` |
| 13 | 操作 | 详情入口 | 图标按钮 |

拆分时不可删减上述列，也不可引入规格外的新主列。

## 2 搜索、筛选与主要交互

### 2.1 搜索

- 搜索框 ID：`searchInput`
- placeholder：`搜索：案件编号 / 案件名称 / 主申请人 / Group`
- 原型搜索范围：
  - 案件编号
  - 案件名称
  - 案件类型
  - 主申请人
  - Group

### 2.2 数据范围切换

- 三段式 segmented control：
  - `我的`
  - `本组`
  - `全所（管理员）`

### 2.3 默认筛选

| # | 筛选维度 | 控件 ID | 默认值 |
|---|---------|---------|-------|
| 1 | 当前阶段 | `filterStage` | 全部 |
| 2 | 负责人 | `filterOwner` | 全部 |
| 3 | Group | `filterGroup` | 全部 |
| 4 | 风险状态 | `filterRisk` | 全部 |
| 5 | 校验状态 | `filterValidation` | 全部 |

### 2.4 重置

- `resetFiltersBtn` 清空所有筛选条件
- 重置后回到 `scope=mine`
- 搜索框与所有 select 恢复默认值

## 3 列表页批量动作

| # | 动作 | 控件 | 约束 |
|---|------|------|------|
| 1 | 批量指派负责人 | `bulkOwnerSelect` + `bulkOwnerApplyBtn` | 需至少选中 1 条 |
| 2 | 批量指派协作者 | `bulkCollaboratorSelect` + `bulkCollaboratorApplyBtn` | 需至少选中 1 条 |
| 3 | 批量调整截止日 | `bulkDueDateInput` + `bulkDueDateApplyBtn` | 需至少选中 1 条 |
| 4 | 批量生成任务 | `bulkTaskSelect` + `bulkTaskApplyBtn` | 需至少选中 1 条 |

### 3.1 批量选择机制

- 表头全选 checkbox：`selectAllCases`
- 每行独立 checkbox：`data-case-select`
- 已选择计数：`selectedCount`
- 清除按钮：`bulkClearBtn`
- 批量操作栏 `bulkActionBar` 仅在 `selectedCount > 0` 时可见
- 全选需支持 `indeterminate`

## 4 页面入口与单行操作

### 4.1 顶部入口

- `家族签批量建案` → `../case-create.html#family-bulk`
- `新建案件` → `../case-create.html`

### 4.2 单行动作

- 行尾详情按钮使用 `data-navigate="../case-detail.html"`
- 拆分后仍需保留“从列表进入详情”的可发现入口

## 5 本地状态与演示数据

当前案件原型没有草稿能力，但包含以下原型态本地状态：

| 能力 | 说明 |
|------|------|
| 静态案件列表 | `data/case-list.js` 中硬编码 6 条示例案件 |
| 静态负责人字典 | `owners` map 仅用于原型渲染 |
| 前端内存筛选状态 | 搜索、scope、筛选条件、selectedIds 全在页面内存中 |
| 前端内存批量修改 | 批量负责人、截止日直接改写前端示例数据 |
| 固定 today 基准 | 近 7 日截止统计使用固定日期 `2026-04-09` |

## 6 状态与异常态

| # | 状态 | 规格要求 | 原型现状 |
|---|------|---------|---------|
| 1 | 空筛选结果 | 提示无符合条件案件，并引导继续建案 | 已实现 |
| 2 | 校验失败 | 高亮 failed 与阻断数 | 已实现 |
| 3 | 近期待办 | 截止日与 summary 卡片体现 | 已实现 |
| 4 | 已归档只读语义 | S9 显示已归档状态 | 已实现为列表阶段态展示 |

拆分后必须保留状态 1、2、3 的用户可见反馈，并保留 S9 的展示语义。

## 8 权限与可见性

| 角色 | 可见范围 | 可编辑范围 | 可导出 |
|------|---------|-----------|--------|
| 管理员 | 全所案件 | 全部字段 | 需留痕 |
| 主办人 | 本组 + 负责/协作的案件 | 负责案件全部字段 | 需留痕 |
| 助理 | 本组 + 协作案件 | 资料、沟通、任务等操作字段 | 受限 |
| 财务 | 全所案件（仅收费相关字段） | 收费 Tab | 财务数据，需留痕 |

原型仅通过 `mine/group/all` 演示部分可见性切换，不代表完整权限实现。
