## 1 信息架构与导航

| 入口 | 位置 | 优先级 | 说明 |

#### 1.2.3 筛选条件保留规则（生产行为）

- **保留范围**：状态筛选值、所属案件筛选值、提供方筛选值、搜索关键词
- **保留时机**：从资料中心列表跳转到案件详情后再返回时
- **不保留**：页面刷新或重新从导航进入时恢复默认
- **实现策略**（生产参考）：URL query params 或 sessionStorage 存储筛选快照

> **原型演示说明**：原型中返回行为通过简单导航实现，不做筛选条件/滚动位置保持。仅演示跳转路径正确性。弹窗关闭后页面保持不跳转。

## 3 默认排序与展示规则

### 3.1 默认排序

- 待审核资料置顶
- 其次按状态优先级：退回补正 > 过期 > 缺件（pending） > 已提交待审核
- 再按截止日升序
- 无截止日的资料项置底

### 3.2 逾期/过期展示

- 截止日列显示"已过期"或"还有 X 天"，以风险色（`badge-red`）强调
- 过期行使用红色背景高亮 `bg-[rgba(220,38,38,0.04)]`
- 共享版本过期的资料项在"状态"列额外标注"共享版本过期"

### 3.3 `relative_path` 展示

- 仅保存相对路径，不保存绝对路径/盘符路径
- 表格字段支持一键复制（点击按钮 → 复制到剪贴板 → toast "已复制"）
- 未登记时显示"未登记"灰色占位

## 6 状态机（P0 口径）

### 6.1 资料项状态定义

| 状态 key | 标签 | Badge 类 | 含义 |
|----------|------|----------|------|
| `pending` | 待提交 | `badge-orange` | 等待提供方提供，尚无附件版本 |
| `uploaded_reviewing` | 已提交待审核 | `badge-blue` | 有附件版本等待审核 |
| `approved` | 通过 | `badge-green` | 审核通过，计入完成率 |
| `rejected` | 退回补正 | `badge-red` | 附件不符合要求，需重新提供 |
| `expired` | 过期 | `badge-red` | 当前有效版本已过有效期 |
| `waived` | 无需提供 | `badge-gray` | 标记为无需提供，从完成率分母剔除 |

### 6.2 允许的状态流转

| 当前状态 | 可流转至 | 触发条件 |
|----------|---------|---------|
| `pending` | `uploaded_reviewing` | 登记资料（有新附件版本） |
| `pending` | `waived` | 标记无需提供 |
| `uploaded_reviewing` | `approved` | 审核通过 |
| `uploaded_reviewing` | `rejected` | 退回补正 |
| `uploaded_reviewing` | `waived` | 标记无需提供 |
| `approved` | `expired` | 当前版本过有效期 / 共享版本过期联动 |
| `approved` | `uploaded_reviewing` | 新版本上传后重新进入审核 |
| `rejected` | `uploaded_reviewing` | 客户重新提供 |
| `rejected` | `waived` | 标记无需提供 |
| `expired` | `uploaded_reviewing` | 重新提供新版本 |
| `waived` | `pending` | 撤销 waived（须留痕） |

### 6.3 完成率口径

- **分子**：状态 ∈ {`approved`} 的资料项数
- **分母**：全部资料项数 − 状态 = `waived` 的资料项数
- **百分比**：分母 > 0 时 `Math.round(分子 / 分母 * 100)`；分母 = 0 时显示"无必需资料"

### 6.4 与 case-detail-config.js 状态枚举对齐

| documents-config key | case-detail status | 映射说明 |
|---------------------|-------------------|---------|
| `pending` | `pending` / `idle` | `idle` 视为 `pending` |
| `uploaded_reviewing` | `submitted` | 语义等同 |
| `approved` | `reviewed` / `done` | 统一为 `approved` |
| `rejected` | `rejected` | 一致 |
| `expired` | `expired` | 一致 |
| `waived` | `waived` | 一致 |

> 在 `documents-config.js` 中以 P0 规格 key 为权威（`pending/uploaded_reviewing/approved/rejected/expired/waived`），提供到 case-detail legacy key 的映射函数。

## 8 关键动作

| 动作 | 触发条件 | 结果 | 权限 | 入口 |
|------|---------|------|------|------|
| 审核通过 | 资料项状态=`uploaded_reviewing` | 状态→`approved`，计入完成率 | 主办人 | 案件详情行内操作 + 跨案件列表行内操作 |
| 退回补正 | 资料项状态=`uploaded_reviewing` | 状态→`rejected`，须填写原因 | 主办人 | 案件详情行内操作 + 跨案件列表行内操作 |
| 发送催办 | 资料项状态∈{`pending`, `rejected`} | 生成催办记录并留痕 | 主办人、助理 | 案件详情行内操作 + 跨案件列表行内操作 |
| 标记 waived | 资料项状态∈{`pending`, `uploaded_reviewing`, `rejected`} | 须记录原因码+文本，从完成率分母剔除 | 主办人 | 案件详情行内操作 |
| 新增资料项 | 案件需要额外材料 | 在案件资料清单新增一项，状态=`pending` | 主办人、助理 | 案件详情 Header 按钮 |
| 登记资料（本地归档） | 需要记录本地文件路径 | 生成新 DocumentFileVersion，资料项→`uploaded_reviewing` | 主办人、助理 | 案件详情 Header 按钮 + 跨案件列表 Header 按钮 |
| 引用既有版本 | 存在可复用的已审核版本 | 新增引用记录（不复制文件），资料项独立流转 | 主办人、助理 | 案件详情行内操作 |

### 8.1 登记资料（本地归档）弹窗

#### 8.1.1 表单字段

| # | 字段 | 控件 | 必填 | 备注 |
|---|------|------|------|------|
| 1 | 关联案件 | select / search | **是** | 跨案件入口需选择；案件内入口自动填充 |
| 2 | 关联资料项 | select | **是** | 仅展示非 waived 且非 approved 的资料项 |
| 3 | `relative_path` | text input | **是** | 相对路径 + 校验（见 §8.1.2） |
| 4 | 文件名/描述 | text input | 否 | 默认从 `relative_path` 尾部推导 |
| 5 | 版本号 | 自动生成 | — | 系统递增，不可编辑 |

#### 8.1.2 `relative_path` 录入规则

- 仅保存相对路径，不保存服务器绝对路径/本地盘符路径
- 建议格式：`{case_no}/{provider}/{doc_item_name}/{yyyymmdd}_{filename}`
- 示例：`A2026-001/main_applicant/passport/20260409_passport.pdf`
- 允许分隔符：`/`
- 禁止：`..`、前导 `~`、空白控制字符、前导 `/`
- 校验失败时：输入框标红 + 提示"请输入有效的相对路径（不含 .. 或绝对路径前缀）"

#### 8.1.3 按钮可用态

- 「登记」启用条件：关联案件 + 关联资料项 + `relative_path` 均已填写且校验通过
- 「取消」关闭弹窗

### 8.2 审核/退回弹窗

| 字段 | 审核通过 | 退回补正 |
|------|---------|---------|
| 审核结论 | 固定"通过" | 固定"退回补正" |
| 退回原因 | — | **必填** textarea |
| 确认按钮 | 「确认通过」 | 「确认退回」 |

### 8.3 标记 waived 弹窗

| # | 字段 | 控件 | 必填 |
|---|------|------|------|
| 1 | 原因码 | select（见 §8.3.1） | **是** |
| 2 | 原因备注 | textarea | 条件（原因码="其他"时必填） |

#### 8.3.1 waived 原因码枚举

| # | 原因码 | 说明 |
|---|-------|------|
| 1 | 无需提供（该签证类型免除） | |
| 2 | 保证人为配偶/直系亲属（免除） | |
| 3 | 客户已在其他案件提供等价材料 | |
| 4 | 入管局确认免除 | |
| 5 | 其他 | 须填备注 |

### 8.4 引用既有版本交互

- 触发入口：资料项行内「引用既有版本」按钮
- 选择器：展示可复用版本列表（`item_code` 一致 + 提供方兼容 + 未过期 + 审核状态=`approved`）
- 每个可选版本展示：来源案件名 + 资料项名 + 版本号 + 审核日期 + 有效期
- 选中确认后：新增引用记录，当前资料项状态独立维护

### 8.5 批量催办

- 触发入口：跨案件列表 Header「批量催办」按钮
- 前置条件：至少选中 1 条状态∈{`pending`, `rejected`} 的资料项
- 结果：对选中项逐一生成催办记录 → toast "已催办 N 项"

### 8.6 批量审核通过

- 触发入口：跨案件列表批量操作栏
- 前置条件：至少选中 1 条状态=`uploaded_reviewing` 的资料项
- 结果：选中项状态→`approved` → toast "已审核通过 N 项"

### 8.7 批量标记 waived

- 触发入口：跨案件列表批量操作栏
- 前置条件：至少选中 1 条
- 弹出 waived 原因弹窗（§8.3），原因码统一应用于全部选中项
- 结果 → toast "已标记 N 项为无需提供"

## 9 过期联动规则（P0 强口径）

- 当一个被多个资料项引用的版本到达 `expiry_date` 或被标记过期时，所有当前引用它的资料项统一转为 `expired`
- 受影响案件的校验通过结果失效，并生成"过期重交/重新确认"提醒
- 资料中心跨案件列表应把此类记录聚合为"共享版本过期"风险卡片（摘要卡 #4）
- 风险说明面板展示：过期版本信息、引用该版本的案件列表、建议操作

## 10 列表页批量选择机制

- 表头全选 checkbox（支持 indeterminate 态）
- 每行独立 checkbox（waived / approved 行 checkbox `disabled` + `opacity-60`）
- 选中计数显示 + 清除按钮
- 批量操作栏仅在 selectedCount > 0 时可见
- 操作栏包含：批量催办 / 批量审核通过 / 批量标记 waived

## 16 跨模块一致性

### 16.1 案件详情资料 Tab

案件详情 `case-detail-config.js` 中每个 `DETAIL_SAMPLES` 场景的 `documents` 数据需与 `documents-config.js` 的状态枚举和 badge 映射保持一致。状态 key 映射见 §6.4。

### 16.2 仪表盘

仪表盘「今日待办」和「待审核资料」卡片中的计数与文案需与资料中心摘要卡的口径一致。

### 16.3 任务模块

催办动作生成的催办记录与任务模块的催办来源需保持 `source_type` 一致。资料状态变更（如过期触发重交）可能生成关联任务。

### 16.4 导航高亮

- 从跨案件资料中心（`documents/index.html`）访问时：「资料中心」导航项高亮
- 从案件详情资料 Tab 访问时：「案件」导航项高亮（由案件详情页管理）
