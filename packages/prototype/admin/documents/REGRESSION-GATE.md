# 资料中心模块回归验收门槛

> 权威来源：[P0-CONTRACT.md](./P0-CONTRACT.md) + [资料中心.md](../../../../docs/gyoseishoshi_saas_md/P0/06-页面规格/资料中心.md)
>
> 用途：后续实现阶段（UI 迁移、脚本拆分、跨模块连通）的**硬性验收门槛**。每轮实现提交前必须逐项确认本清单对应范围全部通过，否则不可合入。
>
> 标记说明：
> - `[原型]` — 原型中可直接视觉/交互验证
> - `[生产约束]` — 生产代码实现时需满足，原型仅在文档中记录
> - `[跨模块]` — 涉及其他模块（仪表盘、案件详情、任务提醒）的一致性

## 验收流程

### 每轮实现提交前

1. **自检**：开发者按 Gate 1–7 逐项确认自己的实现范围
2. **回归**：至少覆盖当前实现范围涉及的所有 Gate 项
3. **标记**：在本文档中标注已通过的项（`[x]`），未通过的项必须修复后再提交

### 最终验收（全部实现完成后）

1. 本文档全部 `[原型]` 项通过
2. `P0-CONTRACT.md` 拆分回归清单全部通过
3. `split-manifest.json` 的 `regressionChecklist` 全部通过
4. `npm run fix` + `npm run guard` 通过

### Gate 失败处理

- 任何 Gate 中有 `[原型]` 项未通过：该轮实现 **不可合入**，修复后重新验收
- `[生产约束]` 项在原型阶段可标记为 `[deferred]`，但必须在 `MIGRATION-MAPPING.md` 中有对应的生产实现说明
- `[跨模块]` 项若受限于其他模块未就绪，标记为 `[blocked: 模块名]`，待对方就绪后补验

### 已知限制

1. **Toast 仅弹出反馈，不变更 DOM 状态**：审核通过/退回/waived 的 toast 弹出后，表格行的 badge 不会实时更新（demo-only 限制，无后端持久化）。

3. **共享版本过期联动为静态标注**：demo 数据中 `sharedVersionExpiry: true` 为手动标记，非运行时联动计算。
4. **案件详情「登记资料」按钮仍指向旧页面**：`data-navigate="../documents.html"` 路径为旧入口，保留不修改（属于 knownLegacyLinks 已记录）。
