## 2 信息架构缺口（Architecture Gap）

> 旧 `leads-messages.html` 采用"左侧线索列表 + 右侧会话面板"的 IM 风格布局。
> 规格文档要求"列表页 + 独立详情页"的标准 CRUD 架构。

| # | 维度 | 规格要求 | 旧页现状 | 缺口说明 | 优先级 |
|---|------|---------|---------|---------|--------|
| A.1 | 页面结构 | 列表页 (`index.html`) + 详情页 (`detail.html`) 双入口 | 单页 chat 式布局，无独立详情页 | **整体架构不匹配**，需全部重建为表格列表页 | P0 |
| A.2 | 列表展示 | 表格 + 表头 + 多列排列 | 卡片式单列列表（仅显示名字、预览、状态） | 需改为规格定义的 11 列表格 | P0 |
| A.3 | 右侧内容区 | 详情页独立入口 | 会话面板（chat bubble + 输入区） | 右侧会话面板废弃；详情页另建 | P0 |
| A.4 | 页面标题区 | 标题 + 副标题 + `新建线索` 主按钮 | 面板内嵌标题 + 浮动 `+` 按钮 | 新页需顶部 header 区 | P0 |

## 16 Deferred Risk（延迟风险）

| 风险项 | 影响范围 | 缓解措施 |
|--------|---------|---------|
| 旧入口 `leads-messages.html` 与新模块并存 | 用户可能访问旧页面看到过时 UI | 在 manifest 中记录；后续导航统一任务处理重定向 |
| 主导航未统一改为新路径 | 从其他页面侧边栏进入仍会到旧页 | 本轮新页面自身导航指向正确路径；全局统一后续处理 |
| 详情页尚未实现时列表行点击无目标 | 列表页先于详情页完成 | 列表行跳转使用 `detail.html?id=xxx`；详情页未就绪时降级为 `#` + toast 提示 |
| 去重逻辑为前端静态匹配 | 与真实后端去重行为可能不一致 | 在 demo-only 标注中明确说明；`data/leads-config.js` 中预设匹配规则 |
