## 2 信息架构缺口（Architecture Gap）

> 旧 `leads-messages.html` 是"左侧线索列表 + 右侧会话面板"的 IM 风格布局，**完全没有独立详情页**。
> 规格 §3 要求独立的详情页，含 4 个 Tab（基础信息 / 跟进记录 / 转化信息 / 日志）。

| # | 维度 | 规格要求 | 旧页现状 | 缺口说明 | 优先级 |
|---|------|---------|---------|---------|--------|
| A.1 | 详情页入口 | 独立 `detail.html`，从列表行点击进入 | 无详情页；点击 lead-item 仅切换右侧会话面板 | **整体缺失**，需全新构建 | P0 |
| A.2 | Header 区 | 线索标题 + 状态 badge + 负责人 + Group + 转化主操作 | 对话面板 header 仅有头像/姓名/来源/两个按钮 | 需全新构建标准详情 header | P0 |
| A.3 | Tab 导航 | 4 个 Tab：基础信息 / 跟进记录 / 转化信息 / 日志 | 无 Tab 导航 | 需全新构建 | P0 |
| A.4 | 只读状态横幅 | 已流失 / 已签约未转化等状态提示 banner | 无 | 需全新构建 | P0 |
| A.5 | 面包屑 | 咨询线索 → {联系人姓名} | 无 | 需全新构建 | P0 |

## 15 Deferred Risk（延迟风险）

| 风险项 | 影响范围 | 缓解措施 |
|--------|---------|---------|
| 旧入口 `leads-messages.html` 与新模块并存 | 用户可能访问旧页面看到过时 UI | 在 manifest 中记录；后续导航统一任务处理重定向 |
