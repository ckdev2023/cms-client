## 5 跟进记录 Tab

### 5.1 跟进时间线

每条记录必须包含：

| 字段 | 说明 |
|------|------|
| 渠道标识 | 胶囊 chip：电话 / 邮件 / 面谈 / IM（各有独立颜色） |
| 跟进摘要 | 文本 |
| 结论 | 文本 |
| 下一步动作 | 文本 |
| 下次跟进时间 | 日期 |
| 跟进时间 | 时间戳 |
| 一键转任务 | 每条记录右侧小按钮（demo-only toast） |

样例数据至少 3 条，覆盖电话、邮件、面谈三种渠道。

### 5.2 新增跟进记录表单

`.apple-card` 包裹的录入区域：

| 字段 | 控件 | 必填 |
|------|------|------|
| 渠道 | select（电话/邮件/面谈/IM） | 是 |
| 摘要 | textarea | 是 |
| 结论 | text input | 否 |
| 下一步动作 | text input | 否 |
| 下次跟进时间 | datetime-local | 否 |

提交后：toast + DOM 追加到时间线顶部（demo-only）。

### 5.3 空态

无跟进记录时显示引导"录入第一条跟进记录" + 跟进录入表单展开。

## 6 转化信息 Tab

### 6.1 去重匹配面板

| 场景 | 展示内容 |
|------|---------|
| 电话匹配到已有 Lead | Lead 摘要卡（姓名 + 编号 + 电话 + Group） + "继续创建"/"查看已有" |
| 邮箱匹配到已有 Customer | Customer 摘要卡（姓名 + 编号 + 邮箱 + Group） + "继续创建"/"查看已有" |
| 无匹配 | 说明文案"未检测到重复记录" |

面板样式：`.apple-card` 内嵌，橙色/黄色背景。

### 6.2 转客户操作（Lead → Customer）

| # | 要求 | 说明 |
|---|------|------|
| 1 | 操作入口 | Header 主操作按钮"转客户" |
| 2 | 确认弹窗 | 标准 `.apple-modal`：预填信息 + Group 继承提示 |
| 3 | Group 默认继承 | Lead.group → Customer.group（03 §2.2） |
| 4 | 改组留痕 | 弹窗内可改 Group，需说明原因（demo-only） |
| 5 | 转化后回填 | Customer ID + 跳转链接回填到 Tab 3 |
| 6 | 状态变更 | 线索状态更新（通过样本切换展示） |

### 6.3 转案件操作（Lead → Case）

| # | 要求 | 说明 |
|---|------|------|
| 1 | 操作入口 | Header 主操作按钮"转案件" |
| 2 | 确认弹窗 | 标准 `.apple-modal`：选择案件类型 + 主申请人预填 + 负责人指定 |
| 3 | Case.group 继承 | Customer.group → Case.group（03 §2.2） |
| 4 | 转化后回填 | Case ID + 跳转链接回填到 Tab 3 |
| 5 | 状态变更 | 线索状态标记已转案件（通过样本切换展示） |

### 6.4 转化后展示

| # | 内容 | 展示方式 |
|---|------|---------|
| 1 | 已转化 Customer 卡片 | `.apple-card`：客户姓名 + 编号 + 跳转按钮 |
| 2 | 已转化 Case 卡片 | `.apple-card`：案件标题 + 编号 + 类型 + 跳转按钮 |
| 3 | 多次转化记录 | 列表展示多条转化记录（同一 Lead 转多个 Case） |
| 4 | Header 按钮动态切换 | 已转化后"转客户"变为"查看客户"，"转案件"变为"查看案件" |

## 8 状态与异常态

必须保留以下 6 种状态：

| # | 状态 | 展示方式 |
|---|------|---------|
| 1 | 正常跟进态（new / following / pending_sign） | Header 状态 badge + Tab 1 字段 + Tab 2 跟进时间线 + 跟进录入入口 |
| 2 | 已流失态 | 只读 banner"该线索已标记为流失，仅供查阅" + 灰色状态 badge + 全字段不可编辑 |
| 3 | 已签约未转化态 | Warning banner"该线索已签约但尚未创建客户/案件，请完成转化" + 转化按钮高亮 |
| 4 | 已转客户态 | Tab 3 已转化 Customer 卡片 + 跳转入口 + Header 按钮变为"查看客户" |
| 5 | 已转案件态 | Tab 3 已转化 Case 卡片 + 跳转入口 + Header 按钮变为"查看案件" |
| 6 | 空跟进记录态 | Tab 2 空态引导"录入第一条跟进记录" |

### 8.1 只读 Banner

- 浅灰底 `--surface-2` + 锁图标 + 文案
- 不可关闭
- 已流失态时显示
- `pointer-events: none` / `opacity` 演示只读

### 8.2 Warning Banner

- 浅黄底 `rgba(245,158,11,0.1)` + 警告图标 + 文案 + 行动按钮
- 不可关闭
- 已签约未转化态时显示

## 17 Deferred Risk（延迟风险）

| 风险项 | 影响范围 | 缓解措施 |
|--------|---------|---------|
| 旧入口 `leads-messages.html` 与新模块并存 | 用户可能访问旧页面看到过时 UI | 在 manifest 中记录；后续导航统一任务处理重定向 |
| 列表页尚未实现时详情页无法从列表进入 | 详情页可能先于列表页完成 | 直接通过 URL 访问 `detail.html`；样本切换覆盖主要场景 |
| 去重逻辑为前端静态匹配 | 与真实后端去重行为可能不一致 | 在 demo-only 标注中明确说明；预设匹配规则覆盖电话/邮箱两种场景 |
| 转化链路无真实数据写入 | 转化后切换页面会丢失状态 | 通过样本切换独立展示"已转化"场景；toast 提供视觉反馈 |
| 跟进记录与任务系统未真实联通 | 一键转任务仅为 UI 演示 | toast + 文案说明 demo-only |
| Tab 2 跟进记录结构与旧页 chat 布局差异大 | 从旧页迁移时用户体验跳跃 | 在 MIGRATION-MAPPING-DETAIL.md 中记录旧 chat → 新时间线的对照关系 |
