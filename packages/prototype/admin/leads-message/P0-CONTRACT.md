| # | 区块 | 说明 |

| 1 | 页面头部 | 标题"咨询线索"、副标题、`新建线索` 主按钮 |
| 2 | 筛选面板 | 状态 / 负责人 / Group / 业务类型 / 跟进时间范围 / 搜索 / 重置 |

## 2 列表字段（咨询线索.md §2.1）

### 2.1 表格列定义

| # | 列头 | 规格字段 | 呈现方式 |
|---|------|---------|---------|
| 1 | 线索编号 | 编号 | 辅助文本 |
| 2 | 联系人姓名 | 姓名 | 主文案 |
| 3 | 电话 / 邮箱 | 联系方式 | 辅助行 |
| 4 | 意向业务类型 | 业务类型 | 文本 |
| 5 | 来源 / 介绍人 | 来源 | 文本 |
| 6 | 当前状态 | 线索状态 | Status Badge（6 种） |
| 7 | 负责人 | 负责人 | 头像缩写 + 姓名 |
| 8 | 所属 Group | Group | chip |
| 9 | 下一步动作 | 下一步 | 文本 |
| 10 | 下次跟进时间 | 跟进时间 | 日期 |
| 11 | 最近更新时间 | 更新时间 | 日期 |

拆分时不可删减上述列，也不可引入规格外的新主列。

### 2.2 状态 Badge 配色

| 状态 | 枚举值 | 配色 |
|------|--------|------|
| 新咨询 | `new` | `--warning`（橙色） |
| 跟进中 | `following` | `--primary`（蓝色） |
| 待签约 | `pending_sign` | 紫色 chip |
| 已签约 | `signed` | `--success`（绿色） |
| 已流失 | `lost` | 灰色 |
| 已签约未转化 | `signed`（高亮） | `--warning` 高亮 |

## 3 搜索与筛选（咨询线索.md §2.2）

### 3.1 筛选维度

| # | 筛选维度 | 控件类型 | 默认值 | 最少选项 |
|---|---------|---------|-------|---------|
| 1 | 当前状态 | select | 全部 | 全部 + 6 种状态 |
| 2 | 负责人 | select | 全部 | 全部 + 至少 3 人（Suzuki / Tanaka / Sato） |
| 3 | 所属 Group | select | 全部 | 全部 + 至少 2 组（G1 / G2） |
| 4 | 意向业务类型 | select | 全部 | 全部 + 至少 6 类（高度人才 / 技人国 / 家族滞在 / 设立法人 / 永住 / 其他） |
| 5 | 跟进时间范围 | date range | 空 | 起止日期 |
| 6 | 搜索（全文） | text input | 空 | — |
| 7 | 重置 | button | — | 恢复所有筛选到默认值 |

### 3.2 搜索范围

- 联系人姓名
- 电话 / 邮箱
- 线索编号
- 意向业务类型

## 4 批量动作（咨询线索.md §2.3）

### 4.1 批量选择机制

- 表头全选 checkbox：`selectAllLeads`
- 每行独立 checkbox：`data-lead-select`
- 已选计数：`selectedCount`
- 清除按钮：`bulkClearBtn`
- 批量操作栏仅在 `selectedCount > 0` 时可见
- 全选需支持 `indeterminate`

### 4.2 批量动作列表

| # | 动作 | 控件 | 约束 |
|---|------|------|------|
| 1 | 批量指派负责人 | select + apply button | 需至少选中 1 条 |
| 2 | 批量调整下次跟进时间 | datetime input + apply button | 需至少选中 1 条 |
| 3 | 批量标记状态 | select + apply button | 需至少选中 1 条 |

## 5 新建线索弹窗（咨询线索.md §4）

### 5.1 字段

| # | 字段 | 控件 | 必填 | 说明 |
|---|------|------|------|------|
| 1 | 联系人姓名 | text input | 是 | — |
| 2 | 电话 | tel input | 条件必填 | 电话/邮箱至少填一项 |
| 3 | 邮箱 | email input | 条件必填 | 电话/邮箱至少填一项 |
| 4 | 来源 | select | 否 | Web / 介绍 / 来访 / 电话 / 其他 |
| 5 | 介绍人 | text input | 条件显示 | 来源为"介绍"时显示 |
| 6 | 意向业务类型 | select | 否 | 高度人才 / 技人国 / 家族滞在 / 设立法人 / 永住 / 其他 |
| 7 | 归属 Group | select | 否 | G1 / G2 |
| 8 | 负责人 | select | 否 | Suzuki / Tanaka / Sato |
| 9 | 下一步动作 | text input | 否 | — |
| 10 | 下次跟进时间 | datetime-local | 否 | 默认明天 10:00 |
| 11 | 备注 | textarea | 否 | — |
| 12 | 首选语言 | select | 否 | 保留（demo 增值，规格未明确要求） |

### 5.2 校验规则

- 姓名为空时不可保存
- 电话与邮箱都为空时不可保存，toast 提示"请至少填写电话或邮箱"

### 5.3 去重提示面板

弹窗内嵌去重结果区域，触发条件为电话或邮箱输入后匹配预设规则：

| 场景 | 匹配依据 | 展示内容 |
|------|---------|---------|
| 命中已有 Lead | 电话匹配 | 匹配到的 Lead 摘要卡（姓名 + 编号 + 电话 + Group） |
| 命中已有 Customer | 邮箱匹配 | 匹配到的 Customer 摘要卡（姓名 + 编号 + 邮箱 + Group） |
| 人工确认继续 | 用户选择 | "继续创建"按钮 + "查看已有"按钮 |

去重面板样式：黄色/橙色背景卡片（`border-[#fde68a] bg-[#fffbeb]`），与客户模块去重面板保持一致。

## 6 页面入口与单行操作

## 8 行高亮与条件样式

| 条件 | 样式 |

## 11 Toast / 反馈

| 触发场景 | title | desc |
|----------|-------|------|
| 创建线索成功 | 已创建 | 新线索已添加到列表 |
| 电话/邮箱缺失 | 无法创建 | 请至少填写电话或邮箱 |
| 批量指派负责人 | 批量指派负责人（示例） | 已选择 N 条，负责人：xxx |
| 批量调整跟进时间 | 批量调整跟进时间（示例） | 已选择 N 条，下次跟进：yyyy-mm-dd |
| 批量标记状态 | 批量标记状态（示例） | 已选择 N 条，状态已更新为：xxx |
| 去重命中 | 检测到可能重复 | 电话/邮箱匹配到已有记录，请确认 |
| 转化成功 | 转化成功（示例） | 已创建客户/案件，可在详情页查看 |

## 17 Deferred Risk（延迟风险）

| 风险项 | 影响范围 | 缓解措施 |
|--------|---------|---------|
| 旧入口 `leads-messages.html` 与新模块并存 | 用户可能访问旧页面看到过时 UI | 在 manifest 中记录；后续导航统一任务处理重定向 |
| 主导航未统一改为新路径 | 从其他页面侧边栏进入仍会到旧页 | 本轮新页面自身导航指向正确路径；全局统一后续处理 |
| 详情页尚未实现时列表行点击无目标 | 列表页先于详情页完成 | 列表行跳转使用 `detail.html?id=xxx`；详情页未就绪时降级为 `#` + toast 提示 |
| 去重逻辑为前端静态匹配 | 与真实后端去重行为可能不一致 | 在 demo-only 标注中明确说明；预设匹配规则覆盖电话/邮箱两种场景 |
