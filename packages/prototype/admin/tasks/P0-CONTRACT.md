## 1 信息架构与导航

### 1.1 页面入口

| 入口 | 位置 | 说明 |
|------|------|------|
| 全局入口 | 左侧主导航「任务与提醒」 | 进入任务列表页 |
| 仪表盘入口 | 仪表盘「今日待办」卡片 | 跳转到任务列表页（预筛「今日到期」） |
| 案件内入口 | 案件详情「任务」Tab | 带案件上下文（P0 仅预留链接入口，不实现独立 Tab） |

### 1.2 导航返回规则

- 从**全局入口**进入任务详情：返回后回到任务列表，保留进入前的筛选条件与滚动位置
- 从**案件详情任务 Tab**进入任务详情：返回后回到该案件的「任务」Tab，保留案件上下文

> **原型演示说明**：原型中返回行为通过简单导航实现，不做滚动位置 / 筛选条件保持。仅演示跳转路径。

## 2 列表页必保留字段

| # | 列头 | 规格字段 | 原型中的呈现 |

## 3 默认排序与展示规则

### 3.1 默认排序

- 逾期任务置顶
- 其次按截止时间升序
- 无截止时间的任务置底

### 3.2 逾期展示

- 截止时间列显示"逾期 X 天"，以风险色（`tag-red`）强调
- 原型中使用 `tag-red` 静态标记，demo 数据中硬编码逾期天数

### 3.3 来源标签

- 列表行内以轻量标签展示来源（模板/催办/校验失败/补正/续签提醒/手动），便于快速识别系统生成 vs. 手动创建任务

### 3.4 「本周到期」标签处理

- 保留 `tag-blue`"本周到期"作为行内展示标签
- **不**将其升级为侧边栏视图入口或筛选项（P0 明确不做本周到期视图）
- 在 `tasks-config.js` 的到期 tag 配置中标注为"行内标签，非视图"

## 4 列表页搜索与筛选

### 4.1 搜索

- 范围：任务名称 / 案件编号 / 责任人
- 控件：`search-input`，位于筛选栏

### 4.2 筛选器

| # | 筛选维度 | 控件类型 | 默认值 |
|---|---------|---------|-------|
| 1 | 完成状态 | select（todo / doing / done / canceled / 全部） | 全部 |
| 2 | 截止时间范围 | select（今日 / 逾期 / 全部） | 全部 |
| 3 | 责任人 | select | 全部 |
| 4 | 所属 Group | select | 全部 |

> **P0 不做「本周到期」筛选项**——规格明确后置至 P1。

### 4.3 重置

- 筛选栏末尾「重置」按钮，清空所有筛选条件

## 5 工作台视图

工作台侧边栏提供以下快捷视图入口，本质是预置筛选条件的快捷方式：

| # | 视图 | 语义 | 计数 badge |
|---|------|------|-----------|
| 1 | 我的待办 | 责任人=当前用户 且 状态∈{todo, doing} | 显示计数 |
| 2 | 今日到期 | 截止日=今天 且 状态∈{todo, doing} | 显示计数 |
| 3 | 已逾期 | 截止日 < 今天 且 状态∈{todo, doing} | 显示计数 |
| 4 | 提醒日志 | 切换到提醒日志面板 | — |

点击视图入口时，主列表区的筛选条件同步切换。当前选中视图高亮。

> **原型现状差距**：侧边栏已有四个入口并显示 badge 计数。结构基本对齐，拆分时保留并确保视图切换联动筛选。

## 6 列表页批量动作

| # | 动作 | 控件 | 约束 |
|---|------|------|------|
| 1 | 批量指派责任人 | select + 应用按钮 | 需至少选中 1 条任务 |
| 2 | 批量调整截止日 | date picker + 应用按钮 | 需至少选中 1 条任务，变更须留痕 |
| 3 | 批量完成 | 按钮 | 需至少选中 1 条任务，状态→done |
| 4 | 批量取消 | 按钮 + 原因输入 | 需至少选中 1 条任务，状态→canceled，须填原因 |

### 6.1 批量选择机制

- 表头全选 checkbox（支持 indeterminate 态）
- 每行独立 checkbox
- 选中计数显示 + 清除按钮
- 批量操作栏仅在 selectedCount > 0 时可见

### 6.2 批量动作权限约束

- 仅对"当前用户具备编辑权限"的任务生效
- 选中任务中若存在不可编辑项，需明确提示不可编辑原因，并跳过该项
- 批量取消必须填写原因（枚举 + 备注可选），并写入操作记录与审计事件

> **原型现状差距**：现有原型有行内 checkbox 但无批量操作栏。拆分时需补齐全选/indeterminate/批量操作栏。

## 11 状态机（P0 口径）

### 11.1 状态定义

| 状态 | 含义 | 备注 |
|------|------|------|
| **todo** | 待处理（默认） | 新建任务的初始状态 |
| **doing** | 进行中 | 可选，P0 不强制使用；用于跨天/需持续跟进的任务 |
| **done** | 已完成 | 记录完成时间 |
| **canceled** | 已取消 | 必须记录取消原因（见 §7 取消原因枚举） |

### 11.2 允许的状态流转

| 当前状态 | 可流转至 |
|----------|---------|
| todo | doing / done / canceled |
| doing | done / todo / canceled |
| done | **不可回退** |
| canceled | **不可回退** |

> done / canceled 如需更正，创建新任务并在描述中关联原因。

### 11.3 截止与逾期判定

- 任务"逾期"= 存在截止时间 **且** 截止时间早于当前日期（以租户时区为准）**且** 状态 ∈ {todo, doing}
- done / canceled 不计入逾期与待办统计

## 12 关键动作

| 动作 | 触发条件 | 结果 | 权限 |
|------|---------|------|------|
| 创建任务 | 手动或系统自动 | 新建任务关联案件 | 主办人、助理 |
| 完成任务 | 任务处理完毕 | 状态→done，记录完成时间 | 任务责任人 |
| 指派责任人 | 新建或重新指派 | 更新责任人，站内通知新负责人 | 主办人 |
| 调整截止日 | 需要延期或提前 | 更新截止日并留痕 | 主办人 |
| 取消任务 | 任务不再需要 | 状态→canceled，须填原因 | 主办人 |

## 13 站内提醒与自动生成（P0 口径）

### 13.1 站内提醒触发（最小集合）

| 触发场景 | 发送时机 | 内容要素 |
|---------|---------|---------|
| 指派责任人 | 立即 | 任务名、截止时间、关联案件入口 |
| 今日到期提醒 | 工作日固定时间（参考 [03 §11](../../../../docs/gyoseishoshi_saas_md/P0/03-业务规则与不变量.md#11-提醒规则)） | 截止日为今天且未完成的任务 |
| 逾期提醒 | 每日（频率与时间参考 [03 §11](../../../../docs/gyoseishoshi_saas_md/P0/03-业务规则与不变量.md#11-提醒规则)） | 逾期且未完成任务 |

### 13.2 系统自动生成任务的去重策略

- 对"模板/催办/校验失败/补正/续签提醒"等系统来源的任务，必须具备可追溯的来源键（source key）
- **去重口径**：同一案件 + 同一来源 + 同一来源键，若存在状态 ∈ {todo, doing} 的任务，则不重复生成；若已 done / canceled，则允许再次生成

> **原型演示说明**：原型中提醒触发和自动生成为 demo-only 静态数据，不实现真实调度或去重逻辑。

## 14 状态与异常态

| # | 状态 | 规格要求 | 原型现状 |
|---|------|---------|---------|
| 1 | 空状态（无任务） | 显示"暂无待办"引导 | 已预留（`#taskEmptyState` 占位，demo 数据存在时 `hidden`） |
| 2 | 逾期未完成 | 列表标红逾期任务，仪表盘计入风险 | 部分实现（有 `badge-danger` 逾期标签，无仪表盘联动） |
| 3 | 批量自动生成 | 模板建案或补正通知后系统批量生成任务，需确保不重复 | 未实现（原型仅静态数据） |
| 4 | 提醒发送失败 | 提醒日志中标记失败原因 | 已在 `reminder-log-panel.html` 中实现失败行展示 |
| 5 | 权限不足 | 对不可编辑任务隐藏/禁用关键动作；批量动作时明确提示不可编辑原因并跳过该项 | 已预留（done/canceled 行 checkbox `disabled` + `opacity-60`） |
| 6 | 并发修改 | 任务已被他人更新时，提示刷新后重试，避免覆盖最新更改 | 未实现（原型无并发检测） |

拆分后必须保留 **状态 2** 和 **状态 4** 的视觉标识，并为 **状态 1**、**状态 5** 预留空状态与禁用态占位。状态 3 和 6 为生产约束，原型仅在文档中记录。
